# Milestone v0.7.0: Code Review

**Status:** ✅ SHIPPED 2026-05-27
**Phases:** 24–29 + 29.1 (9 phases total)
**Total Plans:** 22 plans

## Overview

Code review mode: inspect, navigate, and annotate agent-generated diffs before a PR, then return structured feedback to the agent. Adds a `/code-review` route with full branch diff viewer, commit navigation, inline hunk/file comments, and review submission — all wired into Claude Code via slash command and pre-PR hook.

## Milestone Summary

**Decimal Phases:**

- Phase 26.1: Commit Navigation Bug Fixes (inserted after Phase 26 to fix code review findings before Phase 27)
- Phase 26.2: Commit Navigation UX Polish (inserted after Phase 26.1 to address 5 UX issues found in testing)
- Phase 29.1: Fix POST /api/decide schema mismatch (inserted after Phase 29 to fix critical 422 regression blocking agent feedback path)

**Key Decisions:**

- Single "Send Review" button over Approve/Request-Changes split (D-06 design pivot) — user direction during Phase 28 human verification; agent decides outcome from `{message?,comments?}` payload
- `post_decide` handler accepts `serde_json::Value` with key-presence dispatch (Phase 29.1) — unblocks code-review path without frontend changes; plan-review path retains strict `behavior` field validation (422 on missing key)
- `connectivity.ts` + `useHeartbeat.ts` moved to `ui/src/shared/` (Phase 28) — enables code-review and reviewer-v2 to share heartbeat without coupling; ESLint rule updated accordingly
- `pre-pr-hook` filters on `gh pr create` + `git push` (excluding `--tags`/`--delete`) — precise trigger set confirmed by unit tests in Phase 29

**Issues Resolved:**

- POST /api/decide schema mismatch (422 on every code-review submit) — fixed by Phase 29.1 key-presence dispatch
- ArrowLeft/ArrowRight keyboard guard wrong direction — fixed in Phase 26.1
- SHA validation missing in fetchCommitDiffOnce — added in Phase 26.1 (CR-01)
- `checkedCommitShas` re-seeding on user deselect — fixed with `seededRef` sentinel in Phase 26.1

**Issues Deferred:**

- ARIA violation: `<li role='button'>` containing `<input type='checkbox'>` in CommitRow (26.1 WR-01)
- Empty commit selection vacuously triggers error panel (26.1 post-phase CR-01)
- `contextExpanded` not reset on selector change (26.2 CR-01)
- `Cmd+Enter` in CodeReviewSubmitPopover bypasses `canSend` gate (Phase 28 warning)
- No re-entry guard on `handleSend` — concurrent POST possible on double-click (Phase 28 warning)
- 7/8 phases missing VALIDATION.md (Nyquist compliance)

**Technical Debt Incurred:**

- Pre-existing ESLint errors in CodeReviewApp.tsx, AppToolbar.tsx, useDiff.ts (not introduced by v0.7.0)
- `try_list_commits` walks ALL commits when base_oid is None (Phase 24 WR-02)
- `previous_filename` not set for Copied deltas (Phase 24 WR-03)
