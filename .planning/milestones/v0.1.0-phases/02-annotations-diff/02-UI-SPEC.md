## Layout Contract

### Overall structure (Phase 2 replaces Phase 1 single-column)

```
┌─────────────────────────────────────────────────────────────────┐
│ HEADER (48px, sticky, var(--color-surface))                     │
│  "Plan Review"                          [Plan] [Diff] tabs      │
├────────────────────────────────────┬────────────────────────────┤
│ PLAN / DIFF CONTENT (flex-grow)    │ ANNOTATIONS SIDEBAR        │
│  max-width: no cap (fills space)   │  width: 320px, fixed       │
│  padding: 32px                     │  padding: 16px             │
│  overflow-y: auto                  │  overflow-y: auto          │
│                                    │  background: var(--color-  │
│                                    │    surface)                │
│                                    │  border-left: 1px solid    │
│                                    │    var(--color-border)     │
├────────────────────────────────────┴────────────────────────────┤
│ ACTION BAR (sticky bottom, var(--color-surface))                │
│  [Approve ↵ Enter]  [Deny]                                      │
└─────────────────────────────────────────────────────────────────┘
```

- The outer container is `display: flex; flex-direction: column; min-height: 100vh` — full width, no max-width cap (Phase 1 had 900px cap; Phase 2 removes it to support the two-column layout).
- Content area: `display: flex; flex-direction: row; flex-grow: 1; overflow: hidden`.
- Left column (plan/diff): `flex-grow: 1; overflow-y: auto`.
- Right column (sidebar): `width: 320px; flex-shrink: 0; overflow-y: auto`.
- Sidebar is always visible during the `reviewing` state, regardless of which tab is active. It collapses to zero width (or becomes an empty panel with the global comment field only) in the Diff tab.
- Primary focal point: the plan content area (left column) — the action bar CTAs are the secondary focal point that draws the eye after reading.

### Tab bar

Tabs live in the header, right-aligned. Two tabs: **Plan** (default active) | **Diff**. Tab switching is `useState<'plan' | 'diff'>`. No URL change, no react-router.

Tab anatomy:
- Height: 48px (fills header)
- Padding: 0 16px
- Active state: `color: var(--color-tab-active)`, bottom border `2px solid var(--color-tab-active)`
- Inactive state: `color: var(--color-tab-inactive)`, no border
- Font: 16px weight 600 (active) / 400 (inactive)
- Gap between tabs: 4px

### Annotations sidebar

Sidebar sections (top to bottom):
1. **Overall Comment field** — always visible, pinned at top. Label: "Overall Comment". Textarea: 80px min-height, resizable vertically. No character limit.
2. **Divider** — `1px solid var(--color-border)`, 16px vertical margin.
3. **Annotation list** — ordered by anchor position (top-to-bottom in plan). Each annotation is a card.
4. **Empty state** — shown when annotation list is empty: "Select text in the plan to add an annotation." (text-secondary, centered, 14px).

### Annotation card anatomy

```
┌─[type badge]──────────────────────[× remove]─┐
│ "[Quoted anchor text]"                        │
│ [comment textarea or replacement text]        │
└───────────────────────────────────────────────┘
```

- Card background: `var(--color-surface)`
- Card border: `1px solid var(--color-border)`
- Card border-radius: 6px
- Card left border: 3px solid — color from annotation type (comment: blue, delete: red, replace: amber)
- Card padding: 8px
- Type badge: 14px, weight 600, uppercase, matching left-border color, background `rgba(color, 0.15)`, border-radius 4px, padding 4px 8px
- Quoted anchor: 14px, monospace, color text-secondary, truncated to 2 lines with `text-overflow: ellipsis`
- Remove button: `×` symbol, 16px, text-secondary, top-right corner, no background, hover: text-primary

### "Add annotation" affordance

When the user has selected text in the plan:
- A floating button group appears in the sidebar at the position corresponding to the selection, or fixed below the global comment divider if sidebar scroll differs.
- Button group shows three inline pills: **Comment** | **Delete** | **Replace**
- Each pill: 14px, weight 600, 32px height, 12px horizontal padding, border-radius 4px
- Comment pill: `--color-annotation-comment` background at 20% opacity, text at full color
- Delete pill: `--color-accent-deny` background at 20% opacity, text at full color
- Replace pill: `--color-annotation-replace` background at 20% opacity, text at full color
- Clicking a pill creates an annotation card for that type and clears the selection

Implementation note: use `document.getSelection()` + `selectionchange` event. Store selection as `{ anchorText: string, startOffset: number, endOffset: number }` keyed against the plan HTML text content.

### Anchor highlight interaction

When the user hovers an annotation card:
- The corresponding anchor text in the plan receives `background: var(--color-annotation-highlight)` and `border-radius: 3px`.
- Implemented via a CSS class `.annotation-highlighted` applied to a `<mark>` element wrapping the anchor span.
- On card hover-out: class removed.

### Diff tab view

When Diff tab is active:
- Left column renders the `@pierre/diffs` component with the raw unified diff string from `GET /api/diff`.
- Container: `width: 100%; overflow-x: auto; background: var(--color-code-bg); border-radius: 8px; padding: 0`.
- `@pierre/diffs` renders its own Shadow DOM — do not attempt to style internals via global CSS.
- Annotations sidebar remains visible but only the Overall Comment field is shown (annotation list is hidden with `display: none` in diff tab — text selection not supported across diff view).
- Empty state (no diff): centered in the left column, see Copywriting section.

## Annotation Serialization Contract (for deny message)

This is the output format Claude reads. The frontend assembles this string client-side before `POST /api/decide`.

```
{textarea text, if any}

## Overall Comment
{global comment, if any}

## Annotations

### Comment on: "{quoted anchor text}"
{comment text}

### Delete: "{quoted anchor text}"

### Replace: "{quoted anchor text}"
With: {replacement text}
```

Rules:
- `## Overall Comment` section omitted if global comment is empty.
- `## Annotations` section omitted if no anchored annotations.
- Individual `### …` entries omitted if that annotation was removed.
- If no annotations at all: message = textarea text only (preserves Phase 1 behavior).
- Textarea is optional when at least one annotation exists; required when zero annotations exist.
- Sections separated by a single blank line.

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Text selected in plan | `selectionchange` event, selection non-empty | Add annotation affordance appears in sidebar |
| Annotation card hovered | `mouseenter` on card | Anchor text in plan gets `--color-annotation-highlight` background |
| Annotation card hover cleared | `mouseleave` on card | Highlight removed |
| Deny textarea: no annotations, empty textarea | Default | Submit Denial button at opacity 0.4, pointer-events none |
| Deny textarea: has annotations, empty textarea | Annotation added | Submit Denial button active (opacity 1, pointer-events auto) |
| Tab: Plan active | Default on load | Plan tab label weight 600, bottom border 2px white, Diff label weight 400 no border |
| Tab: Diff active | Click Diff tab | Diff tab label weight 600, bottom border 2px white, Plan label weight 400 no border |
| Diff tab: no diff | GET /api/diff returns empty string | Empty state copy shown, centered |
| Diff tab: error | GET /api/diff fetch fails | Error state copy shown, centered |

## Accessibility Contract

- All annotation type pills have explicit `aria-label` attributes: "Add Comment annotation", "Add Delete annotation", "Add Replace annotation".
- Annotation cards have `role="listitem"` inside a `role="list"` wrapper.
- Remove annotation button: `aria-label="Remove annotation"` and keyboard-accessible (Enter/Space).
- Tab bar: `role="tablist"`, each tab `role="tab"`, `aria-selected="true/false"`, `aria-controls` pointing to the corresponding panel.
- Tab panels: `role="tabpanel"`, `id` matching tab `aria-controls`.
- Diff container: `aria-label="Code diff view"`.
- Annotation highlight marks: `<mark aria-label="Annotated text">` — mark element is semantically correct for highlighted text.
- Focus management: when annotation card is removed, focus moves to the "Select text" empty state or the Overall Comment field.
- Keyboard shortcut: Enter to approve is suppressed when focus is inside the annotation sidebar (textarea, buttons).
