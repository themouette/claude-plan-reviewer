**src/hook.rs** — Hook protocol serde types:

Define these exact structs (field names and serde attributes are protocol-critical):

```rust
use serde::{Deserialize, Serialize};

// --- Input (stdin) ---

#[derive(Deserialize, Debug)]
pub struct HookInput {
    pub session_id: String,
    pub transcript_path: Option<String>,
    pub cwd: String,
    pub hook_event_name: String,
    pub tool_name: String,
    pub tool_input: ToolInput,
}

#[derive(Deserialize, Debug)]
pub struct ToolInput {
    pub plan: Option<String>,
    #[serde(flatten)]
    pub extra: serde_json::Map<String, serde_json::Value>,
}

// --- Output (stdout) ---

#[derive(Serialize, Debug)]
pub struct HookOutput {
    #[serde(rename = "hookSpecificOutput")]
    pub hook_specific_output: HookSpecificOutput,
}

#[derive(Serialize, Debug)]
pub struct HookSpecificOutput {
    #[serde(rename = "hookEventName")]
    pub hook_event_name: String,
    pub decision: PermissionDecision,
}

#[derive(Serialize, Debug)]
pub struct PermissionDecision {
    pub behavior: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub message: Option<String>,
}
```

Add a constructor on `HookOutput` for convenience:

impl HookOutput {

**src/render.rs** — comrak markdown-to-HTML helper:

```rust
use comrak::{markdown_to_html, Options};

pub fn render_plan_html(markdown: &str) -> String {
    let mut options = Options::default();
    options.extension.table = true;
    options.extension.tasklist = true;
    options.extension.strikethrough = true;
    options.extension.autolink = true;
    // Do NOT set options.render.unsafe_ = true — plan content is sanitized
    markdown_to_html(markdown, &options)
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin -> binary | JSON from Claude Code process; trusted but could be malformed |
| binary -> stdout | Final JSON response; must be exactly one valid JSON object |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | stdin JSON | accept | Input comes from Claude Code on same machine; malformed JSON causes parse error + exit 1, not data corruption |
| T-01-02 | Information Disclosure | stderr output | accept | stderr diagnostics are visible only to the local user running the binary; no sensitive data is logged beyond plan byte count |
| T-01-03 | Denial of Service | stdin read | mitigate | `read_to_string` on stdin could block forever if stdin is never closed; this is the documented Claude Code hook protocol behavior (pipe closes after JSON). If stdin hangs, Plan 02's timeout timer (HOOK-05) will force exit. |
| T-01-04 | Spoofing | comrak HTML output | mitigate | comrak sanitizes HTML by default (unsafe_ is NOT enabled); XSS in rendered plan HTML is blocked at the rendering layer |
</threat_model>
