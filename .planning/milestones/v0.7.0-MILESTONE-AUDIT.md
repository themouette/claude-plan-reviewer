## Critical Gap: SUBMIT-03 — POST /api/decide Schema Mismatch

### Root Cause

The Phase 28 frontend (`buildCodeReviewPayload.ts`) produces a payload that was redesigned from the Phase 24 backend contract without updating the backend:

**Frontend sends** (Phase 28 `buildReviewPayload`):
```json
{ "message": "optional string", "comments": [...] }
```

**Backend expects** (Phase 24 `plan_review.rs` `Decision` struct):
```json
{ "behavior": "allow" | "deny", "message": "optional string" }
```

`behavior` is a non-Optional `String` with no `#[serde(default)]`. Axum's JSON extractor returns `422 Unprocessable Entity` on every code-review submission. The frontend's `res.ok` check fails, it falls to the clipboard path (user sees "Copied"), but the server process stays running and hits the 540-second timeout in `decision_rx.await`, then emits `{"behavior":"deny","message":"Review timed out — plan was not approved"}`.

### Impact

- **SUBMIT-03** (structured feedback → agent): agent never receives the code review outcome via stdout
- **INTEG-01** (slash command): agent running `plan-reviewer code-review` in background always gets `deny` after 9 minutes
- **INTEG-02** (pre-PR hook): `run_pre_pr_hook_flow` always outputs `HookOutput::deny("Review timed out")` as the Claude PreToolUse response after 9 minutes, blocking `gh pr create`

### Evidence

| Location | Detail |
|----------|--------|
| `src/plan_review.rs:14-18` | `struct Decision { behavior: String, message: Option<String> }` — non-optional `behavior` |
| `src/plan_review.rs:44` | `async fn post_decide(Json(d): Json<Decision>, ...)` — extractor fails on missing field |
| `ui/src/code-review/buildCodeReviewPayload.ts:34-50` | `buildReviewPayload` produces `{message?,comments?}` — no `behavior` field |
| `ui/src/code-review/AppToolbar.tsx:77-98` | `handleSend` POSTs to `/api/decide`, checks `res.ok || res.status === 409`, falls to clipboard on 422 |
| `tests/integration/server_cycle.rs:85-88` | All existing server tests POST `{"behavior":"allow"}` — old format — none test new code-review payload |
| `src/main.rs:855` | `run_pre_pr_hook_flow` → `run_code_review_flow` → `decision_rx.await` with 540s timeout → `deny` |

### Fix Options

**Option A (recommended):** Add `POST /api/code-review/decide` in `diff_api.rs` that accepts `{message?,comments?}`, derives a `behavior` (allow when no comments and no message signaling changes, deny otherwise — or always allow and let agent decide), and sends to a new `decision_tx` that `run_code_review_flow` awaits.

**Option B:** Replace `Decision` with a `CodeReviewDecision` enum in `plan_review.rs` that accepts the frontend format and derives `behavior` from content presence. Update `decision_tx` type.

**Option C:** Add `behavior: "allow"` to `buildReviewPayload` output and update agent-side parsing to look at `comments` array for decision context.
