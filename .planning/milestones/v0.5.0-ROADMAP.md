# Milestone v0.5.0: Offline Resilience

**Status:** ✅ SHIPPED 2026-05-07
**Phases:** 12–16
**Total Plans:** 8

## Overview

When Claude Code kills the background `plan-reviewer review` process, the browser UI now
detects the loss, keeps the user working offline, and exports annotations via clipboard.
The `annotate.md` slash command closes the loop by prompting Claude to accept a pasted
clipboard JSON as a fallback when no stdout result arrives.

## Milestone Summary

**Key Decisions:**

- `ConnectivityStatus` is a parallel type to `AppState` — offline is NOT an AppState variant; merging these concerns is the primary anti-pattern avoided
- `navigator.clipboard.writeText()` called synchronously in the click handler (no await before it) — preserves transient activation in Safari and Firefox
- 3 consecutive failures required before declaring offline — prevents false alarms from transient loopback hiccups
- Clipboard JSON format is `{"behavior":"allow"}` or `{"behavior":"deny","message":"..."}` — byte-for-byte identical to `build_opencode_output`; must not drift
- `get_ping` is parameter-less (no State extractor) to enforce statelessness at the type level
- SPA-fallback guard in route tests asserts non-text/html Content-Type — defends against axum FallbackBehavior::Ok masking missing routes with index.html

**Issues Resolved:**

- Offline annotation UX — users previously had no path when the server was killed
- Clipboard transient activation — synchronous write preserves browser security policy compliance
- Full loop closure — Phase 16 connects Phase 15's clipboard to the Claude conversation via annotate.md

**Issues Deferred:**

- SLC-02: Double-result disambiguation when both stdout and pasted JSON arrive (deferred post-v0.5.0)
- OFX-03: Textarea fallback when navigator.clipboard is blocked (deferred)
- OFX-04: Graceful online recovery — restore normal submit flow when server comes back (deferred)

**Technical Debt Incurred:**

- `install_returns_err_when_binary_path_is_none` test pre-existing failure — not caused by v0.5.0 but not fixed; install() never reads ctx.binary_path
- WR-02 (Phase 16 code review): No instruction for invalid clipboard paste in annotate.md Step 4
