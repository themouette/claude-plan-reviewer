**git2 reference iteration pattern** — copy from `find_base_commit` style (lines 68–88):
```rust
// Iterate all refs, peel each to a commit, compare SHA
let commit_sha = c.id();
let mut branches: Vec<String> = Vec::new();
let mut tags: Vec<String> = Vec::new();
if let Ok(refs) = repo.references() {
    for r in refs.flatten() {
        if let Ok(peeled) = r.peel_to_commit() {
            if peeled.id() == commit_sha {
                if let Some(name) = r.shorthand() {
                    if r.is_branch() {
                        branches.push(name.to_string());
                    } else if r.is_tag() {
                        tags.push(name.to_string());
                    }
                }
            }
        }
    }
}
```

**Error handling pattern** — same as rest of file: use `.ok()?` / `.unwrap_or_default()` / `.flatten()` idioms; never propagate panics. No new error codes needed — `branches`/`tags` degrade to empty vecs silently.

**Test pattern** — the existing `#[cfg(test)]` block uses `make_repo_with_main_and_feature` fixtures and `do_get` helper. New tests for branches/tags should assert the JSON field is present and is an array; use `repo.branch(...)` in the fixture to create a named branch pointing at the commit under test.

**Click handler pattern** — new `handleCommitClick(sha, event)`:
```typescript
function handleCommitClick(sha: string, event: React.MouseEvent) {
  const isMeta = event.metaKey || event.ctrlKey
  if (isMeta) {
    setSelectedCommitShas((prev) =>
      prev.includes(sha) ? prev.filter((s) => s !== sha) : [...prev, sha]
    )
  } else if (event.shiftKey && selectedCommitShas.length > 0) {
    // Range select: slice between last selected and this commit
    const lastSha = selectedCommitShas[selectedCommitShas.length - 1]
    const lastIdx = commits.findIndex((c) => c.sha === lastSha)
    const thisIdx = commits.findIndex((c) => c.sha === sha)
    const [start, end] = lastIdx < thisIdx ? [lastIdx, thisIdx] : [thisIdx, lastIdx]
    const range = commits.slice(start, end + 1).map((c) => c.sha)
    setSelectedCommitShas((prev) => Array.from(new Set([...prev, ...range])))
  } else {
    setSelectedCommitShas([sha])
  }
}
```

## Shared Patterns

### Source-assertion test pattern
**Source:** All `*.test.ts` files in `ui/src/code-review/`
**Apply to:** All modified component test files

Every test file uses this exact boilerplate:
```typescript
/// <reference types="node" />
import { describe, it, expect } from 'vitest'
import { readFileSync } from 'fs'
import { resolve } from 'path'
import ComponentName from './ComponentName'

const source = readFileSync(resolve(__dirname, './ComponentName.tsx'), 'utf-8')

describe('ComponentName', () => {
  it('exports a function as default', () => {
    expect(typeof ComponentName).toBe('function')
  })
  // All other assertions use source.toContain(...) or source.not.toContain(...)
})
```
This pattern tests structural/source attributes without a DOM renderer. When removing props/behaviors, add a negative assertion (`not.toContain`). When adding new behavior, add a `toContain` assertion for the new string literal.

### CSS variable token pattern
**Source:** `ui/src/code-review/CommitDrawer.tsx`, `DiffPane.tsx`, `AppToolbar.tsx`
**Apply to:** All component files

All inline styles use CSS variables — never raw color values:
- `var(--color-bg)` — page background
- `var(--color-surface)` — card/panel background
- `var(--color-border)` — borders, dividers
- `var(--color-text-primary)` — active/selected text
- `var(--color-text-secondary)` — subdued text
- `var(--color-focus)` — focus ring, active border accent
- `var(--color-accent-deny)` — deletions, errors (red)
- `var(--color-accent-approve)` — additions (green)

### No-import-from-reviewer-v2 rule
**Source:** Every component test file (`not.toContain('reviewer-v2/')`)
**Apply to:** All TypeScript files

Every test file asserts `expect(source).not.toContain('reviewer-v2/')`. Every modified source file must not add any import from `reviewer-v2/`.

### Injectable doFetch seam
**Source:** `ui/src/code-review/hooks/useDiff.ts` (lines 6, 34–53)
**Apply to:** Any new fetch logic added in this phase

All fetch logic follows the pure-function + injectable `doFetch` pattern:
```typescript
export type DoFetch = (url: string) => Promise<Response>

export async function fetchXxxOnce(doFetch: DoFetch, ...): Promise<FetchDiffResult> {
  try {
    const res = await doFetch(url)
    if (!res.ok) return { files: [], error: 'fetch failed' }
    const data = (await res.json()) as FileDiff[]
    return { files: data, error: null }
  } catch {
    return { files: [], error: 'network error' }
  }
}
```
No new fetch calls are added in this phase (all diff endpoints already exist), so this pattern is reference-only.

### Rust struct field addition pattern
**Source:** `src/diff_api.rs` (lines 14–28, `FileDiff` struct)
**Apply to:** `Commit` struct additions

Use `#[serde(skip_serializing_if = "Option::is_none")]` only for `Option` fields. Always-present fields (like `Vec<String>`) need no attribute — they serialize as `[]` when empty. Maintain the `pub` visibility and explicit type annotation style.

### Deferred setState via setTimeout(0)
**Source:** `ui/src/code-review/CodeReviewApp.tsx` (lines 61–64, 69–75)
**Apply to:** Any `useEffect` that needs to call `setState`

```typescript
useEffect(() => {
  const id = setTimeout(() => setXxx(value), 0)
  return () => clearTimeout(id)
}, [dep])
```
This avoids the `react-hooks/set-state-in-effect` lint rule. Never call `setState` synchronously inside a `useEffect` body.
