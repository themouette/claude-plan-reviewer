## Post-checkpoint Fix: Prevent Text Selection on Shift+Click

**Reported by:** User (checkpoint verification)
**Issue:** When shift+clicking to range-select commits in CommitDrawer, the browser's native text selection fired, causing commit row text to visually highlight blue.
**Fix:** Added `if (e.shiftKey) e.preventDefault()` at the start of the `li` `onClick` handler in `CommitDrawer.tsx`, before delegating to `onCommitClick`. The fix is applied in the inline wrapper so the DOM event is cancelled at the earliest point, before the parent callback fires.
**Commit:** 45cfa83 — `fix(26.2-02): prevent text selection on shift+click in CommitDrawer`
