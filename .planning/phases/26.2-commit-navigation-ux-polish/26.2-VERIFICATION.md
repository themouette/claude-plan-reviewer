#### 1. Drawer Layout Push (D-01)

**Test:** Open `/code-review`, click the Commits toggle button to open the drawer. Observe the diff pane.
**Expected:** The diff panel shifts right by 296px; the drawer is NOT layered on top of the diff content; resizing the window shows the proportional layout.
**Why human:** Visual CSS layout behavior (flex vs absolute positioning effect) cannot be verified from source code alone.

#### 2. Click / CMD+click / Shift+click Selection Model (D-02, D-03)

**Test:** In the commit drawer, click one row, then CMD+click a second, then Shift+click a third further down the list.
**Expected:** Single click highlights exactly one row and shows that commit's diff. CMD+click adds the second row to selection (two rows highlighted), diff updates to union. Shift+click range-selects all rows from the last selected to the clicked one.
**Why human:** Mouse event modifier behavior (metaKey, ctrlKey, shiftKey) is runtime-only; plus Enter-key currently passes a cast KeyboardEvent which silently works for plain Enter but mis-routes Shift+Enter/Meta+Enter (CR-02 in code review).

#### 3. All-Selected Branch Label (D-05)

**Test:** Use Shift+click or CMD+click to select all commits in the drawer.
**Expected:** The diff pane title strip changes to "diff from branch `<branchName>`" where `<branchName>` is the actual branch name (e.g. `main`, `gsd/v0.7.0-code-review`), not the literal `HEAD` fallback.
**Why human:** `branchName` is derived from `commits[0]?.branches?.[0] ?? 'HEAD'` — requires a live repo where the first commit returned by `/api/commits` has a branch ref populated by the Rust backend. The fallback to `'HEAD'` when no branch ref exists may or may not be acceptable UX.

#### 4. Stats Strip + Per-File Collapse (D-07, D-09)

**Test:** Open the code-review view on a branch with changed files. Check the top of the diff pane for the stats strip. Click a file header chevron.
**Expected:** Stats strip shows "`N` files changed", a green `+M` sum, a red `−K` sum. Clicking the file header collapses the body (chevron flips from ▼ to ▶). FileListPane jump-to link still scrolls to the collapsed file header. Clicking the header again re-expands.
**Why human:** Stats arithmetic correctness requires a real diff response; per-file collapse UX (chevron flip, anchor scroll behavior) requires visual confirmation.

#### 5. Expand Files / Collapse Files Toolbar Button (D-08)

**Test:** Collapse one or more files via chevron click, then click "Expand Files" in the toolbar. Then click "Collapse Files".
**Expected:** "Expand Files" expands all files at once and label changes to "Collapse Files". "Collapse Files" collapses all files and label reverts. The adjacent "Expand All" (context-lines) button is unchanged.
**Why human:** Toggle state interaction is runtime behavior; also WR-02 from code review notes that `allFilesExpanded` shows "Collapse Files" even when `files.length === 0` (minor label bug during loading).

#### 6. Branch and Tag Pills (D-11)

**Test:** Open a repo where the current branch has commits that are also branch tips or tagged. Open the commit drawer.
**Expected:** Each qualifying commit row shows `branch:<name>` and/or `tag:<name>` inline pills after the short SHA chip. Commits with no refs show no pills.
**Why human:** Requires a real repo with named refs; the test suite uses unit tests with mocked data only.
