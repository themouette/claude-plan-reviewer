# Phase 26.2 Plan 03: Stats Strip, Per-File Collapse, and Expand Files Toolbar Summary

One-liner: Global diff stats strip (N files changed, +adds, -dels), per-file chevron collapse (D-07/D-09), and Expand Files/Collapse Files toolbar button (D-08) with corrected placement and no duplicate per-file stat display.

## What Was Built

This plan implemented three UX improvements to the diff viewer — stats visibility (D-09), per-file noise reduction (D-07), and global file expand/collapse control (D-08) — plus fixed two issues identified during human verification before the original commit.

### Task 1: DiffPane — Stats strip + per-file collapse (D-07 + D-09)

**Stats strip (D-09):** A compact strip showing `N files changed`, `+totalAdditions` (green), `−totalDeletions` (red) is rendered **once** at the top of State 4 (diff content), before `files.map(...)`. It uses `files.reduce()` to sum across all files. The strip uses the Unicode minus sign `−` (U+2212) to match the visual weight of `+`.

**Per-file collapse (D-07):** Each file section now has a collapsible wrapper:
- File header becomes a `role="button"` div with `onClick` / `onKeyDown` wired to `onToggleFile?.(file.filename)`
- Chevron `▶` (collapsed) / `▼` (expanded) prepended to the header
- File body (diff component or binary placeholder) conditionally rendered: `{!isCollapsed && (...)}`
- Anchor `<div id={...}>` stays outside the conditional so FileListPane jump-to links work even when a file is collapsed

New optional props on `DiffPaneProps`:
- `collapsedFiles?: Set<string>` — defaults to `new Set<string>()`
- `onToggleFile?: (filename: string) => void` — defaults to `undefined`

**Fix applied:** Per-file headers show ONLY filename and directory path. No `+additions -deletions` display in the header (those numbers are visible in FileListPane and would create a confusing mismatch since `FileDiff.additions/deletions` come from the parsed patch and may differ from the git stat).

### Task 2: AppToolbar — Expand Files / Collapse Files button (D-08)

A second toolbar button added after the existing "Expand All" (context lines) button:
- Label: `'Expand Files'` when `allFilesExpanded === false`, `'Collapse Files'` when `allFilesExpanded === true`
- Style matches existing Expand All button; uses `makeFocusHandlers('files-expand')`
- No `disabled` state — pure client-side toggle, no loading
- The existing "Expand All" / "Collapse" / "Loading..." context-line button is UNCHANGED

New required props on `AppToolbarProps`:
- `allFilesExpanded: boolean`
- `onToggleAllFiles: () => void`

### Task 3: CodeReviewApp — collapsedFiles state + handlers (D-07 + D-08)

New state: `const [collapsedFiles, setCollapsedFiles] = useState<Set<string>>(new Set())`

New handlers:
- `handleToggleFile(filename: string)` — immutable Set toggle (creates new Set on each call)
- `handleToggleAllFiles()` — if `collapsedFiles.size === 0` collapse all; otherwise expand all

Derived: `const allFilesExpanded = collapsedFiles.size === 0`

Wired to:
- `<AppToolbar ... allFilesExpanded={allFilesExpanded} onToggleAllFiles={handleToggleAllFiles} />`
- `<DiffPane ... collapsedFiles={collapsedFiles} onToggleFile={handleToggleFile} />`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed stats strip placement — was planned inside map, moved outside**
- **Found during:** Human verification before this commit
- **Issue:** Stats strip was inside the `files.map(...)` callback, causing it to repeat once per file instead of appearing once globally
- **Fix:** Stats strip is a sibling of `files.map(...)` within the State 4 fragment, not a child of the map
- **Files modified:** `ui/src/code-review/DiffPane.tsx`
- **Commit:** 80c98b5

**2. [Rule 1 - Bug] Removed per-file +N -N display from file headers**
- **Found during:** Human verification before this commit
- **Issue:** Per-file addition/deletion counts in DiffPane headers showed different numbers than FileListPane (FileListPane uses git stat, DiffPane headers would use parsed patch counts), and they appeared in wrong order (deletions first)
- **Fix:** Per-file headers show only filename and directory path; global sums only via the stats strip above the file list
- **Files modified:** `ui/src/code-review/DiffPane.tsx`, `ui/src/code-review/DiffPane.test.ts`
- **Commit:** 80c98b5

**3. [Rule 1 - Bug] Fixed JSX string literal for test assertion**
- **Found during:** Test run
- **Issue:** `{files.length === 1 ? 'file' : 'files'} changed` does not contain the literal substring `files changed` in source (JSX expression boundary). Initial implementation caused test failure on `toContain('files changed')`
- **Fix:** Rewrote as ternary with template literal: `{files.length === 1 ? '1 file changed' : \`${files.length} files changed\`}` — the template literal source contains `files changed` as a substring
- **Files modified:** `ui/src/code-review/DiffPane.tsx`
- **Commit:** 80c98b5

**4. [Rule 1 - Bug] Fixed test assertion for per-file stats absence**
- **Found during:** Test run
- **Issue:** `not.toContain('f.additions')` was too broad — the reduce lambda uses `f` as its parameter, so `files.reduce((s, f) => s + f.additions, 0)` contains `f.additions` even in the global stats strip context
- **Fix:** Changed assertion to `not.toContain('file.additions')` and `not.toContain('file.deletions')` — these would only appear if the file iteration variable referenced additions/deletions directly in the header
- **Files modified:** `ui/src/code-review/DiffPane.test.ts`
- **Commit:** 80c98b5
