# Phase 26.2: Commit Navigation — UX Polish - Context

**Gathered:** 2026-05-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 26.2 delivers five UX improvements to the commit navigation feature shipped in Phase 26. All five were captured in the post-Phase-26 testing TODO. In scope: layout push (drawer overlays → pushes content), selection model redesign (checkbox → click/CMD+click/Shift+click), per-file expand/collapse in the diff viewer, diff stats strip in the diff pane header, and branch/tag ref pills in commit rows (requires Rust backend addition).

Not in scope: inline comments (Phase 27), review submission (Phase 28).

</domain>

<decisions>
## Implementation Decisions

### Drawer Layout (D-01)

- **D-01:** Remove `position: absolute; zIndex: 10` from `CommitDrawer`. Make it a flex sibling inside the `display: flex` container in `CodeReviewApp.tsx` (line 102) so it pushes the diff panel instead of overlaying it. Width stays within the 280–320px range from Phase 26 D-04.

### Selection Model (D-02–D-06)

- **D-02:** Replace `checkedCommitShas: string[]` + `activeCommitSha: string | null` + `viewMode: 'branch' | 'commit'` with a single `selectedCommitShas: string[]` (ordered, deduped). All three state variables collapse into one.
- **D-03:** Interaction rules:
  - Single click → `selectedCommitShas = [sha]` (replaces any prior selection)
  - CMD+click (macOS `metaKey`, Windows/Linux `ctrlKey`) → toggle `sha` in `selectedCommitShas`
  - Shift+click → range-select from last-clicked commit to this one (order = commit list order)
- **D-04:** View mode derived from selection size:
  - 0 selected → full branch diff (existing `/api/diff/branch` path)
  - 1 selected → per-commit view (`/api/diff/commit/{sha}`)
  - 2+ selected → union diff via `fetchFilteredBranchDiff(selectedCommitShas, …)` — reuses existing logic
- **D-05:** All commits selected → main panel header labels the view as "diff from branch {branchName}" (branchName already available from the existing `/api/commits` response or derived from `HEAD`).
- **D-06:** No checkboxes remain in `CommitDrawer`. Row highlighting (background or left-border accent) indicates selected state. The `onCheckChange` prop and `checkedCommitShas` prop are removed.

### Per-File Expand/Collapse (D-07–D-08)

- **D-07:** Each file section in the diff viewer gets a collapse/expand toggle (chevron in the file header). Default state: all expanded. The existing `@pierre/diffs` `PatchDiff` component does not provide this natively — wrap each file diff in a collapsible container in `DiffPane`.
- **D-08:** The AppToolbar "expand all" button (existing element) controls global collapse/expand state for all file sections. A new boolean `allExpanded: boolean` (or `collapsedFiles: Set<string>`) lives in `CodeReviewApp`. Claude's discretion on exact state shape.

### Diff Stats Strip (D-09)

- **D-09:** A compact stats strip renders in the diff pane, between the commit title strip and the file list / diff content. It displays: `{N} files changed  +{additions}  −{deletions}` summed across all visible `FileDiff` entries. Uses `FileDiff.additions`, `FileDiff.deletions`, and array length — all already in client state. Uses `var(--color-text-secondary)` for the text, `var(--color-surface)` background, `var(--color-border)` bottom border. Claude's discretion on exact layout.

### Branch/Tag Pills (D-10–D-11)

- **D-10:** Add `branches: Vec<String>` and `tags: Vec<String>` to the `Commit` struct in `src/diff_api.rs`. Populate them by iterating `repo.references()` and matching each ref's peel-to-commit SHA against the commit SHA. This is the only backend change in this phase.
- **D-11:** In `CommitDrawer`, render branch/tag values as small inline pills after the short SHA chip. Format: `branch:main`, `tag:v0.6.0`. Styling: `var(--color-surface)` background with `var(--color-border)` border, `var(--color-text-secondary)` text, small font (11–12px). Only non-empty arrays produce pills — no empty state needed.

### Claude's Discretion

- Exact state shape for collapse tracking (`allExpanded: boolean` + individual overrides vs `collapsedFiles: Set<string>`)
- Shift+click range-selection algorithm (slice between last-selected index and current index)
- Exact stats strip layout (inline, spaced, or with separator glyphs)
- Performance of ref iteration in Rust (cache or compute per request — 500 commit limit already exists)
