<objective>
Deliver the selection-model redesign (D-02–D-06), drawer layout push (D-01), branch/tag pills (D-11), and the all-selected branch label (D-05) in one consolidated UI plan. These changes are inseparable because they share three files (CodeReviewApp.tsx, CommitDrawer.tsx, DiffPane.tsx) and depend on a single coherent state model. Splitting them would force interim states where some props/state exist but their consumers don't.

Purpose: Replace the awkward checkbox + viewMode + activeCommitSha tri-state with a single ordered `selectedCommitShas: string[]` driven by direct mouse interaction (click / CMD+click / Shift+click) — the standard macOS Finder selection idiom users already know. Drawer becomes a real layout element instead of an overlay so the diff panel stays readable. Branch/tag pills make it obvious which commits are tagged or branch tips.

Output: Updated CodeReviewApp.tsx, CommitDrawer.tsx, DiffPane.tsx, plus updated test files asserting the new structure. End-of-plan human verification checkpoint validates that the drawer pushes content, click selection works, multi-select works, all-selected shows the branch label, and pills render.
</objective>
