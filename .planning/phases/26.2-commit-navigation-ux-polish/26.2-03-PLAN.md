<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filename string→Set<string> | FileDiff.filename comes from server JSON (already trust-validated in Phase 25 PatchDiff path); used as Set key only — no eval, no innerHTML |
| reducer over files[] | Numeric sums computed locally; no network call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26.2-08 | Tampering | Filename collision in Set<string> across refetches | accept | If two refetches return overlapping filenames, the Set behavior is correct (set membership). If a file disappears in a refetch, its stale Set entry is harmless (no UI rendered for absent filename). No security impact. |
| T-26.2-09 | Information Disclosure | Stats strip leaking file count for restricted files | accept | All files in the diff are user-owned (local repo). No multi-tenant model. files.length and reduce sums are derived from the same data already rendered file-by-file. |
| T-26.2-10 | Denial of Service | Reducer over 10,000-file diff | accept | files.reduce is O(N) and runs on every render but N is bounded by the user's diff size; React memoization not needed for a single-user local tool. If profiling later shows a bottleneck, wrap in useMemo. |
| T-26.2-11 | Cross-Site Scripting | filename string in chevron header | mitigate | React text interpolation `{file.filename.split('/').pop() ?? file.filename}` escapes HTML automatically. No dangerouslySetInnerHTML. |
| T-26.2-SC | Tampering | npm installs | mitigate | No new dependencies. |
</threat_model>
