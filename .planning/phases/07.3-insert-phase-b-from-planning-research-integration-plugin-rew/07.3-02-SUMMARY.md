## What Was Built

### perform_claude_migration (src/update.rs)

Migrates a pre-plugin Claude install to the plugin model:
1. Calls `write_claude_plugin_files` to create plugin directory with `hooks.json` using `"plan-reviewer review-hook"`
2. Patches `~/.claude/settings.json`: adds `extraKnownMarketplaces["plan-reviewer-local"]` and `enabledPlugins["plan-reviewer@plan-reviewer-local"]`
3. Removes old bare `ExitPlanMode` hook entry from settings.json via `arr.retain`
4. Uses idempotent `entry()` pattern — safe to call multiple times

### perform_gemini_migration (src/update.rs)

Migrates a pre-extension Gemini install to the extension model:
1. Calls `write_gemini_extension_files` to create extension directory with `hooks.json` using `"plan-reviewer review-hook"`
2. Patches `~/.gemini/settings.json`: removes old `BeforeTool` entries whose `hooks[]` array contains `name == "plan-reviewer"` via `arr.retain`

### Case 2 Detection in refresh_integrations_with_home (src/update.rs)

Two detection blocks at the START of the function, before Case 3 (version-stale):
- **Claude**: if `plugin.json` absent AND settings.json exists AND `claude_legacy_hook_installed` → run `perform_claude_migration`
- **Gemini**: if `gemini-extension.json` absent AND settings.json exists AND `gemini_legacy_hook_installed` → run `perform_gemini_migration`

Skip conditions enforced:
- No legacy entry in settings.json → no action (fresh system)
- Manifest already exists → Case 3 handles it (no double-migration)

### Integration module changes (claude.rs, gemini.rs)

Removed `#[allow(dead_code)]` from `claude_legacy_hook_installed` and `gemini_legacy_hook_installed` — both are now actively imported and called from `update.rs`.
