  **Section 4 — Usage**

  Subsection: **Approve, deny, annotate**

  Describe the flow: hook fires → browser tab opens at `http://127.0.0.1:{port}` → user reads plan → optionally writes a comment in the annotation field → clicks Approve or Deny → decision returned to agent.

  Note the 540-second timeout: if no decision, the reviewer defaults to deny after 9 minutes.

  Global flags:
  | Flag | Default | Purpose |
  |------|---------|---------|
  | `--no-browser` | — | Skip auto-open; URL is printed to stderr |
  | `--port N` | OS-assigned | Bind the review server to a specific port |

  Subsection: **Review a file without hook JSON**

  ```sh
  plan-reviewer review path/to/plan.md
  ```

  Reads the file directly (does not read stdin). Opens browser review UI. Outputs `{"behavior":"allow"}` or `{"behavior":"deny","message":"..."}` to stdout. Exit code 1 if file not found. Useful for scripts and agent workflows that don't construct hook JSON.

  Use a table (Claude's discretion — table is cleaner for a reference section):

  | Subcommand | Synopsis | Description |
  |------------|----------|-------------|
  | `review-hook` | `plan-reviewer review-hook` | Read hook JSON from stdin, open browser review UI. Explicit form of the default behavior. |

  **update flags** (from src/update.rs):

  | Flag | Purpose |
  |------|---------|
  | `--check` | Print current and latest version + changelog URL, no download |
  | `--version X` | Pin to a specific version (e.g. `--version v0.2.0`) |
  | `-y` / `--yes` | Skip confirmation prompt |

  Examples:
  ```sh
  plan-reviewer update --check        # check without downloading
  plan-reviewer update                # update to latest (with prompt)
  plan-reviewer update -y             # update to latest, no prompt
  ```
