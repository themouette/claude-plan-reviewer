### CR-01, WR-02, WR-03: Eliminate TOCTOU race, fix TempDir leak, add stdout guards

**Files modified:** `tests/integration/server_cycle.rs`
**Commit:** `3d0cbd1`
**Applied fix:**

CR-01 and WR-02 both required restructuring the same helper functions
(`spawn_hook_flow` and `spawn_code_review_flow`), so all three findings that
touched the test file were applied atomically in a single commit to avoid a
broken intermediate state.

**CR-01 (TOCTOU race):** Removed `find_free_port()` entirely. Both spawn
helpers now pass `--port 0` to the binary and read the OS-assigned port from
the binary's stderr. A new `read_port_from_stderr<R: Read>` helper scans lines
until it finds `"Review UI: http://127.0.0.1:<port>"` and parses the port. By
the time that line is printed, the binary has already bound to the port — no
race window. The `wait_for_port` polling loop was removed because it is no
longer needed (the server is already up when `spawn_hook_flow` returns).

**WR-02 (TempDir leak):** Both spawn functions now return
`(Child, tempfile::TempDir, u16)` instead of leaking the `TempDir` with
`Box::leak`. All test callers destructure the tuple and bind the `TempDir` to
`_home` so it is dropped automatically when the test frame exits (after
`wait_with_output()` completes).

**WR-03 (missing stdout guard):** Added
`assert!(!output.stdout.is_empty(), "stdout was empty — binary may have exited before writing JSON")`
before every `serde_json::from_slice` call in all five `server_cycle_*` tests.

All 30 integration tests pass after this change.

### WR-01: Reject missing `behavior` in plan-review mode with 422

**Files modified:** `src/plan_review.rs`, `src/server.rs`, `src/main.rs`
**Commit:** `19f7bc1`
**Applied fix:** requires human verification

Added a `ReviewMode` enum (`PlanReview | CodeReview`) to `src/plan_review.rs`
and a matching `mode: ReviewMode` field to `AppState`. `post_decide` now
matches on `state.mode` when `behavior` is absent:

- `ReviewMode::CodeReview` — defaults to `"allow"` (the code-review frontend
  intentionally sends `{ message, comments }` with no `behavior` key).
- `ReviewMode::PlanReview` — logs a warning to stderr identifying the missing
  field and returns `StatusCode::UNPROCESSABLE_ENTITY` (422), preventing
  accidental approval from typos or misrouted payloads.

`ReviewMode` is re-exported from `server.rs` alongside `Decision` and
`AppState`. `server::start_server` gained a `mode: ReviewMode` parameter that
is stored in `AppState`. `async_main` in `main.rs` also gained this parameter,
and each of its five call sites now explicitly specifies the mode:
- `run_opencode_flow`, `run_review_flow`, `run_hook_flow` → `PlanReview`
- `run_code_review_flow`, `run_pre_pr_hook_flow` → `CodeReview`

The change is semantically correct (the integration tests confirm the
code-review path still returns 200), but the plan-review 422 path has no
dedicated test — flag for human verification that the new branch behaves as
intended.
