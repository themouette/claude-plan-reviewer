|-----------|----------|-----------|-------------|-----------------|
| T-20-08 | Spoofing | annotation `id` field | accept | `crypto.randomUUID()` produces RFC 4122 v4 UUIDs; collision probability is negligible for the bounded annotation counts this app produces (tens per review session) |
| T-20-09 | Tampering | `CSS.highlights.set('comment-hover', ...)` | accept | The Range passed to `Highlight` is constructed from integer offsets in component state, not user-typed strings. The CSS Highlights API renders style only — it cannot mutate DOM or execute scripts |
| T-20-10 | Information Disclosure | annotation state lifted to Shell | accept | All annotations live in client-side React memory and remain on the client until the (future) Phase 22 submit. No new disclosure surface |
| T-20-11 | Denial of Service | useEffect running on every hoveredCommentId change | mitigate | Hover events fire at human-input rate (low frequency); the effect performs O(n) lookup and one `rangeFromOffsets` call (also O(n) over text nodes). For an n=20 annotation document this completes in <1ms. Pitfall 1 (scroll-adjusted coords) is enforced by the formula in CommentPane (Plan 02) |
| T-20-12 | Elevation of Privilege | replacing the right-aside placeholder with `<CommentPane>` | accept | The replacement is a build-time JSX change; no privileged API access added. The `aria-label="Comments"` attribute exposes information already visible in the surrounding `<aside>` |
| T-20-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; no changes to package.json or lockfiles. The full v0.6.0 milestone goal of zero new runtime dependencies for Phase 20 holds |
</threat_model>
