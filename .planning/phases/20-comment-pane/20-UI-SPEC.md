## Interaction Contract

### Bidirectional Hover (COMMENT-02)

**Bubble → anchor highlight:**
1. `onMouseEnter` on CommentBubble calls `onHover(id)`
2. Shell passes `hoveredCommentId` to ContentPane
3. ContentPane runs `CSS.highlights.set('comment-hover', new Highlight(range))` using `rangeFromOffsets(planRef.current, annotation.anchorStart, annotation.anchorEnd)`
4. `::highlight(comment-hover)` rule must be declared in `index.css`: `background-color: rgba(255,255,255,0.14)` (same value as existing `::highlight(annotation-hover)`)
5. `onMouseLeave` on CommentBubble calls `onHover(null)` → `CSS.highlights.delete('comment-hover')`

**Anchor hover → bubble highlight:**
- When ContentPane detects paragraph hover or text selection on a range that overlaps an annotation's `[anchorStart, anchorEnd]` interval, it calls `onHover(matchingAnnotationId)` — bubbles rendered with `hoveredCommentId` matching their id apply hovered visual state.
- Note: Paragraph-hover → CommentPane highlight is DEFERRED (per CONTEXT.md deferred section). Only text-selection overlap is in scope for Phase 20.

**Focus on click:**
1. Clicking anywhere on a CommentBubble calls `onFocus(id)`
2. Shell sets `focusedCommentId`
3. CommentPane uses `focusedCommentId` to determine which bubble is expanded vs compact
4. `z-index: 2` on the focused bubble, `z-index: 1` on all others (ensures focused is on top)

### Anchor Y Positioning (COMMENT-01)

- On mount: CommentPane walks `annotations`, calls `rangeFromOffsets(planRef.current, a.anchorStart, a.anchorEnd)` for each, gets `.getBoundingClientRect().top` adjusted for the planRef scroll container scroll offset. Stores `Map<id, anchorY>`.
- On `mainRef.current` scroll: recompute all anchor Y positions.
- On `ResizeObserver` of `planRef.current`: recompute all anchor Y positions.
- `anchorY` is relative to the inner wrapper top (absolute positioning context), not the viewport.
- Formula: `anchorY = range.getBoundingClientRect().top - planRef.current.getBoundingClientRect().top + mainRef.current.scrollTop`

### Overlap / Collapse Layout (COMMENT-03)

Executor must evaluate `sidenotes@2.0.1` first (per D-05 in CONTEXT.md). If adopted, follow its API. If not adopted, implement `useCommentLayout` as follows:

**`useCommentLayout` function signature (if custom):**
```ts
export function computeCommentLayout(
  items: Array<{ id: string; anchorY: number; isExpanded: boolean; height: number }>,
): Array<{ id: string; top: number; isCompact: boolean }>
```

**Algorithm (greedy, top-down):**
1. Sort items by `anchorY` ascending.
2. For each item, set `top = max(anchorY, previousBottom + 8)` where `previousBottom = previousTop + previousHeight`.
3. If `top > anchorY + 40` (pushed more than 40px below anchor), mark `isCompact: true` for this item.
4. Expanded item (focusedCommentId) always gets `top = anchorY` (snaps to anchor) and is sorted to be processed last — other items defer to it.
5. Output is used to set `position: absolute; top: {top}px` on each bubble.

**Heights:**
- Compact bubble: 48px fixed (use as `height` for layout calculation)
- Expanded bubble: measured via `getBoundingClientRect()` after render (use 0 on first pass, recompute after layout effect)

## Accessibility Contract

- CommentPane outer element: `<aside aria-label="Comments">` (replaces the current anonymous `<aside>` in Shell)
- Each bubble: `<article>` with `aria-label` per copywriting contract above
- Focused bubble: `aria-expanded="true"`, compact: `aria-expanded="false"`
- Hover link to anchor: purely visual — no ARIA description required (decorative enhancement)
- Focus ring: `outline: 2px solid var(--color-focus); outline-offset: 2px` on `:focus-visible` — same pattern as OutlinePane `.outline-button:focus-visible`
- Keyboard navigation: Tab moves between bubbles; Enter/Space expands/collapses (i.e. triggers `onFocus(id)`)
