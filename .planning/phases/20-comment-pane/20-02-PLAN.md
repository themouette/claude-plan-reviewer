<objective>
Build the two presentational pieces that turn `Annotation[]` into a scrolling sidebar of anchor-aligned comment bubbles. `CommentBubble.tsx` renders a single comment as an `<article>` with absolute positioning, a 3px left border colored by annotation type, and two visual states (compact 48px-tall 2-line clamp vs expanded auto-height). `CommentPane.tsx` is the container: it subscribes to `mainRef` scroll and a `ResizeObserver` on `planRef`, recomputes the `Map<id, anchorY>` on each event, runs `computeCommentLayout` to produce final tops + compact flags, and renders one `CommentBubble` per annotation inside a `position: relative; minHeight: 100%` wrapper.

Purpose: COMMENT-01 (anchor-Y positioning + scroll-follow), COMMENT-02 (one half of bidirectional hover — the bubble side signals via `onHover`/`onMouseEnter`), and COMMENT-03 (compact/expanded layout via `computeCommentLayout`) are entirely owned by these two components. Plan 03 only has to wire them to Shell state.

Output: 2 new components + 2 new source-inspection test files. The components consume the contracts defined in Plan 01 (`Annotation` type with `anchorStart`/`anchorEnd`, `computeCommentLayout` pure function) without modifying any existing file.
</objective>
