import { defineConfig } from 'vite'

**Step 6: Create build.rs at project root:**

```rust
fn main() {
    // Gate: skip frontend build when SKIP_FRONTEND_BUILD is set
    // (useful for CI cross-compilation where npm is not available)
    if std::env::var("SKIP_FRONTEND_BUILD").is_ok() {
        return;
    }

    let ui_dir = concat!(env!("CARGO_MANIFEST_DIR"), "/ui");

    // Run npm install (ensure deps are present)
    let install_status = std::process::Command::new("npm")
        .args(["install"])
        .current_dir(ui_dir)
        .status()
        .expect("failed to run npm install — is Node.js installed?");
    if !install_status.success() {
        panic!("npm install failed in ui/");
    }

    // Run npm run build
    let build_status = std::process::Command::new("npm")
        .args(["run", "build"])
        .current_dir(ui_dir)
        .status()
        .expect("failed to run npm run build — is Node.js installed?");
    if !build_status.success() {
        panic!("npm run build failed in ui/");
    }

    // Tell cargo to re-run this script if frontend sources change
    println!("cargo:rerun-if-changed=ui/src");
    println!("cargo:rerun-if-changed=ui/index.html");
    println!("cargo:rerun-if-changed=ui/package.json");
    println!("cargo:rerun-if-changed=ui/vite.config.ts");
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| comrak HTML -> React dangerouslySetInnerHTML | Server-rendered HTML injected into DOM |
| User input -> deny message | Textarea content sent to /api/decide |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | dangerouslySetInnerHTML | mitigate | comrak sanitizes HTML by default (unsafe_ is OFF in src/render.rs). XSS vectors in plan markdown are stripped at the server before reaching the browser. Do NOT set unsafe_=true. |
| T-03-02 | Information Disclosure | plan content | accept | Plan content is already visible to the user in Claude Code; showing it in the browser does not expose new information. Localhost-only server prevents network exfiltration. |
| T-03-03 | Tampering | deny message input | accept | The deny message is sent to Claude Code via stdout JSON. It is a free-text field by design. No sanitization needed — Claude interprets it as natural language feedback. |
</threat_model>
