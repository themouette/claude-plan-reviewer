# Phase 1 — UI Design Contract

> Visual and interaction contract for the Hook & Review UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled with Tailwind CSS utilities |
| Icon library | none — text labels only in Phase 1 |
| Font | System sans-serif stack: `ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` for chrome; `ui-monospace, "Cascadia Code", "Fira Code", monospace` for rendered plan code blocks |

**Rationale (Claude's Discretion):** This is a single-page developer tool with two interactive controls (Approve, Deny). A component library like shadcn adds build complexity for minimal payoff. Tailwind CSS via the Vite pipeline produces a small, self-contained bundle compatible with rust-embed. No icon library is needed — actions are labeled with text.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline tag padding |
| sm | 8px | Button internal padding (vertical), input padding |
| md | 16px | Default element spacing, button internal padding (horizontal), card padding |
| lg | 24px | Section padding, gap between plan body and action bar |
| xl | 32px | Layout horizontal padding (page edges) |
| 2xl | 48px | Major section breaks (plan heading to plan body) |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions: Approve button minimum touch target 44px height (keyboard accessibility). Deny textarea minimum height 80px.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.3 |
| Display | 28px | 600 (semibold) | 1.2 |

**Notes:**
- Body (16px/400/1.6) is used for rendered plan prose — the comrak HTML output. The higher line height (1.6 vs standard 1.5) aids readability of dense plan text.
- Label (14px/400/1.4) is used for button text, textarea label, status messages, and confirmation page copy.
- Heading (20px/600/1.3) is used for plan section headings (h2 level in comrak output) and the page-level "Plan Review" title.
- Display (28px/600/1.2) is used only for the confirmation page decision state ("Plan approved" / "Plan denied").
- Plan code blocks rendered by comrak use the monospace font stack at 14px/400/1.5 with a distinct background.

## Color

**Theme: Dark.** Developer tool — dark background reduces eye strain during extended review sessions and matches the terminal/IDE context.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` | Page background, main surface |
| Secondary (30%) | `#1a1d27` | Plan content card, action bar background |
| Accent-approve (8%) | `#22c55e` (green-500) | Approve button background, approve confirmation state |
| Accent-deny (2%) | `#ef4444` (red-500) | Deny submit button background when active, deny confirmation state |
| Destructive | `#ef4444` (red-500) | Deny submit button only |
| Text primary | `#f1f5f9` (slate-100) | Body text, headings, button labels |
| Text secondary | `#94a3b8` (slate-400) | Label text, helper copy, placeholder text |
| Border | `#2d3148` | Card borders, textarea border, separator lines |
| Code surface | `#12151f` | Inline code and fenced code block background in rendered plan |
| Focus ring | `#3b82f6` (blue-500) | Keyboard focus indicator on all interactive elements |

**Accent reserved for:**
- Approve button background (accent-approve)
- Approve button hover state (accent-approve darkened to `#16a34a`)
- Deny submit button background when textarea has valid input (accent-deny / destructive)
- Confirmation page icon/heading color matching the decision made

Never use accent colors for decorative purposes, plan content styling, or hover states of secondary elements.

## Interaction Contract

### Approve Flow (UI-03)

- **Trigger:** Press `Enter` key anywhere on the page, OR click the "Approve" button.
- **Enter key scope:** Global `window` `keydown` listener. The Enter key fires `approve()` ONLY when `document.activeElement?.tagName !== 'TEXTAREA'`. This prevents the shortcut from firing while the user is typing a deny message.
- **On activate:** POST `{ behavior: 'allow' }` to `/api/decide`. Transition to confirmation page state.
- **Button state:** Always enabled unless the decision has already been submitted (idempotency guard).

### Deny Flow (UI-04, D-06)

- **Step 1:** Click "Deny" button — expands deny form inline in action bar. The Approve button remains visible.
- **Step 2:** User types in textarea. "Submit Denial" button is disabled (`pointer-events: none`, reduced opacity 40%) until textarea contains at least one non-whitespace character.
- **Step 3:** Click "Submit Denial" or press `Enter` while textarea is focused — POST `{ behavior: 'deny', message: <textarea value> }` to `/api/decide`. Transition to confirmation page state.
- **Cancel:** Clicking "Deny" again (or pressing Escape) collapses the form without submitting.

### Confirmation Page State (UI-05)

- **After approve:** Page transitions to confirmation state. Shows "Plan approved" in accent-approve color (28px/600). Below: "You can close this tab." in text-secondary (14px/400).
- **After deny:** Page transitions to confirmation state. Shows "Plan denied" in accent-deny color (28px/600). Below: "Your feedback has been sent. You can close this tab." in text-secondary (14px/400).
- **Auto-close attempt:** After 500ms delay, call `window.close()`. This will be blocked by the browser (tab was not opened via `window.open()`). This is expected per RESEARCH.md Pitfall 4. Do not show an error if it fails.
- **No meta-refresh redirect.** The "You can close this tab" message is sufficient. Keep the page stable so the user can read the confirmation.

### Loading State

- On initial page load, while `GET /api/plan` is in-flight: show a centered spinner (CSS-only rotating border animation, 32px diameter, border-color text-secondary, border-top-color accent-approve).
- If fetch fails: show error state (see Copywriting Contract below).

### Focus Management

- On page load: Approve button receives `autofocus`. This ensures Enter key immediately works without requiring a click first.
- After deny form expands: focus moves to the textarea automatically.
- After deny form collapses: focus returns to the Deny button.

## Component Inventory

| Component | Type | Implementation |
|-----------|------|----------------|
| `PlanContent` | React component | Renders `dangerouslySetInnerHTML={{ __html: planHtml }}`. Plan HTML is sanitized by comrak server-side — unsafe render is OFF. Apply Tailwind prose styles via a custom CSS class. |
| `ApproveButton` | React component | `<button autoFocus>` with global Enter key handler. Green background, white text. |
| `DenySection` | React component | Toggle-able deny form. Textarea + Submit Denial button. |
| `ConfirmationView` | React component | Replaces plan + action bar after decision submitted. |
| `LoadingSpinner` | React component | CSS-only spinner shown during `/api/plan` fetch. |
| `PageHeader` | React component | Fixed top bar with "Plan Review" title. |

**No third-party React component libraries.** All components are hand-rolled with Tailwind utility classes.

## Tailwind Configuration Notes

Since no `tailwind.config.*` exists yet, the executor must initialize Tailwind as part of the Vite frontend setup:

```bash
cd ui && npm install -D tailwindcss @tailwindcss/vite
```

Add to `vite.config.ts`:
```typescript
import tailwindcss from '@tailwindcss/vite'
// ...
plugins: [react(), tailwindcss()],
```

No custom Tailwind theme extension is needed — use default Tailwind utilities with the color values declared in this spec applied via inline styles or CSS custom properties where Tailwind's palette does not include the exact values (e.g., `#0f1117`, `#1a1d27`, `#2d3148`, `#12151f`).

Define the following CSS custom properties at `:root` in `src/index.css`:

```css
:root {
  --color-bg: #0f1117;
  --color-surface: #1a1d27;
  --color-border: #2d3148;
  --color-code-bg: #12151f;
  --color-text-primary: #f1f5f9;
  --color-text-secondary: #94a3b8;
  --color-accent-approve: #22c55e;
  --color-accent-approve-hover: #16a34a;
  --color-accent-deny: #ef4444;
  --color-focus: #3b82f6;
}

body {
  background-color: var(--color-bg);
  color: var(--color-text-primary);
  font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
  font-size: 16px;
  line-height: 1.6;
}
```
