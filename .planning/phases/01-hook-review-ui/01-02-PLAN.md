async fn post_decide(

    Json(body): Json<Decision>,
) -> impl IntoResponse {
    let tx = state.decision_tx.lock().unwrap().take();
    match tx {
        Some(tx) => {
            let _ = tx.send(body);
            StatusCode::OK

This function:
a. Creates `oneshot::channel::<Decision>()` for the decision
b. Creates `CancellationToken` for graceful shutdown
c. Builds `AppState` with `plan_html` and `decision_tx`
d. Builds axum Router:
   - `GET /api/plan` -> `get_plan`
   - `POST /api/decide` -> `post_decide`
   - `GET /` -> serves the placeholder HTML page
   - `.with_state(Arc::new(state))`
e. Binds `TcpListener` to `127.0.0.1:0` (OS-assigned port)
f. Extracts the assigned port via `listener.local_addr()?.port()`
g. Returns a `ServerHandle` struct containing the port and a way to await the decision.

Actually, restructure for cleaner separation: `run_server` should:
1. Bind the listener, get the port
2. Spawn the axum server task with graceful shutdown
3. Return `(port, decision_rx)` so main.rs can handle browser opening and timeout

fn main() {
    let args = Args::parse();

    // 1. Sync stdin read (before runtime)
    let input_json = std::io::read_to_string(std::io::stdin())
        .expect("failed to read stdin");
    let hook_input: HookInput = serde_json::from_str(&input_json)
        .unwrap_or_else(|e| {
            eprintln!("Failed to parse hook input: {}", e);
            std::process::exit(1);
        });

    // 2. Render plan markdown to HTML
    let plan_md = hook_input.tool_input.plan.unwrap_or_default();
    let plan_html = render::render_plan_html(&plan_md);
    eprintln!("Plan rendered ({} bytes HTML)", plan_html.len());

    // 3. Start tokio runtime (current_thread for single-user tool)
    let rt = tokio::runtime::Builder::new_current_thread()
        .enable_all()
        .build()
        .unwrap();

    let output = rt.block_on(async_main(args, plan_html));

    // 4. Write decision to stdout — THE ONLY stdout write
    serde_json::to_writer(std::io::stdout(), &output)
        .expect("failed to write hook output");
}

3. Implement `async_main`:

```rust
async fn async_main(args: Args, plan_html: String) -> HookOutput {
    // Start server
    let (port, decision_rx) = match server::start_server(plan_html).await {
        Ok(v) => v,
        Err(e) => {
            eprintln!("Failed to start server: {}", e);
            return HookOutput::deny(format!("Internal error: {}", e));
        }
    };

    let url = format!("http://127.0.0.1:{}", port);

    // Always print URL to stderr (UI-06)
    eprintln!("Review UI: {}", url);

    // Open browser unless --no-browser (CONF-02)
    if !args.no_browser {
        if let Err(e) = webbrowser::open(&url) {
            eprintln!("Failed to open browser: {}", e);
            eprintln!("Open manually: {}", url);
        }
    }

    // Race: user decision vs timeout (540 seconds, per D-07)
    const TIMEOUT_SECS: u64 = 540;

    let decision = tokio::select! {
        result = decision_rx => {
            match result {
                Ok(d) => d,
                Err(_) => {
                    eprintln!("Decision channel closed unexpectedly");
                    server::Decision {
                        behavior: "deny".to_string(),
                        message: Some("Internal error: decision channel closed".to_string()),
                    }
                }
            }
        }
        _ = tokio::time::sleep(std::time::Duration::from_secs(TIMEOUT_SECS)) => {
            eprintln!("Review timed out after {} seconds", TIMEOUT_SECS);
            server::Decision {
                behavior: "deny".to_string(),
                message: Some("Review timed out \u{2014} plan was not approved".to_string()),
            }
        }
    };

    // Spawn 3-second watchdog for clean exit (HOOK-04)
    tokio::spawn(async {
        tokio::time::sleep(std::time::Duration::from_secs(3)).await;
        std::process::exit(0);
    });

    // Convert Decision to HookOutput
    match decision.behavior.as_str() {
        "allow" => HookOutput::allow(),
        "deny" => HookOutput::deny(
            decision.message.unwrap_or_else(|| "Denied without message".to_string())
        ),
        other => {
            eprintln!("Unknown decision behavior: {}", other);
            HookOutput::deny(format!("Unknown behavior: {}", other))
        }
    }
}
```

CRITICAL constraints:
- The timeout message MUST be exactly: `"Review timed out \u{2014} plan was not approved"` (em dash, per D-07)
- No `println!` anywhere — all diagnostics use `eprintln!`
- The 3-second watchdog ensures HOOK-04 compliance (exit within 3s of decision)
- URL is ALWAYS printed to stderr (UI-06), even when browser opens successfully

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/decide | Decision POST from browser to local server; same-origin, localhost only |
| browser -> /api/plan | Plan fetch from browser; read-only, localhost only |
| server bind | Server binds to 127.0.0.1 only; not externally reachable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | POST /api/decide | mitigate | Mutex<Option<Sender>> ensures only the first POST is accepted; second returns 409. Localhost binding prevents remote access. |
| T-02-02 | Tampering | Decision JSON body | accept | Local-only tool; the user is the only person who can POST. Invalid behavior values fall through to deny with error message. |
| T-02-03 | Denial of Service | Server bind | accept | Port 0 avoids collisions with concurrent sessions. If bind fails, binary exits with error on stderr. |
| T-02-04 | Elevation of Privilege | Browser launch | accept | webbrowser::open runs the OS default browser opener; no privilege escalation vector beyond what the user's browser already has. |
| T-02-05 | Repudiation | Decision audit | accept | No audit log needed for a local single-user tool. The decision is written to stdout for Claude Code to consume. |
</threat_model>
