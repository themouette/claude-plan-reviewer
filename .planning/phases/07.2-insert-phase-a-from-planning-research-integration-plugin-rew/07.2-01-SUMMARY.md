# Phase 07.2 Plan 01: ClaudeIntegration Plugin Directory Model Summary

Claude Code integration rewritten to use the plugin directory model instead of bare `settings.json` hook entries. Plugin config now lives in files plan-reviewer owns (`~/.local/share/plan-reviewer/claude-plugin/`), enabling version-aware rewrites by `update`.

## What Was Built

`src/integrations/claude.rs` completely rewritten to:

1. **install()** writes two artifacts:
   - Plugin directory at `~/.local/share/plan-reviewer/claude-plugin/.claude-plugin/plugin.json` (name, version, description)
   - Hook config at `~/.local/share/plan-reviewer/claude-plugin/hooks/hooks.json` (PermissionRequest/ExitPlanMode with bare `"plan-reviewer"` command)
   - Adds `extraKnownMarketplaces["plan-reviewer-local"]` to `~/.claude/settings.json` (directory source pointing to plugin dir)
   - Adds `enabledPlugins["plan-reviewer@plan-reviewer-local"] = true` to `~/.claude/settings.json`

2. **uninstall()** reverses all artifacts:
   - Removes plugin directory recursively (`remove_dir_all`)
   - Removes both `extraKnownMarketplaces` and `enabledPlugins` entries from settings.json
   - Does NOT touch old-style `hooks.PermissionRequest` entries (Phase 07.3 concern)

3. **is_installed()** checks `enabledPlugins["plan-reviewer@plan-reviewer-local"]` (new idempotency key)

4. **claude_legacy_hook_installed()** kept `pub(crate)` for Phase 07.3 migration use â€” detects old-style ExitPlanMode hook entries

## Tests

21 unit tests in `src/integrations/claude.rs` covering:
- Plugin directory creation and file contents
- Version field matches CARGO_PKG_VERSION
- extraKnownMarketplaces and enabledPlugins entries
- Idempotency (double install produces exactly one of each entry)
- Key preservation (existing settings.json keys survive install)
- Error on missing binary_path
- Uninstall removes plugin dir and both entries
- Old-style hook entry preserved on uninstall
- is_installed checks new key not legacy hook

Integration tests in `tests/integration/install_uninstall.rs` updated to match new plugin model (replaces checks for hooks.PermissionRequest with checks for plugin directory files and settings.json registration entries).

Full suite: 76 unit tests + 13 integration tests all pass.
