## Summary

Phase 07.2 goal is **fully achieved**. All five success criteria are satisfied:

1. Claude Code plugin directory model is correctly implemented in `src/integrations/claude.rs`. The install writes a plugin manifest (`plugin.json` with `env!("CARGO_PKG_VERSION")`), a hooks config (`hooks/hooks.json` with ExitPlanMode hook), and two `~/.claude/settings.json` registration entries (`extraKnownMarketplaces` + `enabledPlugins`). Idempotency is enforced via `enabledPlugins[PLUGIN_KEY]` check.

2. Gemini CLI extension directory model is correctly implemented in `src/integrations/gemini.rs`. The install writes `gemini-extension.json` and `hooks/hooks.json` to `~/.gemini/extensions/plan-reviewer/` and does not touch `~/.gemini/settings.json`. Tests explicitly verify this constraint.

3. `update.rs` `refresh_integrations()` is wired into `perform_update()` after binary replacement. It detects installed integrations by manifest presence (Claude's `plugin.json`, Gemini's `gemini-extension.json`) or version comment (OpenCode `.mjs`), and rewrites stale files. 10 unit tests cover all paths including absent integrations (no-op).

4. The OpenCode plugin template (`opencode_plugin.mjs`) has the version comment on line 2 (`// plan-reviewer-version: __PLAN_REVIEWER_VERSION__`), replaced at install time. `read_mjs_version()` correctly parses it. Tests verify both placeholder replacement and comment parsing.

5. Idempotency is enforced for all three integrations: Claude uses `enabledPlugins` key check, Gemini always rewrites (files are self-contained JSON, re-writing is safe), OpenCode uses array membership check. Integration tests `install_claude_is_idempotent` and `install_gemini_is_idempotent` pass.

The HelpView gap-closure plan (07.2-04) was also completed: `ui/src/App.tsx` now shows `plan-reviewer install/uninstall <integration>` commands with per-integration descriptions. All manual settings.json surgery instructions have been removed.

Test suite: **103/103 tests pass** (90 unit + 13 integration).
