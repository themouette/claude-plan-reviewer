**install() implementation:**

1. Extract binary_path from ctx (error if None).
2. Compute `plugin_dir = claude_plugin_dir(&ctx.home)`.
3. Write plugin directory files:
   a. Create `plugin_dir/.claude-plugin/` via `create_dir_all`.
   b. Write `plugin_dir/.claude-plugin/plugin.json`:
      ```json
      {"name":"plan-reviewer","version":"<crate_version>","description":"Plan review hook for Claude Code"}
      ```
   c. Create `plugin_dir/hooks/` via `create_dir_all`.
   d. Write `plugin_dir/hooks/hooks.json`:
      ```json
      {"hooks":{"PermissionRequest":[{"matcher":"ExitPlanMode","hooks":[{"type":"command","command":"plan-reviewer"}]}]}}
      ```
      Note: Use bare "plan-reviewer" for command (not binary_path). This is consistent with the architecture doc. Phase 07.3 changes it to "plan-reviewer hook".
4. Read or create `~/.claude/settings.json` (same pattern as current code).
5. Idempotency check: if `enabledPlugins[PLUGIN_KEY]` exists in settings, print "already configured" and return Ok(()).
6. Add `extraKnownMarketplaces` entry using entry().or_insert_with():
   ```json
   "plan-reviewer-local": {"source":{"source":"directory","path":"<plugin_dir absolute path>"}}
   ```
7. Add `enabledPlugins` entry:
   ```json
   "plan-reviewer@plan-reviewer-local": true
   ```
8. Write settings.json back with serde_json::to_string_pretty.
9. Print success messages to stdout.

**Do NOT remove the old PermissionRequest hook entry during install.** The old entry may coexist until Phase 07.3 migration.

**uninstall() implementation:**

1. Remove plugin directory recursively: check `plugin_dir.exists()` first, then `std::fs::remove_dir_all(plugin_dir)`. Print skip message if absent.
2. Read `~/.claude/settings.json`. If missing, print "nothing to uninstall" and return Ok(()).
3. Remove `extraKnownMarketplaces["plan-reviewer-local"]` if present (use `as_object_mut().map(|obj| obj.remove("plan-reviewer-local"))`).
4. Remove `enabledPlugins["plan-reviewer@plan-reviewer-local"]` if present.
5. Do NOT touch old-style `hooks.PermissionRequest` entries (Phase 07.3 handles that).
6. Write settings.json back.

**is_installed() implementation:**

Check if `enabledPlugins["plan-reviewer@plan-reviewer-local"]` exists in settings.json. This is the new idempotency key (replaces the old ExitPlanMode matcher check).

**Keep the old `claude_is_installed` function renamed to `claude_legacy_hook_installed`** for Phase 07.3 migration use. Make it `pub(crate)` so it can be used from update.rs later.

| Boundary | Description |
|----------|-------------|
| Binary -> filesystem | Plugin directory written to ~/.local/share/; settings.json mutated |
| settings.json content | Untrusted existing JSON parsed and mutated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.2-01 | Tampering | settings.json | mitigate | Parse with serde_json; reject invalid JSON with descriptive error ("refusing to overwrite"); never use string manipulation for JSON mutation |
| T-07.2-02 | Tampering | plugin.json | accept | Plugin files are always overwritten on install; a tampered plugin.json is replaced on next install. Low risk for local-only tool |
| T-07.2-03 | Information Disclosure | plugin_dir path | mitigate | Path computed from HOME + hardcoded relative path only; no user-supplied path fragments accepted (same pattern as T-07-05) |
| T-07.2-04 | Denial of Service | remove_dir_all | mitigate | Check path.exists() before calling remove_dir_all; handle ErrorKind::NotFound gracefully |
</threat_model>
