pub(crate) fn gemini_extension_dir(home: &str) -> PathBuf {

**Remove entirely:** `gemini_settings_path`, `gemini_hook_entry`, `gemini_is_installed` (the old settings.json helpers). They are no longer needed because Gemini uses auto-discovery.

**Keep the old `gemini_is_installed` function renamed to `gemini_legacy_hook_installed`** as `pub(crate)` for Phase 07.3 migration use. It takes a `serde_json::Value` and checks for `hooks.BeforeTool` entries with `name: "plan-reviewer"`.

**install() implementation:**

1. Extract binary_path from ctx (error if None). The binary_path value is validated to exist as a guard, but hooks.json writes the bare command name `"plan-reviewer"` (not the full binary_path). No explicit PATH check is performed during install -- the bare command relies on PATH availability at runtime, which is acceptable because cargo-dist installations place the binary in PATH by default, and Phase 07.3 changes the command to `"plan-reviewer hook"` with the same PATH dependency.
2. Compute `ext_dir = gemini_extension_dir(&ctx.home)`.
3. Write extension directory files:
   a. Create `ext_dir/` via `create_dir_all`.
   b. Write `ext_dir/gemini-extension.json`:
      ```json
      {"name":"plan-reviewer","version":"<crate_version>","description":"Plan review hook for Gemini CLI"}
      ```
   c. Create `ext_dir/hooks/` via `create_dir_all`.
   d. Write `ext_dir/hooks/hooks.json`:
      ```json
      {"hooks":{"BeforeTool":[{"matcher":"exit_plan_mode","hooks":[{"name":"plan-reviewer","type":"command","command":"plan-reviewer","timeout":300000}]}]}}
      ```
      Note: Use bare "plan-reviewer" for command. Phase 07.3 changes it to "plan-reviewer hook". Include timeout 300000 (5 minutes) -- mandatory per Phase 6 research.
4. Print success messages to stdout.
5. Do NOT read or modify `~/.gemini/settings.json`. Gemini CLI auto-discovers extensions.

**uninstall() implementation:**

1. Compute `ext_dir = gemini_extension_dir(&ctx.home)`.
2. If `ext_dir.exists()`: call `std::fs::remove_dir_all(&ext_dir)`, print "removed" message.
3. If `!ext_dir.exists()`: print "not found (skipping)" message.
4. Do NOT touch `~/.gemini/settings.json`. Old settings.json entries are Phase 07.3 migration concern.
5. Return Ok(()).

**is_installed() implementation:**

Check if `gemini_extension_dir(&ctx.home).join("gemini-extension.json").exists()`. This is the new idempotency key.

Write all tests listed in the behavior block. Use tempdir pattern.
