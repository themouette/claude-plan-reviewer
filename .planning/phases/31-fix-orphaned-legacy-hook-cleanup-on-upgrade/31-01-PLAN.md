# Plan 31-01: Fix Orphaned Legacy Hook Cleanup on Upgrade

## Objective

When `refresh_integrations_with_home` runs Case 3 (plugin manifest exists, version stale), any bare `ExitPlanMode` / `BeforeTool` legacy entry left in `settings.json` from a pre-plugin install is never cleaned up, causing the hook to fire twice per plan approval. Extract the bare-entry removal into shared helpers and call them from Case 3.

## Context

- The removal logic already exists in `perform_claude_migration` (`update.rs:239-244`) and `perform_gemini_migration` (`update.rs:267-284`).
- Case 2 (no manifest, bare entry present) already calls these migration functions which remove the bare entry.
- Case 3 (manifest present, version stale) calls only `write_claude_plugin_files` / `write_gemini_extension_files` and never touches `settings.json` â€” leaving the bare entry in place.
- The test `test_claude_case2_skips_when_manifest_exists` (`update.rs:858`) currently asserts that when the manifest exists, the old entry survives. This test must be inverted to assert the new behavior.

## Tasks

### Task 1 â€” Extract `remove_claude_legacy_hook` helper

<read_first>
- src/update.rs (lines 196-249 â€” perform_claude_migration, specifically the settings.json mutation block)
- src/integrations/claude.rs (claude_legacy_hook_installed, claude_settings_path)
</read_first>

<action>
In `src/update.rs`, add a private function `remove_claude_legacy_hook(home: &str)` that:
1. Reads `{home}/.claude/settings.json` (returns early silently if missing or unparseable)
2. Removes any entry from `root["hooks"]["PermissionRequest"]` whose `matcher` equals `"ExitPlanMode"`
3. Writes `settings.json` back only if a removal actually occurred (compare array length before/after)
4. Prints `plan-reviewer: removed legacy ExitPlanMode hook entry from Claude settings` if removed

Extract this logic from `perform_claude_migration` (lines 239-244) rather than duplicating it â€” call `remove_claude_legacy_hook(home)` from `perform_claude_migration` after the registration entries are added.
</action>

<acceptance_criteria>
- `remove_claude_legacy_hook` is a private `fn` in `src/update.rs`
- `perform_claude_migration` calls it instead of inlining the array-filter logic
- When called with a home dir containing `settings.json` with an `ExitPlanMode` entry, the entry is absent after the call
- When called with a home dir containing `settings.json` without any `ExitPlanMode` entry, `settings.json` is not rewritten (mtime unchanged)
- When called with a home dir without `settings.json`, the function returns without error
</acceptance_criteria>
