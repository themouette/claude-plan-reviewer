Add a new helper function (placed after `extract_diff`, before `main`):

```rust
/// Extract plan Markdown from hook input.
///
/// Claude Code sends the plan inline in `tool_input.plan`.
/// Gemini CLI sends a filesystem path in `tool_input.plan_path`.
/// If both are present, prefer the inline plan (Claude Code case).
/// If neither is present, return an empty string.
fn extract_plan_content(tool_input: &hook::ToolInput) -> String {
    if let Some(ref plan) = tool_input.plan {
        return plan.clone();
    }
    if let Some(ref plan_path) = tool_input.plan_path {
        match std::fs::read_to_string(plan_path) {
            Ok(content) => return content,
            Err(e) => {
                eprintln!("Failed to read plan file at {}: {}", plan_path, e);
                std::process::exit(1);
            }
        }
    }
    String::new()
}
```

// Build integration-specific output JSON
let output_json: serde_json::Value = if hook_input.is_gemini() {
    // Gemini CLI: flat {decision, reason, systemMessage} format
    match decision.behavior.as_str() {
        "allow" => serde_json::json!({ "decision": "allow" }),
        _ => serde_json::json!({
            "decision": "deny",
            "reason": decision.message.as_deref().unwrap_or("Denied without message"),
            "systemMessage": "Plan denied by plan-reviewer. Please revise the plan."
        }),
    }
} else {
    // Claude Code: nested hookSpecificOutput format
    let hook_output = match decision.behavior.as_str() {
        "allow" => HookOutput::allow(),
        "deny" => HookOutput::deny(
            decision.message.unwrap_or_else(|| "Denied without message".to_string()),
        ),
        other => {
            eprintln!("Unknown decision behavior: {}", other);
            HookOutput::deny(format!("Unknown behavior: {}", other))
        }
    };
    serde_json::to_value(hook_output).expect("HookOutput serialization cannot fail")
};

serde_json::to_writer(std::io::stdout(), &output_json).expect("failed to write hook output");

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin JSON -> plan_path -> fs::read_to_string | Untrusted path from agent hook payload used for file read |
| Decision -> stdout JSON | Browser decision converted to agent-specific JSON format |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | Path traversal via plan_path | accept | `plan_path` is used for read-only `fs::read_to_string` only -- never executed, never passed to shell, never used to construct write paths. The path originates from Gemini CLI (trusted local process), not from network input. Even if a crafted path reads an unintended file, the content is only rendered as Markdown in the local browser UI (same trust level as the user). |
| T-06-06 | Spoofing | Stdout pollution corrupting JSON response | mitigate | All debug output uses `eprintln!` exclusively. The only `stdout` write is `serde_json::to_writer(std::io::stdout(), &output_json)`. No `println!` allowed in hook-flow code paths. Enforced by code review and existing convention. |
| T-06-07 | Tampering | Wrong JSON envelope for integration | mitigate | Detect integration via `hook_input.is_gemini()` (checks `hook_event_name == "BeforeTool"`). Claude Code path falls through to existing `HookOutput` serialization. Test both formats in unit tests. |
| T-06-08 | Denial of Service | Hook timeout with Gemini CLI | accept | Gemini CLI enforces a hook timeout (default 60s, we set 300000ms in install). If user takes longer than 5 minutes, Gemini kills the hook. This is Gemini CLI behavior, not controllable by plan-reviewer. The 300000ms timeout set in Plan 01 is the mitigation. |
</threat_model>
