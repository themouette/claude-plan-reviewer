  <action>
Replace the stub in `src/integrations/gemini.rs` with a full implementation mirroring `claude.rs` structure. The implementation has these private helper functions and one public struct:

**`gemini_settings_path(home: &str) -> PathBuf`**: Returns `PathBuf::from(home).join(".gemini/settings.json")`.

**`gemini_hook_entry(binary_path: &str) -> serde_json::Value`**: Returns:
```json
{
    "matcher": "exit_plan_mode",
    "hooks": [
        {
            "name": "plan-reviewer",
            "type": "command",
            "command": "<binary_path>",
            "timeout": 300000
        }
    ]
}
```
The `timeout: 300000` (5 minutes) is MANDATORY — Gemini CLI defaults to 60 seconds which is too short for interactive review (per 06-RESEARCH.md Pitfall 1).

**`gemini_is_installed(settings: &serde_json::Value) -> bool`**: Scans `settings["hooks"]["BeforeTool"]` (array) for any entry whose `hooks[]` sub-array contains an element with `"name": "plan-reviewer"`. This is the idempotency key — NOT the binary path (matches the claude.rs convention, robust against binary relocation).

**`GeminiIntegration::install(&self, ctx: &InstallContext) -> Result<(), String>`**:
1. Extract `binary_path` from ctx, return Err if None
2. Compute settings_path via `gemini_settings_path(&ctx.home)`
3. Read existing JSON or start with `serde_json::json!({})`
4. If file exists but is not valid JSON, return Err with message "cannot parse {path}: {error} -- refusing to overwrite. Fix or remove the file first."
5. Ensure root is an object (reset to `{}` with eprintln warning if not)
6. Ensure `root["hooks"]` exists as an object (use `entry("hooks").or_insert_with(|| json!({}))`)
7. Validate `root["hooks"]` is an object, return Err if not
8. Ensure `root["hooks"]["BeforeTool"]` exists as an array (use `entry("BeforeTool").or_insert_with(|| json!([]))`)
9. Validate `root["hooks"]["BeforeTool"]` is an array, return Err if not
10. Call `gemini_is_installed(&root)` — if true, println message "plan-reviewer: BeforeTool hook already configured in {path} (no changes made)" and return Ok
11. Push `gemini_hook_entry(binary_path)` into the BeforeTool array
12. Pretty-print JSON with `serde_json::to_string_pretty`
13. Create parent directory `~/.gemini/` if it does not exist (`fs::create_dir_all`)
14. Write the file
15. println "plan-reviewer: BeforeTool hook installed in {path}"
16. println "plan-reviewer: hook command set to: {binary_path}"
17. Return Ok

**`GeminiIntegration::uninstall(&self, ctx: &InstallContext) -> Result<(), String>`**:
1. Compute settings_path
2. If file does not exist: println "no settings file found at {path} (nothing to uninstall)", return Ok
3. Read and parse JSON. If invalid JSON: eprintln warning, return Ok (same pattern as claude.rs)
4. Call `gemini_is_installed` — if false: println "BeforeTool hook not found in {path} (no changes made)", return Ok
5. Filter: for each entry in `root["hooks"]["BeforeTool"]` array, retain entries where NONE of the `hooks[]` sub-elements have `"name": "plan-reviewer"`. This removes the entire matcher block that contains plan-reviewer, not just the inner hook.
6. Pretty-print and write back
7. println "plan-reviewer: BeforeTool hook removed from {path}"
8. Return Ok

**`GeminiIntegration::is_installed(&self, ctx: &InstallContext) -> bool`**:
1. Compute settings_path
2. If file does not exist, return false
3. Read and parse — if any error, return false
4. Return `gemini_is_installed(&json)`

Use `use super::{InstallContext, Integration};` and `use std::path::PathBuf;` as imports. Follow claude.rs error message style exactly (prefix with "plan-reviewer:" or "plan-reviewer install:" as appropriate).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Install writes to ~/.gemini/settings.json | Filesystem write to user config directory |
| Install reads current_exe() for binary_path | Trusts OS-reported binary location |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | settings.json corruption | mitigate | Return `Err` if JSON parse fails -- refuse to overwrite corrupt file (same pattern as claude.rs). Pretty-print on write to maintain readability. |
| T-06-02 | Tampering | JSON injection via binary_path | accept | binary_path comes from `std::env::current_exe()` in install.rs, not from user input. The value is embedded as a JSON string via `serde_json::json!()` which auto-escapes. Low risk. |
| T-06-03 | Denial of Service | Duplicate hook entries | mitigate | Idempotency check via `gemini_is_installed()` before any write. Match on `"name": "plan-reviewer"` in hooks sub-array. |
| T-06-04 | Information Disclosure | Settings file permissions | accept | Follows OS default file permissions for `~/.gemini/settings.json`. Same trust model as Gemini CLI itself. |
</threat_model>
