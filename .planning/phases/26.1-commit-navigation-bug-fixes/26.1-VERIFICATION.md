### Human Verification Required

#### 1. Empty commit selection produces error panel (post-phase CR-01)

**Test:** Open `/code-review` in the browser. Open the CommitDrawer. Uncheck all commit checkboxes one by one until none are checked.

**Expected:** The DiffPane should show an appropriate empty/no-selection state (e.g., "No commits selected" or similar), NOT the generic "Could not load diff — The server returned an error" error panel.

**Why human:** The REVIEW.md for this phase (filed 2026-05-25) identified a new BLOCKER (CR-01 in the review — distinct from the original Phase 26 CR-01) that was NOT part of the 6 bugs this phase was scoped to fix. The root cause is: `CodeReviewApp.tsx:28` passes `shas: []` to `fetchFilteredBranchDiff` when `checkedCommitShas.length < commits.length` (which includes the 0 < N case). `[].every(...)` is vacuously true in JavaScript, so `allFailed` becomes `true` and the function returns `{ files: [], error: 'all fetches failed' }`. DiffPane renders this as the error panel. A fix requires either a guard in `fetchFilteredBranchDiff` for `shas.length === 0` (Option A) or an additional `checkedCommitShas.length > 0` condition in the selector (Option B). This was not in-scope for Phase 26.1 and needs a follow-up plan. Human must decide whether to block on this or proceed to Phase 27 with a known regression.
