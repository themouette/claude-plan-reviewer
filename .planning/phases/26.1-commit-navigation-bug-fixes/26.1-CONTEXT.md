<decisions>
## Implementation Decisions

### WR-02: fetchFilteredBranchDiff Return Type (D-01)

- **D-01:** Change `fetchFilteredBranchDiff` from `Promise<FileDiff[]>` to `Promise<FetchDiffResult>`, using `Promise.allSettled` internally. Returns `{ files: [], error: 'all fetches failed' }` when every SHA-level fetch fails or returns non-ok. Returns `{ files: [...], error: null }` when at least one SHA succeeds. Consistent with `fetchDiffOnce` and `fetchCommitDiffOnce` ‚Äî all three fetch functions share the same return shape.
- **D-01b:** The three existing `fetchFilteredBranchDiff` tests that check `result.length` must be updated to check `result.files.length`.

### CR-02: seededRef Sentinel (D-02)

- **D-02:** Replace the `commits.length === 0` guard with a `useRef<boolean>(false)` sentinel (`seededRef`). One-and-done: set to `true` on first seed, never reset. The app has no retry-fetch mechanism today, so re-seeding on re-fetch is out of scope.

### Testing Coverage (D-03)

- **D-03:** Two new Vitest test cases required alongside the fixes:
  1. `fetchCommitDiffOnce` rejects an invalid SHA (e.g. `'../branch'`) without calling `doFetch` ‚Äî covers CR-01.
  2. `fetchFilteredBranchDiff` returns `{ files: [], error: 'all fetches failed' }` when all SHAs return 500 ‚Äî covers WR-02 all-fail path.
  3. Update the 3 existing `fetchFilteredBranchDiff` tests to use `result.files` after the return type change.

### Plan Structure (D-04)

- **D-04:** One consolidated plan (`26.1-01-PLAN.md`). TDD-first within the plan: write failing tests for CR-01 and WR-02 first, then implement all 6 fixes, then verify `npm test` passes. No wave split.

### Claude's Discretion

- Exact SHA regex pattern (the fix specifies `/^[0-9a-f]{7,40}$/i` ‚Äî this is the canonical form; no deviation needed)
- Error string wording for WR-02 all-fail case (`'all fetches failed'` or equivalent)
- Whether CR-01 validation is in `fetchCommitDiffOnce` (pre-fetch guard) or extracted to a helper ‚Äî either is fine

</decisions>

## Existing Code Insights

### Reusable Assets
- `FetchDiffResult` interface in `useDiff.ts` ‚Äî already the shared return type for `fetchDiffOnce` and `fetchCommitDiffOnce`; `fetchFilteredBranchDiff` adopts the same interface (D-01)
- `Promise.allSettled` ‚Äî use this instead of `Promise.all` so partial failures don't abort the whole batch
- `useRef` from React ‚Äî already imported in `CodeReviewApp.tsx`; `seededRef` follows the same pattern as the existing keyboard-nav cleanup refs

### Established Patterns
- **Injectable doFetch pattern**: all fetch functions take a `DoFetch` parameter for testability ‚Äî CR-01 validation goes before the `doFetch(url)` call
- **TDD-first (CLAUDE.md)**: write failing tests before implementation; `fetchCommitsOnce` and `fetchDiffOnce` set the test structure precedent
- **cancelledRef / seededRef**: `useRef(false)` for one-shot guards in effects ‚Äî already used for cancel-on-unmount; `seededRef` is the same idiom

### Integration Points
- `CodeReviewApp.tsx` ‚Äî caller of `fetchFilteredBranchDiff`; will need to read `result.files` and `result.error` after D-01 type change
- `useDiff.ts` (export list) ‚Äî `fetchFilteredBranchDiff` is already exported and imported by tests; signature change is non-breaking at the module boundary but requires test updates

## Specific Ideas

- CR-01 fix: validate SHA with `/^[0-9a-f]{7,40}$/i` in `fetchCommitDiffOnce` before building the URL. Return `{ files: [], error: 'invalid sha' }` immediately for non-matching input.
- CR-01 test: call `fetchCommitDiffOnce('../branch', doFetch)` and assert `doFetch` was never called (use a spy or flag).
- WR-02 all-fail detection: use `Promise.allSettled`, then `results.every(r => r.status === 'rejected' || r.value.error !== null)` to determine all-fail condition.
- WR-03: add `role="button"`, `tabIndex={0}`, `onKeyDown={e => e.key === 'Enter' && onCommitClick(sha)}` to the CommitRow `<li>` element (lines 109-170 of CommitDrawer.tsx).
- WR-04: in `DiffPane.tsx`, when `commits.find(c => c.sha === activeCommitSha)` returns undefined, show the short SHA as fallback (e.g. `activeCommitSha.slice(0, 7)`).
