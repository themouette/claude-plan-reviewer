# Phase 26.1 Plan 01: Commit Navigation Bug Fixes Summary

SHA-validated fetchCommitDiffOnce, FetchDiffResult-returning fetchFilteredBranchDiff, seededRef re-seed sentinel, findIndex -1 guard, keyboard-accessible CommitRow, and short-SHA fallback strip.

## What Was Built

Six targeted patches closing the 2 blockers (CR-01, CR-02) and 4 warnings (WR-01 through WR-04) identified in the Phase 26 code review.

### CR-01: Invalid SHA Guard (Security — Tampering T-26.1-01)

`fetchCommitDiffOnce` now validates the `sha` argument against `SHA_RE = /^[0-9a-f]{7,40}$/i` before constructing the `/api/diff/commit/${sha}` URL. Non-matching input (path-traversal strings like `'../branch'`, arbitrary strings) returns `{ files: [], error: 'invalid sha' }` without calling `doFetch`. The guard is at module scope so it is reusable.

### WR-02: fetchFilteredBranchDiff Return Type (Correctness)

`fetchFilteredBranchDiff` now returns `Promise<FetchDiffResult>` instead of `Promise<FileDiff[]>`. Internally uses `Promise.allSettled` so individual SHA failures don't abort the batch. Returns `{ files: [], error: 'all fetches failed' }` when every fetch fails; returns `{ files: [...], error: null }` on partial success. The `dispatch` branch-union arm now passes the result through directly, surfacing the all-fail error to the hook's `error` state.

### CR-02: seededRef Sentinel (Correctness)

`CodeReviewApp` seed effect replaced the `checkedCommitShas.length === 0` guard with a `useRef(false)` sentinel (`seededRef`). Set to `true` on the first seed, never reset. The new guard `if (seededRef.current || commits.length === 0) return` prevents re-seeding after the user explicitly empties the checked set.

### WR-01: findIndex -1 Guard (Correctness)

The keyboard handler in `CodeReviewApp` now returns early when `commits.findIndex(...)` returns `-1`, preventing out-of-range index access (`commits[idx - 1]`, `commits[idx + 1]`) when `activeCommitSha` is stale.

### WR-03: CommitRow Keyboard Accessibility

The `<li>` inside `CommitDrawer`'s commit list now carries `role="button"`, `tabIndex={0}`, and an `onKeyDown` handler that calls `onCommitClick(commit.sha)` when `e.key === 'Enter'`. The existing `onClick` handler is preserved unchanged.

### WR-04: DiffPane Unknown-SHA Fallback Strip

`DiffPane` now renders a short-SHA fallback strip (`activeCommitSha.slice(0, 7)`) when `viewMode === 'commit' && activeCommitSha !== null && activeCommit === null`. The existing commit title strip is updated with an explicit `activeCommit !== null` guard so the two strips are mutually exclusive.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Updated test SHA fixtures to valid 7-character hex values**
- **Found during:** Task 2 (GREEN phase — tests failed to pass after SHA_RE guard was implemented)
- **Issue:** The plan stated that `'abc123'` (6 chars), `'sha1'` (not hex), and `'sha2'` (not hex) would "pass the SHA regex `/^[0-9a-f]{7,40}$/i`" — this was factually incorrect. These strings all fail the regex (6-char hex minimum, 's' is not a hex digit). After implementing the SHA guard in Task 2, all pre-existing `fetchCommitDiffOnce` and `fetchFilteredBranchDiff` tests that used these strings began failing.
- **Fix:** Updated test fixtures: `'abc123'` → `'abc1234'`, `'sha1'` → `'aa11111'`, `'sha2'` → `'bb22222'`. All are valid 7-character lowercase hex strings that pass the regex.
- **Files modified:** `ui/src/code-review/hooks/useDiff.test.ts`
- **Commit:** 3e90a53 (included in Task 2 commit)
