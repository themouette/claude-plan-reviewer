# Phase 06.1 Plan 02: Integration Test Infrastructure and Install/Uninstall Tests Summary

**One-liner:** Integration test infrastructure with HOME-isolated install/uninstall tests covering Claude and Gemini for both idempotency and clean-system paths.

## What Was Built

### Task 1: Dev-dependencies and fixture files (commit 7996cb3)

Added three dev-dependencies to `Cargo.toml`:
- `assert_cmd = "2"` — CLI subprocess testing (uses `CARGO_BIN_EXE_*` for binary location)
- `predicates = "3"` — composable assertions integrated with assert_cmd
- `ureq = { version = "3", features = ["json"] }` — sync HTTP client for Plan 03's server cycle tests

Created two JSON fixture files in `tests/fixtures/`:
- `hook_input_claude.json` — minimal PermissionRequest stdin payload matching `HookInput` struct (`hook_event_name: "PermissionRequest"`)
- `hook_input_gemini.json` — minimal BeforeTool stdin payload (`hook_event_name: "BeforeTool"`)

Both fixtures use `cwd: "/tmp"` (exists on all Unix systems, not a git repo so `extract_diff` returns empty string safely), synthetic session IDs, and inline `plan` in `tool_input` (no file on disk required).

### Task 2: Install/uninstall integration tests (commit 63e0690)

Created `tests/integration/main.rs` (entry point) and `tests/integration/install_uninstall.rs` with 9 test functions:

| Test | Coverage |
|------|----------|
| `help_includes_port_and_no_browser_flags` | Validates `--no-browser` and `--port` visible in help (depends on Plan 01 adding `--port`) |
| `install_claude_creates_settings_in_isolated_home` | Claude install creates `~/.claude/settings.json` with `ExitPlanMode` matcher |
| `install_claude_is_idempotent` | Double install produces exactly one ExitPlanMode entry |
| `install_gemini_creates_settings_in_isolated_home` | Gemini install creates `~/.gemini/settings.json` with `plan-reviewer` in BeforeTool hooks |
| `install_gemini_is_idempotent` | Double install produces exactly one plan-reviewer BeforeTool entry |
| `uninstall_claude_on_clean_system_exits_zero` | Uninstall with no settings file exits 0 |
| `uninstall_claude_after_install_removes_hook` | Install then uninstall removes ExitPlanMode from PermissionRequest array |
| `uninstall_gemini_on_clean_system_exits_zero` | Gemini uninstall with no settings file exits 0 |
| `uninstall_gemini_after_install_removes_hook` | Install then uninstall removes plan-reviewer from BeforeTool array |

All 8 install/uninstall tests pass. `help_includes_port_and_no_browser_flags` will pass after Plan 01 adds `--port` to the CLI and wave 1 outputs are merged.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Used `main.rs` instead of `mod.rs` for integration test entry point**
- **Found during:** Task 2 verification (`cargo test --test integration`)
- **Issue:** Cargo 2018+ edition requires `tests/integration/main.rs` as the entry point for directory-based integration test crates. Using `mod.rs` results in `error: no test target named 'integration'`.
- **Fix:** Renamed `tests/integration/mod.rs` → `tests/integration/main.rs`. Contents are identical (`mod install_uninstall;`).
- **Files modified:** `tests/integration/main.rs`
- **Commit:** 63e0690

**2. [Rule 1 - Bug] Fixed temporary value dropped while borrowed**
- **Found during:** Task 2 compilation
- **Issue:** `unwrap_or(&vec![])` creates a temporary `Vec` that is freed before the borrow ends, causing `E0716` compiler errors in two places.
- **Fix:** Added `let empty = vec![];` binding before the `unwrap_or` call.
- **Files modified:** `tests/integration/install_uninstall.rs`
- **Commit:** 63e0690
