<objective>
Add `--port <u16>` CLI flag to plan-reviewer so integration tests can bind the review server to a known port instead of relying on OS assignment.

Purpose: TEST-03 (server approve/deny cycle) requires the test to know which port the server is listening on BEFORE the process starts. A fixed `--port` flag is the simplest, most reliable approach (avoids parsing stderr pipes which risks deadlock).

Output: Modified `src/main.rs` (new clap arg, threaded through `run_hook_flow` and `async_main`) and `src/server.rs` (accept `port: u16` parameter, bind to `127.0.0.1:{port}`). Default `0` preserves existing OS-assigned behavior.
</objective>
