   **Helper: `find_free_port()`** — Bind a TcpListener to port 0, extract the assigned port, drop the listener. Returns `u16`. There is a small TOCTOU race between dropping the listener and the binary binding to the same port, but this is acceptable in tests.
   ```rust
   fn find_free_port() -> u16 {
       let listener = std::net::TcpListener::bind("127.0.0.1:0").unwrap();
       listener.local_addr().unwrap().port()
   }
   ```

   **Helper: `wait_for_port(port, timeout)`** — Poll `TcpStream::connect("127.0.0.1:{port}")` in a loop with 50ms sleep. Return `true` if connection succeeds before `timeout`, `false` otherwise.
   ```rust
   fn wait_for_port(port: u16, timeout: Duration) -> bool {
       let deadline = Instant::now() + timeout;
       while Instant::now() < deadline {
           if TcpStream::connect(format!("127.0.0.1:{}", port)).is_ok() {
               return true;
           }
           std::thread::sleep(Duration::from_millis(50));
       }
       false
   }
   ```

   **Helper: `binary_path()`** — Use `assert_cmd::cargo::cargo_bin("plan-reviewer")` to get the path to the compiled binary. This uses the `CARGO_BIN_EXE_plan_reviewer` env var set by Cargo during test compilation.
   ```rust
   fn binary_path() -> std::path::PathBuf {
       assert_cmd::cargo::cargo_bin("plan-reviewer")
   }
   ```

   **Helper: `spawn_hook_flow(fixture_json, port)`** — Encapsulates the common pattern of spawning the binary with `--no-browser --port {port}`, writing fixture JSON to stdin, closing stdin, and returning the `Child`. This avoids repeating boilerplate across tests.
   ```rust
   fn spawn_hook_flow(fixture_json: &str, port: u16) -> std::process::Child {
       let home = tempfile::TempDir::new().unwrap();
       // Leak the TempDir to keep it alive for the child process lifetime.
       // This is acceptable in tests — the OS will clean /tmp on reboot.
       let home = Box::leak(Box::new(home));

       let mut child = Command::new(binary_path())
           .env("HOME", home.path())
           .args(["--no-browser", "--port", &port.to_string()])
           .stdin(Stdio::piped())
           .stdout(Stdio::piped())
           .stderr(Stdio::piped())
           .spawn()
           .expect("failed to spawn plan-reviewer");

       // Write hook JSON to stdin, then CLOSE stdin (critical: binary blocks on read_to_string)
       child
           .stdin
           .as_mut()
           .unwrap()
           .write_all(fixture_json.as_bytes())
           .unwrap();
       drop(child.stdin.take()); // Close stdin pipe — EOF signals binary to proceed

       child
   }
   ```

   IMPORTANT: `drop(child.stdin.take())` is CRITICAL. Without it, the binary's `read_to_string(stdin)` blocks forever waiting for EOF, and the test deadlocks. This is documented in RESEARCH.md Pitfall 4.
