# Phase 06.1 Plan 03: Server Cycle Integration Tests Summary

**One-liner:** Integration tests for the full server approve/deny cycle using stdin injection, HTTP POST to `/api/decide`, and stdout JSON assertions for both Claude and Gemini output formats.

## What Was Built

### Task 1: Server cycle test helpers and approve/deny tests (commit c07b3d3)

Created `tests/integration/server_cycle.rs` with 4 test functions and 3 helper functions:

**Helpers:**
- `find_free_port()` — Binds TcpListener to port 0, extracts OS-assigned port, drops listener
- `wait_for_port(port, timeout)` — Polls TcpStream::connect in 50 ms loop until success or timeout
- `spawn_hook_flow(fixture_json, port)` — Spawns binary with `--no-browser --port <N>`, writes stdin JSON, closes pipe (critical: prevents deadlock)

**Tests:**

| Test | Input | POST body | Assertion |
|------|-------|-----------|-----------|
| `server_cycle_approve_claude` | hook_input_claude.json | `{"behavior":"allow"}` | `hookSpecificOutput.decision.behavior == "allow"`, `hookEventName == "PermissionRequest"` |
| `server_cycle_deny_claude` | hook_input_claude.json | `{"behavior":"deny","message":"Missing error handling"}` | `hookSpecificOutput.decision.behavior == "deny"`, `message == "Missing error handling"` |
| `server_cycle_approve_gemini` | hook_input_gemini.json | `{"behavior":"allow"}` | `decision == "allow"`, no `hookSpecificOutput` key |
| `server_cycle_deny_gemini` | hook_input_gemini.json | `{"behavior":"deny","message":"Needs tests"}` | `decision == "deny"`, `reason == "Needs tests"`, `systemMessage` contains "Plan denied" |

Updated `tests/integration/main.rs` to add `mod server_cycle;`.
