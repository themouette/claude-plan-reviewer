## Component Inventory

These are the new components this phase introduces, all under `ui/src/reviewer-v2/`:

### `ContentPane` (container)
- Role: `<main>` scroll container filling the center column
- Background: `var(--color-bg)`
- Overflow: `overflowY: 'auto'`
- Padding: 32px (xl) top/bottom, 32px left, 0px right (right edge is used by gutter icon)
- Max width for prose: 720px centered within the flex column (prevents line lengths > 80ch on wide viewports)
- Houses `PlanContent` and `SelectionToolbar`

### `PlanContent` (markdown host)
- `dangerouslySetInnerHTML` with HTML produced by `marked` + `marked-highlight` + `remark-gfm` equivalent
- Applies the `.plan-prose` CSS class (already defined in `index.css`)
- Uses `useTextSelection` hook (v2 copy at `reviewer-v2/useTextSelection.ts`) for character-offset serialization
- Mouse event delegation on the container detects paragraph hover (uses `closest('p, li, blockquote, h1, h2, h3, h4, h5, h6')`)
- `position: relative` to anchor the gutter icon

### `ParagraphHoverLayer` (state + event)
- Not a visible DOM element — tracks `hoveredParagraph: Element | null` via `onMouseMove` delegation
- Clears on `onMouseLeave` from the container
- Hidden while `selectedText` is non-empty (selection supersedes hover)

### `GutterIcon` (absolute-positioned `+` button)
- Visible only when `hoveredParagraph` is set and `selectedText` is empty
- Position: right edge of the paragraph element, vertically centered in the paragraph's bounding rect
- `position: absolute`, `right: -12px` (overlaps the boundary between center and right column — matches CONTENT-02 "right edge of the paragraph")
- Size: 24px × 24px
- Appearance: `+` character or inline SVG plus icon
- Color: `var(--color-focus)` at 70% opacity at rest, 100% on hover
- Background: `var(--color-surface)` circle, `border-radius: 50%`
- `zIndex: 5`
- `aria-label="Add comment to paragraph"`

### `SelectionToolbar` (floating action bar)
- Visible only when `selectedText` is non-empty
- Anchored to the bottom-right of the current selection range (`getBoundingClientRect()` of the Range end)
- Position: `position: fixed` (not absolute) — avoids scroll-offset math
- Layout: row of pills + "more" expander, identical visual treatment to `FloatingAnnotationAffordance` in `App.tsx`
- Height: 28px per pill
- Pill gap: 6px (sm/xs boundary)
- Background: `var(--color-surface)`, `border-radius: 6px`, `padding: 4px`, `box-shadow: 0 2px 8px rgba(0,0,0,0.3)`
- Pills: "Comment" (blue), "Delete" (red), "Replace" (amber) — same colors as existing `App.tsx` affordance
- `zIndex: 20`
- Toolbar disappears when selection is cleared

## Interaction States

### Paragraph Hover State
| What happens | Visual specification |
|---|---|
| Mouse enters paragraph | Background: `rgba(255,255,255,0.04)` applied to the `<p>` element via inline style or toggled CSS class `paragraph-hovered` |
| GutterIcon appears | `opacity: 1`, positioned at right edge of paragraph, vertically centered |
| Mouse leaves paragraph | Background cleared, GutterIcon hidden (`opacity: 0` or unmounted) |
| Transition | `transition: background 0.1s ease` on paragraph background |

### Text Selection State
| What happens | Visual specification |
|---|---|
| User selects text (mouseup) | Paragraph hover highlight cleared; `::highlight(selection-lock)` applied at `rgba(59,130,246,0.35)` |
| SelectionToolbar appears | Fixed-positioned at selection end rect bottom-right |
| User clicks toolbar pill | Annotation created; selection cleared; highlight removed |
| User clicks elsewhere in container | Selection cleared; highlight removed; hover state resumes |

### Focus States (keyboard accessibility)
- All interactive elements (GutterIcon, toolbar pills, "more" expander): `outline: 2px solid var(--color-focus)`, `outlineOffset: 2px`
- No custom focus style suppression

## Markdown Rendering Contract

The v2 ContentPane uses `marked` + `marked-highlight` (already in `package.json`) — NOT `react-markdown` as stated in REQUIREMENTS.md CONTENT-01. The project already uses `marked@18` with `marked-highlight@2` and `highlight.js@11`. The requirement names `react-markdown + remark-gfm + rehype-highlight` as the implementation but the existing codebase uses `marked` for the same outcome. The executor MUST use `marked` to stay consistent with the existing `App.tsx` pattern and avoid adding a new markdown library.

Rationale: `marked` with `gfm: true` already provides tables, task lists, strikethrough, and code highlighting via `marked-highlight` + `hljs`. Adding `react-markdown` would duplicate this capability with a different API and bundle cost.

GFM feature rendering contract:

| GFM feature | Rendered as |
|---|---|
| Tables | `<table>` with `<th>` / `<td>`, styled by `.plan-prose table/th/td` |
| Task list items | `<input type="checkbox" disabled>` inside `<li>`, `pointer-events: none` |
| Strikethrough | `<del>` element, browser default strikethrough styling |
| Code blocks (fenced) | `<pre><code class="hljs language-X">` with highlight.js tokens, styled by `.plan-prose pre` |
| Inline code | `<code>` styled by `.plan-prose code` |
| Blockquote | Left border 3px `var(--color-border)`, `var(--color-text-secondary)` |
| Horizontal rule | `border-color: var(--color-border)` |

Syntax highlighting theme: `highlight.js/styles/github-dark.css` (already imported in `App.tsx` — reuse in v2 entry or include via `index.css`).

## Accessibility Contract

- Content container: `role` not set (it is `<main>` semantically from the shell)
- GutterIcon: `<button>` element, `aria-label="Add comment to paragraph"`, keyboard activatable
- SelectionToolbar: pills are `<button>` elements, toolbar wrapped in `role="toolbar"` or `role="group"` with `aria-label="Annotation actions"`
- Toolbar pill mousedown must call `e.preventDefault()` to prevent selection clearing before click fires (critical pitfall from `App.tsx` line comment)
- Task list checkboxes: `disabled` attribute and `pointer-events: none`
