      1. Parse the SHA with `let oid = match git2::Oid::from_str(&sha) { Ok(o) => o, Err(_) => return (StatusCode::BAD_REQUEST, Json(serde_json::json!({"error": "invalid sha"}))).into_response() };`.       2. Open the repo: `let repo = match git2::Repository::open(&state.repo_path) { Ok(r) => r, Err(_) => return (StatusCode::INTERNAL_SERVER_ERROR, Json(serde_json::json!({"error": "failed to open repository"}))).into_response() };`.
      3. Find the commit: `let commit = match repo.find_commit(oid) { Ok(c) => c, Err(_) => return (StatusCode::NOT_FOUND, Json(serde_json::json!({"error": "commit not found"}))).into_response() };`. `find_commit` returns `Err` when the OID parses but does not exist in the repo — surface that as 404, distinct from the 400 above.
      4. Diff against the parent tree, falling back to the empty tree for the first commit (Pattern 6): `let commit_tree = match commit.tree() { Ok(t) => t, Err(_) => return (StatusCode::INTERNAL_SERVER_ERROR, Json(serde_json::json!({"error": "failed to read commit tree"}))).into_response() }; let parent_tree = commit.parent(0).ok().and_then(|p| p.tree().ok()); let mut opts = git2::DiffOptions::new(); opts.old_prefix("a/").new_prefix("b/"); let diff = match repo.diff_tree_to_tree(parent_tree.as_ref(), Some(&commit_tree), Some(&mut opts)) { Ok(d) => d, Err(_) => return (StatusCode::INTERNAL_SERVER_ERROR, Json(serde_json::json!({"error": "diff failed"}))).into_response() };`. Note: passing `None` for `parent_tree.as_ref()` when the commit has no parent makes git2 diff against the empty tree.
      5. Reuse the existing `build_file_diffs(&diff)` helper from 24-01 and return `(StatusCode::OK, Json(file_diffs)).into_response()`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → axum (127.0.0.1) | Local-only HTTP listener; `:sha` path segment is the only request-controlled input crossing this boundary in Phase 24. |
| axum handler → libgit2 | Path-extracted `sha: String` is passed to `git2::Oid::from_str` and then to `repo.find_commit`. libgit2 rejects malformed OIDs at the `Oid::from_str` boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-PT | Tampering / Path traversal | `:sha` path parameter on `/api/diff/commit/:sha` | mitigate | Validate via `git2::Oid::from_str(&sha)`; on `Err` return `StatusCode::BAD_REQUEST` + JSON `{"error":"invalid sha"}`. `Oid::from_str` enforces hex characters and exact length, blocking `../`, NUL, and any non-hex content. Asserted by test #4 in Task 2 (`get_diff_commit_with_invalid_sha_returns_400`). |
| T-24-NF | Information disclosure | Unknown-but-valid SHA leaking repo internals | mitigate | Return `StatusCode::NOT_FOUND` with a static `{"error":"commit not found"}` body — no libgit2 error string is forwarded to the client. Asserted by test #5 (`get_diff_commit_with_unknown_sha_returns_404`). |
| T-24-DOS | Denial of service | Per-commit diff size on a pathological large commit | accept (Phase 24) | No size cap; local single-user model bounds risk. Revisit if the Phase 25 UI hits practical limits. |
| T-24-SC | Tampering | cargo install of `tower` and `http-body-util` (dev-deps) | mitigate | Both crates are listed Approved in the Phase 24 Package Legitimacy Audit (RESEARCH.md) — they are already transitive deps of `axum 0.8` (verified in Cargo.lock); declaring them under `[dev-dependencies]` does not pull anything new from crates.io. No `[ASSUMED]` or `[SUS]` packages introduced. |
| T-24-OPEN | Information disclosure | Repository state on any local request | accept | Inherited from 24-01; out of new scope here. |

The threats specific to 24-01 (T-24-01..T-24-04) carry over and remain in effect; this register only enumerates incremental threats introduced by the per-commit endpoint and the test deps.
