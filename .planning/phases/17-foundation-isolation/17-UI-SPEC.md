## Layout Shell Contract

This section is specific to Phase 17. It defines the exact structure the executor must implement.

### Grid specification

```
┌─────────────────────────────────────────────────────────────┐
│  ReviewerV2Header  (48px height, sticky top)                │
├────────────┬──────────────────────────────┬─────────────────┤
│            │                              │                 │
│  Outline   │      Content                 │  Comments       │
│  pane      │      pane                    │  pane           │
│  (200px    │      (flex: 1 grow,          │  (280px         │
│   fixed)   │       min 0, scrollable)     │   fixed)        │
│            │                              │                 │
└────────────┴──────────────────────────────┴─────────────────┘
```

**Implementation:** CSS Grid or Flexbox with Tailwind 4 classes — both are accepted per CONTEXT.md D-14. Recommended: CSS Grid with `grid-template-columns: 200px 1fr 280px` on the row container; each column is `overflow-y: auto`.

**Sizing rationale:** Column widths are placeholder defaults for Phase 17; later phases may adjust. Content pane takes all remaining width (`1fr` / `flex: 1`).

**Height:** Shell fills the full viewport height below the header (`height: calc(100vh - 48px)` or flex `flex-grow: 1`).

**Column content in Phase 17:** All three columns render empty. Optional: a muted placeholder label ("Outline", "Content", "Comments") in `var(--color-text-secondary)` at 14px / 400 to confirm the boundary is visible during development. These labels are dev scaffolding — they may be removed by the planner if not needed for the success criterion check.

**Source:** CONTEXT.md D-14, REQUIREMENTS.md LAYOUT-01/LAYOUT-02, ROADMAP.md Phase 17 success criterion 1.
