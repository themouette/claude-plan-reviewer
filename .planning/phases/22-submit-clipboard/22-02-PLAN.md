<objective>
Build the controlled `SubmitPopover` component that the Send Feedback dropdown opens. The popover is pure UI — it owns no submit logic, no clipboard logic, and no connectivity awareness. It surfaces a single overall-message textarea and a Send Feedback action button; on submit it calls `onSubmit(message)` and lets the parent (SubmitControls in Plan 03) decide whether to POST or write to the clipboard.

Purpose: Splitting the popover out of SubmitControls keeps the submit-handler component focused on state transitions and submission paths, while the popover stays a pure UI primitive that is trivial to test via source-contract assertions and (in Plan 04) trivial to visually verify.

Output:
- `ui/src/reviewer-v2/SubmitPopover.tsx` — controlled component implementing the visual spec in UI-SPEC §"SubmitPopover (new)" (lines 165-204) and the dismiss + keyboard contracts in UI-SPEC §"Dismiss Popover" and §"Keyboard Contract" (lines 283-308).
- `ui/src/reviewer-v2/SubmitPopover.test.ts` — Vitest source-contract test matching the established pattern in `ReviewerV2Shell.test.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser keyboard/mouse events → popover state | Untrusted browser-level events drive `onDismiss` and `onSubmit`; both delegate to the parent and the textarea value is plain user text. |
| User-typed message → onSubmit callback | The message string is passed to the parent (Plan 03) which embeds it into a JSON body or clipboard payload. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-03 | Tampering | textarea `value` → `onSubmit(message)` | mitigate | The value is passed verbatim to the parent. The parent (Plan 03) calls `serializeAnnotations(message, '', annotations)` which prefixes every line with `> ` (existing serializer behavior verified in `serializeAnnotations.ts` line 23 + 29 + 35) before embedding in markdown. JSON encoding via `JSON.stringify` (in `buildClipboardPayload` and the `fetch` body) escapes embedded quotes / control characters. No raw HTML is ever rendered from the message. |
| T-22-04 | Denial of Service | Cmd+Enter / Escape keydown handlers | accept | Listeners are scoped to `useEffect` cleanup and removed on unmount or when `open` transitions to `false`. No listener leaks; no unbounded state. |
| T-22-05 | Information Disclosure | Popover textarea contents | accept | The textarea is local component state, never persisted to localStorage, sessionStorage, or any external sink within this plan. Discarded on dismiss. |
| T-22-06 | Elevation of Privilege | Outside-click dismiss listener | accept | Listener delegates to `onDismiss` only (no other side effect); cannot escalate privilege. |
| T-22-SC | Tampering | npm/pip/cargo installs | n/a | No packages installed in this plan. |
</threat_model>
