## Spacing Scale

Source: inherited from Phase 21 UI-SPEC. No new spacing tokens introduced.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Gap between Approve and Send Feedback buttons; caret visual padding |
| sm | 8px | Popover internal vertical rhythm; gap between textarea and button row; gap between offline banner lines |
| compact | 12px | Popover textarea padding. Inherited from Phase 21 annotation bubble textarea padding — project-standard token. |
| md | 16px | Popover horizontal padding; header button horizontal padding; confirmation text padding |
| lg | 24px | n/a this phase |
| xl | 32px | n/a this phase |
| 2xl | 48px | n/a this phase |
| 3xl | 64px | n/a this phase |

Exceptions:
- Header button height: 32px — fits inside the existing 48px header with 8px top/bottom margin; smaller than the 44px submit-bar target since this is a desktop-primary tool
- Touch target minimum: 32px is acceptable for this desktop tool (not a mobile app); if the target device list expands to mobile, bump to 44px
- Popover textarea: `min-height: 80px` — 5 lines × 16px line-height; gives enough space to see a multi-line message without scrolling
- Popover width: `min-width: 320px` — enough for a comfortable textarea; does not exceed the header width

## Typography

Source: `index.css` body rules + existing components. No new type scales introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.6 | Body paragraph text (inherited from `<body>`) |
| Label | 14px | 400 | 1.5 | Popover textarea input text + placeholder; offline banner text |
| Emphasis | 14px | 600 | 1.5 | Confirmation inline heading ("Approved", "Feedback sent", "Copied to clipboard", "Clipboard write failed") — semibold label, not a full heading |
| Caption | 12px | 400 | 1.4 | Popover helper text; clipboard error monospace payload textarea |

Exactly 3 distinct sizes in this phase: 16px (body), 14px (label/emphasis), 12px (caption/helper).

Note: No 20px heading is used in this phase. The confirmation sits inline in the header at 14px/600 — a compact emphasis, not a full-screen heading. If the design later opts for a full-screen confirmation overlay, add 20px at that point.

## Interaction Contract

### Gate Contract (SUBMIT-01)

**Approve button enabled condition:** `annotations.length === 0`
- When 0 comments: Approve is enabled
- When ≥1 comments: Approve is disabled (`opacity: 0.4`, `disabled` attribute, tooltip explains why)

**Send Feedback button:** always enabled — feedback may consist of a general message only, inline comments only, or both.

### Approve Flow (online)

1. User has 0 annotations. Approve button is enabled.
2. User clicks "Approve".
3. `shouldUseClipboard(connectivity)` returns `false` → online path.
4. `POST /api/decide` with `{ behavior: 'allow' }`.
5. Response `ok` or `409` → `submitState` → `confirmed_allow`.
6. Button group replaced with inline confirmation: **"Approved"** (green) + "You can close this tab."
7. Auto-close attempt: `setTimeout(() => { try { window.close() } catch {} }, 500)`.

### Approve Flow (offline)

1. User has 0 annotations. Approve button is enabled.
2. User clicks "Approve".
3. `shouldUseClipboard(connectivity)` returns `true` → offline path.
4. `buildClipboardPayload('allow', '', '', annotations)` called synchronously (transient activation safety).
5. `navigator.clipboard.writeText(json)`:
   - `.then()`: `submitState` → `clipboard_confirmed`
   - `.catch()`: store raw JSON in `clipboardJson` state, `submitState` → `clipboard_error`
6. On `clipboard_confirmed`: replace button group with "Copied to clipboard" (green) + "Paste into your Claude conversation."
7. On `clipboard_error`: replace button group with "Clipboard write failed" (red) + inline read-only textarea containing the raw JSON.

### Send Feedback Flow (online)

1. User clicks "Send Feedback ▼".
2. Popover opens below the button. Textarea receives `autoFocus`.
3. User optionally types an overall message.
4. User clicks "Send Feedback" inside the popover (or presses `Cmd+Enter` / `Ctrl+Enter`).
5. `shouldUseClipboard(connectivity)` returns `false` → online path.
6. `serializeAnnotations(overallMessage, '', annotations)` produces the `message` string. Includes whatever comments exist (0 or more) plus the optional overall message.
7. `POST /api/decide` with `{ behavior: 'deny', message }`.
8. Response `ok` or `409` → `submitState` → `confirmed_deny`.
9. Popover closes. Button group replaced with **"Feedback sent"** (red) + "You can close this tab."

### Send Feedback Flow (offline)

1. Steps 1–4 as above.
2. `shouldUseClipboard(connectivity)` returns `true` → offline path.
3. `buildClipboardPayload('deny', overallMessage, '', annotations)` called synchronously.
4. `navigator.clipboard.writeText(json)`:
   - `.then()`: popover closes, `submitState` → `clipboard_confirmed`
   - `.catch()`: popover closes, store JSON in `clipboardJson`, `submitState` → `clipboard_error`

### Dismiss Popover

1. Popover is open.
2. User clicks outside the popover, OR presses `Escape`.
3. Popover closes. `submitState` stays `idle`. Message typed in textarea is discarded.

### SubmitControls internal state machine

```typescript
type SubmitState =
  | 'idle'                // buttons shown; popover closed
  | 'popover_open'        // Send Feedback popover visible
  | 'confirmed_allow'     // approved — show inline confirmation
  | 'confirmed_deny'      // feedback sent — show inline confirmation
  | 'clipboard_confirmed' // offline clipboard success — show inline confirmation
  | 'clipboard_error'     // offline clipboard failure — show inline error + textarea
```

### Keyboard Contract

| Key | Context | Action |
|-----|---------|--------|
| `Escape` | Popover open | Close popover, return focus to Send Feedback button |
| `Cmd+Enter` / `Ctrl+Enter` | Popover textarea focused | Submit Send Feedback |
| `Tab` | Popover open | Natural focus order within popover (textarea → Send Feedback button) |
| No "Enter to approve" shortcut | n/a | v2 shell has many interactive elements; Approve requires explicit click |

## Accessibility Contract

- Approve button: `type="button"`, `disabled={!canApprove}`, `title` with explanation when disabled
- Send Feedback button: `type="button"`, `aria-haspopup="true"`, `aria-expanded={isPopoverOpen}`
- Popover: `role="dialog"`, `aria-label="Send feedback"` — traps focus while open (Tab cycles within popover only)
- Popover textarea: `aria-label="Overall message (optional)"` (no visible label)
- Offline banner: `role="status"` (matches App.tsx pattern)
- Confirmation inline: `role="status"`, `aria-live="polite"`
- Clipboard error textarea: `readOnly`, `aria-label="JSON payload — copy and paste into Claude"`, `onClick` selects all text
- Focus ring: `outline: 2px solid var(--color-focus); outline-offset: 2px` on `:focus-visible` for ALL interactive elements
