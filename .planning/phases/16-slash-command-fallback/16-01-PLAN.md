```
## Step 4 — Handle the result

The background process exits when the user clicks a button in the review UI,
OR when the local server is killed (e.g. by Claude Code's process cleanup).

**If stdout contains JSON:**

- **On `{"behavior":"allow"}`:**
  Say: "Review complete, no comments." Then proceed with your next step.

- **On `{"behavior":"deny","message":"<feedback>"}`:**
  Say: "Feedback received: <feedback>" Then treat the message as revision
  instructions for the reviewed content and propose how you will address it.

**If no output is received (empty stdout):**

The server was likely stopped before the user submitted. The user may have
already copied their decision to the clipboard using the offline "Copy to
clipboard" button.

Ask the user:

> The review process exited without a result. If you used "Copy to clipboard"
> in the review UI, please paste the JSON here so I can continue.

When the user pastes `{"behavior":"allow"}`:
Proceed as if the server returned allow: say "Review complete, no comments."

When the user pastes `{"behavior":"deny","message":"<feedback>"}`:
Proceed as if the server returned deny: say "Feedback received: <feedback>"
then treat the message as revision instructions.
```
