**Part A: Flash-free inline script in ui/index.html (per D-10)**

1. In `ui/index.html`, add a synchronous `<script>` block inside `<head>`, AFTER the `<title>` tag and BEFORE the closing `</head>` tag. The script MUST NOT have `type="module"`, `async`, or `defer` attributes:

   ```html
   <script>
   (function(){
     var t = localStorage.getItem('plan-reviewer-theme');
     if (!t) {
       t = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
     }
     document.documentElement.setAttribute('data-theme', t);
   })();
   </script>
   ```

   This runs synchronously before the browser paints, preventing flash-of-wrong-theme. It reads `localStorage['plan-reviewer-theme']`; if absent, falls back to OS preference via `matchMedia`. Sets `data-theme` attribute on `<html>`.

**Part C: Theme state and toggle in App.tsx (per D-07, D-08, D-09)**

3. In `ui/src/App.tsx`, inside the `App` component function, add theme state using a lazy initializer that mirrors the inline script logic (per RESEARCH.md Pitfall 3). Place this near the top of `App`, alongside the other `useState` calls:

   ```typescript
   const [theme, setTheme] = useState<'dark' | 'light'>(() => {
     const stored = localStorage.getItem('plan-reviewer-theme')
     if (stored === 'dark' || stored === 'light') return stored
     return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'
   })

   function handleThemeToggle() {
     const next = theme === 'dark' ? 'light' : 'dark'
     setTheme(next)
     document.documentElement.setAttribute('data-theme', next)
     localStorage.setItem('plan-reviewer-theme', next)
   }
   ```

4. Extend the `PageHeader` component props to accept `theme` and `onThemeToggle`:

   Change the `PageHeader` function signature from:
   ```typescript
   function PageHeader({ activeTab, onTabChange }: { activeTab: Tab; onTabChange: (tab: Tab) => void })
   ```

   To:
   ```typescript
   function PageHeader({ activeTab, onTabChange, theme, onThemeToggle }: {
     activeTab: Tab
     onTabChange: (tab: Tab) => void
     theme: 'dark' | 'light'
     onThemeToggle: () => void
   })
   ```

5. Inside `PageHeader`'s `<header>`, AFTER the `<TabBar ... />` component, add the theme toggle button. Because the header uses `justifyContent: 'space-between'`, the TabBar and toggle button need to be grouped together on the right side. Wrap `<TabBar>` and the toggle button in a `<div>` with `display: 'flex', alignItems: 'center', gap: '16px'`:

   ```tsx
   <div style={{ display: 'flex', alignItems: 'center', gap: '16px' }}>
     <TabBar activeTab={activeTab} onTabChange={onTabChange} />
     <button
       aria-label={theme === 'dark' ? 'Switch to light mode' : 'Switch to dark mode'}
       title={theme === 'dark' ? 'Switch to light mode' : 'Switch to dark mode'}
       onClick={onThemeToggle}
       style={{
         width: '32px',
         height: '32px',
         borderRadius: '6px',
         background: 'none',
         border: 'none',
         cursor: 'pointer',
         color: 'var(--color-text-secondary)',
         fontSize: '18px',
         outline: 'none',
         display: 'flex',
         alignItems: 'center',
         justifyContent: 'center',
       }}
       onMouseOver={(e) => { e.currentTarget.style.background = 'rgba(148, 163, 184, 0.15)' }}
       onMouseOut={(e) => { e.currentTarget.style.background = 'none' }}
       onFocus={(e) => { e.currentTarget.style.outline = '2px solid var(--color-focus)'; e.currentTarget.style.outlineOffset = '2px' }}
       onBlur={(e) => { e.currentTarget.style.outline = 'none' }}
     >
       {theme === 'dark' ? '\u2600' : '\u263D'}
     </button>
   </div>
   ```

   The unicode characters are: U+2600 (sun, shown when dark mode is active = "click to switch to light") and U+263D (moon, shown when light mode is active = "click to switch to dark").

**CRITICAL reminders (per RESEARCH.md pitfalls):**

- Theme state MUST live in `App`, not in `PageHeader`, because the flash-free script runs before React mounts and the initializer needs to match.
