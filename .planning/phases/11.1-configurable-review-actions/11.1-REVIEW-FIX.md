## Fixed Issues

### WR-01: TOCTOU race in `find_free_port` — port may be taken before the server binds

**Files modified:** `tests/integration/review_subcommand.rs`
**Commit:** 86a691f
**Applied fix:** Changed `find_free_port()` to return `(u16, std::net::TcpListener)` instead of just `u16`. The listener keeps the port reserved in the OS. Updated all five call sites to destructure the tuple, spawn the child process, then explicitly `drop(_port_guard)` immediately after spawn — releasing the port only after the child is already running and about to bind. This eliminates the window where a parallel test or OS process could claim the port between port discovery and child bind.

### WR-02: `ConfirmationView` shows hardcoded "Plan approved/denied" regardless of custom labels

**Files modified:** `ui/src/App.tsx`
**Commit:** d8b7844
**Applied fix:** Added optional `approveLabel` and `denyLabel` props to `ConfirmationView` (both default to `'Approve'`/`'Deny'` for backward compatibility). The `<h2>` heading now renders `{approved ? \`${approveLabel} — done\` : \`${denyLabel} — done\`}` instead of the hardcoded strings. Updated the call site at line 1082 to pass `approveLabel={approveLabel}` and `denyLabel={denyLabel}` from the parent `App` state (which is already populated from `/api/config`). TypeScript type-check passed with no errors.
