## Component Inventory

### CommitDrawer

An overlay panel that slides over the FileListPane aside from the left. Renders the commit list.

**Dimensions:** 296px wide, full height of the content area (below AppToolbar), `position: absolute` over the aside, `zIndex: 10`.

**Structure:**
```
CommitDrawer (aside, role="navigation", aria-label="Branch commits")
  └─ header (12px semibold, color-text-secondary)
       "COMMITS"
  └─ ul (list of CommitRow items)
       CommitRow × N
  └─ Empty state (when commits.length === 0)
  └─ Loading state (when loading === true, commits.length === 0)
```

**Drawer border:** `borderRight: '1px solid var(--color-border)'` — consistent with aside borders in Phase 25.

### CommitRow

One row per commit in the drawer list.

**Layout (flex column, padding: '8px 16px', gap: 4px):**
- Line 1: `[checkbox] [SHA chip] [message]` — flex row, align-center
- Line 2: `[author] · [date]` — 12px, color-text-secondary, indented to align with message

**SHA chip:** monospace pill — background: `rgba(59, 130, 246, 0.12)`, border: `1px solid var(--color-border)`, borderRadius: 4px, padding: `4px 8px`, fontSize: 12px. Displays `short_sha` (7 chars).

**Commit message:** 14px, weight 400, `overflow: hidden`, `textOverflow: ellipsis`, `whiteSpace: nowrap`, `flex: 1`. Truncation at one line; full message on hover tooltip (`title` attribute).

**Active row** (last-clicked commit, `viewMode === 'commit'`): `borderLeft: '2px solid var(--color-focus)'`, `background: 'var(--color-bg)'`, `color: 'var(--color-text-primary)'`, `fontWeight: 600` — identical pattern to FileListPane active row.

**Unchecked row:** checkbox unchecked = row text opacity unchanged (the checkbox state is orthogonal to click-to-view). No visual dimming — per D-13, the two gestures are independent.

**Hover state:** `background: rgba(255,255,255,0.04)` on dark, `rgba(0,0,0,0.05)` on light — matching FileListPane hover style.

**Click target:** entire row is clickable (sets `activeCommitSha`, switches to `viewMode: 'commit'`). Checkbox click is stopped from propagating (`e.stopPropagation()`).

### AppToolbar (extended)

Adds a "Commits" toggle button to the right controls group (before the Unified/Side-by-side toggle). Consistent with existing button style (height 32px, padding `0 16px`, border `1px solid var(--color-border)`, borderRadius 6px, fontSize 14px).

**Label:** "Commits" when drawer is closed; "Commits" with semibold weight (fontWeight 600) and `color-text-primary` when open — matching the "Collapse" / "Expand All" pattern.

**Active state:** `fontWeight: 600`, `color: var(--color-text-primary)` when `drawerOpen === true`. Inactive: `fontWeight: 400`, `color: var(--color-text-secondary)`.

### DiffPane title (per-commit mode)

When `viewMode === 'commit'`, a title bar appears at the top of the diff pane (between the scroll container top and the first file diff):

```
[short_sha — commit message]          (14px, weight 600, color-text-primary)
[author · date]                       (12px, weight 400, color-text-secondary)
```

Styled as a strip with: `background: var(--color-surface)`, `borderBottom: '1px solid var(--color-border)'`, `padding: '8px 16px'`. No separate badge or toolbar indicator (D-06).

In `viewMode === 'branch'`, this title strip is not rendered — the diff pane starts with the first file header directly.

### Loading state (CommitDrawer)

A centered 24×24px CSS spinner — identical to `DiffPane` loading state: `border: '2px solid var(--color-border)'`, `borderTopColor: 'var(--color-focus)'`, `borderRadius: '50%'`, `animation: 'spin 0.8s linear infinite'`. Displayed inside the drawer body when `loading === true && commits.length === 0`.

## Interaction Contract

### Keyboard Navigation (COMMIT-04)

- Arrow-key handler: `useEffect` on `CodeReviewApp` that adds a `keydown` listener to `window`.
- **Left arrow:** navigate to previous commit (lower index). At first commit: stop (no wrap).
- **Right arrow:** navigate to next commit (higher index). At last commit: stop (no wrap).
- Active only when `viewMode === 'commit'`. In `viewMode === 'branch'`, no commit navigation fires.
- No keyboard conflict with diff pane vertical scroll (up/down arrows) — per D-10.

Source: CONTEXT.md D-09, D-10, Specifics section.

### Checkbox / Multi-select Filtering (DIFF-05)

- Default state: all commits checked (`checkedCommitShas` initialized to all shas — opt-out model per D-08).
- When `viewMode === 'branch'` and `checkedCommitShas` is a proper subset of all commits: fetch and union `FileDiff[]` from each checked commit via `/api/diff/commit/{sha}`. Client-side union — no new backend query param (CONTEXT.md Specifics).
- When all commits are checked: use `/api/diff/branch` directly (no per-commit fetching).
- Unchecking while in `viewMode === 'commit'` has no immediate effect on the diff pane — the per-commit diff continues showing the clicked commit's diff (D-13).

### Mode Switching (COMMIT-03)

- **Click a commit row** → `viewMode: 'commit'`, `activeCommitSha: sha`. DiffPane shows `/api/diff/commit/{sha}`.
- **Close drawer (click "Commits" toggle again)** → `drawerOpen: false`, `viewMode: 'branch'`. DiffPane returns to branch diff (D-07). `activeCommitSha` is cleared to `null`.
- The drawer can also be open while in `viewMode: 'branch'` — e.g., before any commit is clicked.

### Boundary Behavior (arrow keys)

At the first commit (index 0): left arrow does nothing (no wrap).
At the last commit (index N-1): right arrow does nothing (no wrap).

Source: CONTEXT.md Claude's Discretion — "wrap vs stop" — stop chosen to match typical git-log navigation UX (no accidental wrap past boundaries).
