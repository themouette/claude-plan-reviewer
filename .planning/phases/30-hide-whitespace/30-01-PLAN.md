# Plan 30-01: Hide Whitespace Toggle

## Objective

Add a "Hide Whitespace" toggle button to the code review toolbar. When active, whitespace-only
changes are excluded from the diff. Implemented entirely client-side by passing
`{ ignoreWhitespace: true }` to `parseDiffFromFile` — no server changes required.

## Background

`parseDiffFromFile` (from `@pierre/diffs`) accepts a `CreatePatchOptionsNonabortable` third
argument that includes `ignoreWhitespace: boolean` from the underlying `diff` library. All text
files (including new files, deleted files, and uncommitted changes) go through the
`FileDiffComponent` path and are affected by this option. The `PatchDiff` fallback is only
reached for binary files, where whitespace is irrelevant.

## must_haves

### truths
- `hideWhitespace` state lives in `CodeReviewApp` — single source of truth
- `AppToolbar` receives `hideWhitespace: boolean` and `onHideWhitespaceToggle: () => void` props
- `DiffPane` receives `hideWhitespace?: boolean` and passes it to `FileDiffRenderer`
- `FileDiffRenderer` adds `hideWhitespace` to the `useMemo` deps for `fileDiffMetadata`
- `parseDiffFromFile` is called with `{ ignoreWhitespace: hideWhitespace }` as third argument
- The toggle button visual style matches existing toolbar buttons (height 32, same border/radius)
- Active state is visually distinct (fontWeight 600, `var(--color-text-primary)`) matching "Expand All" pattern
