## Component Inventory

All new components live under `ui/src/code-review/`. Must NOT import from `reviewer-v2/`.

### AppToolbar (modified)

File: `ui/src/code-review/AppToolbar.tsx`

New props added to existing interface:

The "Reserved" `<div />` placeholder on the right side is replaced with submit controls inline or as a sub-component. Prop shape (inline vs. sub-component) is Claude's discretion per CONTEXT.md.

### CodeReviewSubmitPopover (new)

File: `ui/src/code-review/CodeReviewSubmitPopover.tsx`

Mirrors `reviewer-v2/SubmitPopover.tsx`. Differences:
- `aria-label="Approve review"` (not "Send feedback")
- Textarea label: "Global instruction (optional)"
- Textarea placeholder: "Leave an instruction for the agent (optional)"
- Submit button label: "Confirm Approve"
- Submit button color: `var(--color-accent-approve)` (green, not red)
- Message is always optional (never required) — no `messageRequired` prop
- Keyboard shortcut: Cmd+Enter submits

Visual shell identical to SubmitPopover:
- `position: absolute; top: 40px; right: 0`
- `minWidth: 320px`
- `zIndex: 50`
- `background: var(--color-surface)`
- `border: 1px solid var(--color-border)`
- `borderRadius: 8px`
- `padding: 16px`
- `boxShadow: 0 8px 24px rgba(0,0,0,0.3)`

### Submit State Machine

SubmitState type (mirrors reviewer-v2/SubmitControls.tsx):
```
'idle'
| 'popover_open'
| 'confirmed_approve'
| 'confirmed_request_changes'
| 'clipboard_confirmed'
| 'clipboard_error'
```

State transitions:
- idle → popover_open: user clicks "Approve" (online path)
- idle → confirmed_approve: user clicks "Approve" in offline mode (clipboard written) or popover confirms
- popover_open → confirmed_approve: user confirms in popover
- popover_open → idle: user dismisses popover (Escape or outside click)
- idle → confirmed_request_changes: user clicks "Request Changes" and POST succeeds
- idle → clipboard_confirmed: offline path, clipboard write succeeds
- clipboard_confirmed → idle: auto-reset after 3000ms
- idle → clipboard_error: clipboard write fails or POST fails
- clipboard_error → idle: user dismisses

## Interaction Contracts

### Gate Logic (D-06)

| Button | Enabled when | Disabled when |
|--------|-------------|---------------|
| Approve | `comments.length === 0` | `comments.length > 0` |
| Request Changes | `comments.length > 0` | `comments.length === 0` |

Disabled state: `opacity: 0.4`, `cursor: default`. Enabled state: `opacity: 1`, `cursor: pointer`.

The `title` attribute on the disabled button carries the reason:
- Approve disabled: `title="Cannot approve while comments exist"`
- Request Changes disabled: `title="Add at least one comment before requesting changes"`

### Button Appearance

Approve button:
- Background: `var(--color-accent-approve)` (green)
- Hover: `var(--color-accent-approve-hover)`
- Height: 32px; padding: 0 16px; borderRadius: 6px; border: none

Request Changes button:
- Background: `var(--color-accent-deny)` (red)
- No hover token defined — use `filter: brightness(0.9)` or add `--color-accent-deny-hover` (Claude's discretion)
- Height: 32px; padding: 0 16px; borderRadius: 6px; border: none

Both buttons: `fontSize: 14px; fontWeight: 600; color: #fff; className="submit-btn"` (existing CSS transition).

### Offline Mode

When `connectivity === 'offline'`:
- Both buttons relabel to "Copy to clipboard" (single shared label for the offline state)
- No popover is shown — clipboard write fires immediately on click
- On success: `clipboard_confirmed` state shows status message (green label + secondary text)
- On failure: `clipboard_error` state shows read-only textarea with JSON payload

### Status Messages (confirmed/error states)

After online submission, auto-close tab after 500ms (matching reviewer-v2 pattern).

| State | Green label | Secondary text |
|-------|-------------|----------------|
| confirmed_approve | "Approved" | "You can close this tab." |
| confirmed_request_changes | "Review submitted" | "You can close this tab." |
| clipboard_confirmed | "Copied to clipboard" | "Paste into your Claude conversation." |
| clipboard_error | "Clipboard write failed" | (read-only textarea with JSON + Dismiss button) |

All status containers use `role="status" aria-live="polite"`.

### Focus Management

- Popover opens with `autoFocus` on the textarea (matching SubmitPopover.tsx)
- All buttons use imperative `onFocus/onBlur` handlers for `outline: 2px solid var(--color-focus); outlineOffset: 2px` (matching existing AppToolbar pattern)
