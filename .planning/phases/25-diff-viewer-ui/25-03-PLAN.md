<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL pathname → React routing | The `window.location.pathname` is read once in main.tsx. The pathname is browser-controlled (user-typed URL) and crosses into the React render decision. |
| ARCH-01 removal → existing integrations | Removing `/api/diff` could break any external consumer that depends on the legacy endpoint. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-RTE | Tampering / Information disclosure | `window.location.pathname.startsWith('/code-review')` route check | accept | The route decision is purely a frontend rendering branch — both CodeReviewApp and ReviewerV2 fetch the same set of API endpoints (their accesses are determined by the React tree, not by the pathname). A user-typed path cannot escalate privileges or access different data. The Rust server's authorization model is pathname-independent (listener bound to 127.0.0.1 only). |
| T-25-MIG | Information disclosure | ARCH-01 removal of /api/diff | accept | The legacy `/api/diff` endpoint was unused: it returned `{"diff": state.diff_content}` populated at startup from `extract_diff()`. The frontend code-review path uses `/api/diff/branch` (added in Phase 24). The plan_reviewer path (ReviewerV2) does NOT use `/api/diff` (verified by `grep -RE "/api/diff[^/]" ui/src/reviewer-v2/` returning no matches). No external consumer is documented. Risk of breakage is bounded by the fact that this is a local single-user tool with no API contract guarantees outside the binary. |
| T-25-HV | Information disclosure | Human verification step exposes the diff content in a browser | accept | The diff content is the user's own working repository — they already have read access. The browser tab is local (127.0.0.1). No new exposure. |
| T-25-RACE | Tampering / Race | Toggling Expand All while a previous refetch is still in flight | mitigate | The `useDiff.refetch` setter sets `loading=true` immediately, which disables the Expand All button (via `disabled={contextLoading}` in AppToolbar's contract). Rapid clicks are debounced by the disabled state — only one fetch can be in flight from this UI path at a time. The hook's `cancelledRef` further guards against state updates after unmount. |
| T-25-SC | Tampering | npm/cargo installs | accept | Zero new packages introduced in this plan. |

T-25-MIG is the only material risk: removing a public-looking endpoint. Mitigated by Phase 25's audit step (verified no current consumer references it) and by the project's local-only / no-published-API constraint.
