## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust JSON API → React frontend | FileDiff[] data crosses here. Includes user-controlled content: filename strings (could be path-traversal-like patterns from the repo) and patch strings (raw diff text with arbitrary user content). |
| React state → DOM rendering | The diff content reaches the DOM via @pierre/diffs' PatchDiff component. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-XSS | Information disclosure / Tampering | filename string used in `aria-label={file.filename}` and `title={file.filename}` | mitigate | React escapes all string children and attribute values by default. The filename is set as an attribute, not via `dangerouslySetInnerHTML`. Path traversal-like filenames (e.g. `../../etc/passwd`) are rendered as plain text — they have no effect on the local server or browser. Verified by react@19 default behavior. |
| T-25-PD | Information disclosure | Diff patch content rendered by PatchDiff | mitigate | The patch content is rendered by `@pierre/diffs` (^1.1.12), which uses Shiki for syntax highlighting and emits sanitized HTML internally. PatchDiff does not accept HTML props — only a `patch: string` (raw text). No untrusted HTML reaches the DOM via raw injection in our code. |
| T-25-BIN | Information disclosure | Binary file sentinel `'[binary file]'` | mitigate | The binary-file guard (Pitfall 5) intercepts this exact sentinel string and renders a static placeholder. PatchDiff never receives the sentinel, avoiding undefined parser behavior. |
| T-25-PT | Tampering / Path traversal | `id={\`file-${index}\`}` and `document.getElementById` lookup | accept | The `index` is `number` from `files.map((file, index) => ...)` — not user input. `document.getElementById(\`file-${index}\`)` only matches our own generated anchor IDs. No path-traversal vector. |
| T-25-WIN | Tampering | `window.matchMedia` SSR / non-browser environment | mitigate | The `typeof window !== 'undefined' &&` guard in the DIFF_THEME constant prevents a crash if the module is ever loaded in a non-browser context (Vite SSR, tests). Asserted by `npm run build` and `npm test` both passing. |
| T-25-SC | Tampering | npm install of new packages | accept | Zero new packages introduced. `@pierre/diffs` already in package.json at ^1.1.12. No Package Legitimacy Gate triggered. |

T-25-XSS and T-25-PD are mitigated by the framework (React + @pierre/diffs internal sanitization) and project conventions (no `dangerouslySetInnerHTML`). T-25-BIN is mitigated by the explicit binary-file guard required by Pitfall 5.
