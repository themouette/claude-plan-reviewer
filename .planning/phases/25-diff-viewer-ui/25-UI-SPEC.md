## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none ŌĆö plain CSS custom properties |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot margins, badge padding |
| sm | 8px | Compact element spacing, list item vertical padding |
| md | 16px | Column internal padding, toolbar horizontal padding |
| lg | 24px | Section breaks within diff pane |
| xl | 32px | Not used in Phase 25 |
| 2xl | 48px | Toolbar height (matches ReviewerV2Shell header strip) |
| 3xl | 64px | Not used in Phase 25 |

Exceptions:
- File list item minimum height: 28px (matches OutlinePane button min-height)
- Toggle button height: 32px (matches SubmitControls button height)
- Toolbar height: exactly 48px (`height: 48`, `flexShrink: 0`) ŌĆö must match ReviewerV2Shell header strip precisely so the two routes feel consistent

**Source:** `ReviewerV2Shell.tsx` (header 48px), `OutlinePane.tsx` (min-height 28), `SubmitControls.tsx` (button height 32).

Note: ReviewerV2 uses 14px for all sidebar/toolbar text ŌĆö maintained here for visual consistency. Diff hunk text is rendered by `@pierre/diffs` and uses the library's internal typography.

## File List Visual Design

### File Entry

Structure per entry:
```
[status-dot] [basename]           [+N -M]
```

- Minimum height: 28px
- Padding: `8px 16px`
- Font: 14px, weight 400
- Color: `var(--color-text-secondary)` (inactive), `var(--color-text-primary)` (active)
- Background: `transparent` (inactive), `var(--color-bg)` (active ŌĆö same as OutlinePane active uses `--color-surface`)
- Left border: `2px solid transparent` (inactive), `2px solid var(--color-focus)` (active)
- Hover: `background: rgba(255, 255, 255, 0.04)` (dark), `rgba(0, 0, 0, 0.05)` (light) ŌĆö class `.file-list-btn` following `.outline-button` pattern

### Status Dot

- Size: 8px ├Ś 8px circle (`border-radius: 50%`)
- Margin-right: 8px
- Colors:
  - Added: `#22c55e` (dark) / `#16a34a` (light) ŌĆö reuses `--color-accent-approve`
  - Deleted: `#ef4444` (dark) / `#dc2626` (light) ŌĆö reuses `--color-accent-deny`
  - Modified: `#3b82f6` (dark) / `#2563eb` (light) ŌĆö reuses `--color-focus`
  - Renamed: `#94a3b8` (dark) / `#64748b` (light) ŌĆö reuses `--color-text-secondary`

### Rename Icon

- Renamed files show basename of **new** filename + inline `Ōå│` character before the basename
- Character: `Ōå│` (U+21B3), `font-size: 12px; color: var(--color-text-secondary); margin-right: 4px`
- Tooltip (`title` attribute): `"Renamed from {old_path} to {new_path}"`

### Change Counts

- Rendered right-aligned via `margin-left: auto`
- Format: `+{additions} -{deletions}` where additions > 0 shows green, deletions > 0 shows red
- Added count color: `#22c55e` / `#16a34a`
- Deleted count color: `#ef4444` / `#dc2626`
- Font: 12px, weight 400
- If both additions and deletions are 0 (binary or pure rename): omit the count entirely

### Hover Tooltip

- Native `title` attribute on each file list button
- Content: full relative path from repo root (e.g. `ui/src/reviewer-v2/ReviewerV2Shell.tsx`)
- For renamed files: `"ui/src/old/path.tsx ŌåÆ ui/src/new/path.tsx"`

**Source:** `25-CONTEXT.md` D-07, D-08, D-09; OutlinePane.tsx active/hover pattern.

## Diff Pane States

### Loading State

- Displayed while `useDiff` fetch is in-flight
- Visual: a centered spinner using the existing `@keyframes spin` animation from `index.css`
- Spinner size: 24px ├Ś 24px circle, `border: 2px solid var(--color-border); border-top-color: var(--color-focus); border-radius: 50%; animation: spin 0.8s linear infinite`
- Container: `display: flex; align-items: center; justify-content: center; flex: 1; min-height: 200px`
- No label text ŌĆö spinner alone is sufficient for a local tool with sub-second responses

### Empty State

When the diff API returns an empty file list (no changes on the branch):
- Heading: "No changes on this branch"
- Body: "This branch has no uncommitted or unpushed changes compared to main."
- Container: centered, `padding: 48px 32px; text-align: center`
- Heading: `font-size: 16px; font-weight: 600; color: var(--color-text-primary); margin-bottom: 8px`
- Body: `font-size: 14px; font-weight: 400; color: var(--color-text-secondary)`

### Error State

When the diff API returns a non-2xx response or network error:
- Heading: "Could not load diff"
- Body: "The server returned an error. Check that the repository has a main branch and try reloading."
- Container: same as empty state
- Heading color: `var(--color-accent-deny)` (#ef4444 / #dc2626)
- Body color: `var(--color-text-secondary)`
- Retry button: `<button>Reload Diff</button>` ŌĆö 32px height, `border: 1px solid var(--color-border); background: var(--color-surface); color: var(--color-text-secondary); padding: 0 16px; border-radius: 6px; font-size: 14px; margin-top: 16px`

### Diff Content

One `PatchDiff` component per file entry, stacked vertically with no gap.

Each file section has an anchor element immediately before the `PatchDiff`:
```tsx
<div id={`file-${index}`} aria-label={file.filename} />
```

File section header (rendered by `@pierre/diffs` `renderCustomHeader` prop or a wrapper):
- Background: `var(--color-surface)`
- Border-bottom: `1px solid var(--color-border)`
- Padding: `8px 16px`
- Font: 14px, weight 600, `color: var(--color-text-primary)`
- Shows: basename (bold) + full path (secondary color, 12px)

## Interaction Contracts

### File List ŌåÆ Diff Pane Navigation (D-09)

- Click on file entry: calls `document.getElementById('file-{index}')?.scrollIntoView({ behavior: 'smooth', block: 'start' })`
- Active file: determined by which file's diff section is closest to the top of the diff pane viewport (IntersectionObserver with `rootMargin: '-10px 0px -85% 0px'` ŌĆö same observer pattern as OutlinePane)
- Active file is highlighted with left border `2px solid var(--color-focus)` and `background: var(--color-bg)`

### Per-Hunk Context Expansion (D-04)

- `expansionLineCount: 20` ŌĆö 20 lines revealed per click on the `...` separator (library-native via `@pierre/diffs`)
- The `...` separator is rendered by `@pierre/diffs` natively ŌĆö no custom implementation
- No custom `collapsedContextThreshold` override ŌĆö use library default

### Expand All Toggle (D-05)

- Default state: fetch with no `?context` param (git2 default = 3 context lines)
- Expanded state: re-fetch with `?context=999` ŌĆö replaces the entire diff in state
- Toggle is a 2-state button in AppToolbar
- While re-fetch is in-progress: button shows "Loading..." and is disabled; spinner NOT shown in diff pane (keep current diff visible)
- On error during re-fetch: button returns to previous state, diff pane remains unchanged

### Unified / Side-by-Side Toggle (D-13)

- State lives in `CodeReviewApp` component
- Passed as `options={{ diffStyle: 'unified' | 'split' }}` to every `PatchDiff` instance
- Toggle is instant (no fetch) ŌĆö all PatchDiff components re-render synchronously
- Default: `'unified'`

### Syntax Highlighting Theme (D-12)

- Read once at component initialization (not on mount ŌĆö before first render)
- `const isDark = window.matchMedia('(prefers-color-scheme: dark)').matches`
- Passed as `options={{ theme: isDark ? 'github-dark' : 'github-light' }}` to each `PatchDiff`
- No runtime listener ŌĆö requires page reload on OS theme change (deferred per CONTEXT.md)

### Worker Pool (D-11)

- `disableWorkerPool={true}` on every `PatchDiff` ŌĆö no `WorkerPoolContext` needed

### Focus Ring

All interactive elements (file list buttons, toolbar buttons) use:
- `outline: 2px solid var(--color-focus)` on `:focus-visible`
- `outline-offset: 2px`
- Implemented via `onFocus`/`onBlur` handlers (matching SubmitControls pattern) or CSS class
