    Create `OutlinePane.tsx` with these concrete implementation details:

    **Props** (inline typed object, per PATTERNS.md):
    ```
    sections: Section[]
    activeId: string | null
    mainRef: React.RefObject<HTMLElement | null>
    onActiveIdChange: (id: string) => void
    ```

    **Refs**: `activeItemRef = useRef<HTMLLIElement>(null)`

    **IntersectionObserver useEffect** (deps: `[sections, mainRef, onActiveIdChange]`):
    - Guard: `if (!mainRef.current || sections.length === 0) return`
    - Create observer with `root: mainRef.current`, `rootMargin: '-10px 0px -85% 0px'`, `threshold: 0`
    - Callback: iterate entries; for each `entry.isIntersecting === true`, call `onActiveIdChange(entry.target.id)` and `break` (first intersecting wins)
    - Observe: `sections.forEach(({ id }) => { const el = document.getElementById(id); if (el) observer.observe(el) })`
    - Cleanup: `return () => observer.disconnect()`

    **Outline auto-scroll useEffect** (dep: `[activeId]`):
    - `activeItemRef.current?.scrollIntoView({ behavior: 'smooth', block: 'nearest' })`

    **JSX structure** (from UI-SPEC Component Inventory):
    ```
    <nav aria-label="Document outline">
      <ol style={{ listStyle: 'none', margin: 0, padding: 0 }}>
        {sections.map((section) => (
          <li
            key={section.id}
            ref={section.id === activeId ? activeItemRef : undefined}
          >
            <button
              aria-label={section.text}
              aria-current={section.id === activeId ? 'true' : undefined}
              onClick={() => document.getElementById(section.id)?.scrollIntoView({ behavior: 'smooth', block: 'start' })}
              onFocus={(e) => { e.currentTarget.style.outline = '2px solid var(--color-focus)'; e.currentTarget.style.outlineOffset = '2px' }}
              onBlur={(e) => { e.currentTarget.style.outline = 'none' }}
              style={{
                width: '100%',
                textAlign: 'left',
                background: section.id === activeId ? 'var(--color-surface)' : 'transparent',
                color: section.id === activeId ? 'var(--color-focus)' : 'var(--color-text-secondary)',
                borderLeft: section.id === activeId ? '2px solid var(--color-focus)' : '2px solid transparent',
                paddingLeft: 16 + (section.depth - 1) * 8,
                paddingTop: 6,
                paddingBottom: 6,
                fontSize: 14,
                fontWeight: section.id === activeId ? 600 : 400,
                lineHeight: 1.4,
                cursor: 'pointer',
                border: 'none',  // reset button border; borderLeft is set above separately
                display: 'block',
                minHeight: 28,
                outline: 'none',
              }}
            >
              {section.text}
            </button>
          </li>
        ))}
      </ol>
    </nav>
    ```

    NOTE on border: Buttons have `border: 'none'` to reset browser defaults but `borderLeft` needs to be set separately. In React inline styles, `borderLeft` takes precedence over `border` when both keys exist in the same object only if they follow `border` in object literal order — to be safe, merge as: first set `border: 'none'`, then `borderLeft: section.id === activeId ? '2px solid var(--color-focus)' : '2px solid transparent'`. The `2px solid transparent` on inactive items ensures layout does not shift when an item becomes active.

    NOTE on hover: Add `onMouseEnter`/`onMouseLeave` handlers to toggle background to `rgba(255,255,255,0.04)` (dark hover) when not active, per UI-SPEC.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-05 | Spoofing | OutlinePane `<button>` | accept | Outline items render heading text from the plan document; no user-writable input is involved; this is reviewer-read-only UI |
| T-19-06 | Tampering | IntersectionObserver root | mitigate | `if (!mainRef.current || sections.length === 0) return` guard prevents observer creation with invalid root; `document.getElementById(id)` returns null for non-existent ids and is guarded with optional chaining |
| T-19-07 | Denial of Service | document.getElementById in click handler | accept | The ids are slugified strings derived from heading text; `getElementById` on a missing id returns null; optional chaining `?.scrollIntoView` handles null safely; no loop or recursive call is possible |
| T-19-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
