| Property | Value |

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Badge padding, tree connector dots |

| md | 16px | Left panel `padding`, first-level indent baseline |

| xl | 32px | N/A for this pane |
| 2xl | 48px | N/A for this pane |

Exceptions:
- Left panel width: 200px (fixed, inherited from ReviewerV2Shell.tsx `aside` width — do not change)
- Indent per depth level: 8px × (depth - 1). H1 → 0px additional indent. H2 → 8px. H3 → 16px. H4 → 24px. H5 → 32px. H6 → 40px.
- Active item indicator: 2px left border (no margin change, border replaces padding).
- Touch target minimum: 28px height per item (consistent with SelectionToolbar pill height).

## Component Inventory

### OutlinePane

**File:** `ui/src/reviewer-v2/OutlinePane.tsx`

**Props:**
```ts
interface OutlinePaneProps {
  sections: Section[]      // from ContentPane via Shell
  activeId: string | null  // currently active heading id
}
```

**Structure:**
```
<aside> (owned by ReviewerV2Shell, 200px wide, padding 16px)
  <nav aria-label="Document outline">
    <ol> (no list-style; margin 0; padding 0)
      <li> × N (one per Section)
        <button> (full-width, text-left, depth-indented)
          {section.text}
        </button>
      </li>
    </ol>
  </nav>
```

**Visual states per item:**

| State | Background | Text color | Left border |
|-------|-----------|-----------|-------------|
| Default | transparent | `var(--color-text-secondary)` | none |
| Hover | `rgba(255,255,255,0.04)` dark / `rgba(0,0,0,0.03)` light | `var(--color-text-primary)` | none |
| Active | `var(--color-surface)` | `var(--color-focus)` | `2px solid var(--color-focus)` |
| Active + hover | `var(--color-surface)` | `var(--color-focus)` | `2px solid var(--color-focus)` |
| Focus (keyboard) | same as hover | same as hover | `2px solid var(--color-focus)` (outline) |

**Interaction — click:**
- Call `document.getElementById(section.id)?.scrollIntoView({ behavior: 'smooth', block: 'start' })`
- No URL change, no hash navigation (OUTLINE-02 requirement)

**Interaction — active tracking:**
- `activeItemRef` attached to the currently active `<li>`
- On `activeId` change: `activeItemRef.current?.scrollIntoView({ behavior: 'smooth', block: 'nearest' })`

**Empty state:** When `sections.length === 0`, render nothing (empty `<nav>`). Do not show a placeholder message — the outline panel is still usable as a navigation affordance and showing an empty state would be noise while the document is loading.

**Loading state:** The outline panel renders whatever `sections` it receives. If `sections` is empty (ContentPane still loading), no items appear. No spinner needed — the center pane already shows a loading state.

## Interaction Contract

### Click-to-Scroll (OUTLINE-02)

```
User clicks outline item
  → scrollIntoView({ behavior: 'smooth', block: 'start' })
  → Target element: document.getElementById(section.id) inside <main>
  → No browser URL change
  → No history push
```

### Active Section Tracking (OUTLINE-03)

```
IntersectionObserver watches all heading elements
  root: mainRef.current (the <main> scroll container)
  rootMargin: '-10px 0px -85% 0px'  ← narrow top band
  threshold: 0

On entry: update activeId to section.id of the intersecting heading
On exit: no change (last entered heading stays active)

activeId change:
  → activeItemRef.current?.scrollIntoView({ behavior: 'smooth', block: 'nearest' })
```

The `rootMargin` values (`-10px 0px -85% 0px`) are a starting calibration point. The exact values are left to the implementor's discretion (CONTEXT.md "Claude's Discretion") to calibrate during implementation.

### Data Flow (ContentPane → Shell → OutlinePane)

```
ContentPane:
  useEffect([planHtml]) {
    planRef.current.querySelectorAll('h1,h2,h3,h4,h5,h6')
    → build Section[] using id attribute from markdownRenderer heading renderer
    → call onSectionsFound(sections)
  }

ReviewerV2Shell:
  [sections, setSections] = useState<Section[]>([])
  [activeId, setActiveId] = useState<string | null>(null)
  [mainRef] = useRef<HTMLElement>(null)
  → passes onSectionsFound={setSections} to ContentPane
  → passes sections + activeId to OutlinePane
  → passes mainRef to ContentPane (for IntersectionObserver root)

OutlinePane:
  receives sections + activeId as props
  manages activeItemRef internally
```

### Heading ID Assignment

Implemented in `markdownRenderer.ts` custom heading renderer:
- Slug: lowercase, spaces → hyphens, strip non-alphanumeric (except hyphens)
- Duplicates: append `-2`, `-3`, etc. (module-level counter per render call)
- Example: "My Heading" → `my-heading`, second "My Heading" → `my-heading-2`

The exact slug algorithm is at implementor's discretion (CONTEXT.md "Claude's Discretion"). The above is a concrete recommendation consistent with GitHub-flavored heading anchors.

## Accessibility Contract

- Outline is a `<nav>` with `aria-label="Document outline"` — landmark navigation for screen readers.
- Each item is a `<button>` (not `<a>`) — no URL change occurs on click (OUTLINE-02). Using `<a href="#id">` would push browser history and is explicitly forbidden.
- Active item has `aria-current="true"` on its `<button>`.
- Keyboard: Tab moves between items; Enter/Space triggers click handler.
- Focus ring: `outline: 2px solid var(--color-focus)` with `outlineOffset: 2px` on `:focus-visible` — consistent with GutterIcon and SelectionToolbar focus patterns.
- No mousedown preventDefault needed on outline items (unlike GutterIcon/SelectionToolbar which must preserve text selection). Outline clicks do not interact with text selection.

## Implementation Notes for Executor

1. **No DOM mutation inside `dangerouslySetInnerHTML` children** — OutlinePane is a separate `<aside>` column. It does NOT touch the `planRef` DOM tree. (CONTEXT.md anti-pattern)

2. **No `@testing-library/react`** — Test heading slug logic and IntersectionObserver callback as exported pure functions. The `useOutline` hook (if extracted) must be testable via DI, consistent with `useAnnotations.test.ts` pattern.

3. **IntersectionObserver mock is already registered** in `vitest.setup.ts` (Phase 17). Any new hook using IntersectionObserver works within this mock.

4. **`mainRef` must be a `RefObject<HTMLElement>` not `RefObject<HTMLDivElement>`** — the `<main>` element in Shell is a semantic element, type accordingly.

5. **Slug counter must reset per `renderMarkdown` call** — do not use a module-level incrementing counter that persists across calls. Use a `Map<string, number>` local to each `renderMarkdown` invocation.

6. **`block: 'nearest'` for outline auto-scroll** — prevents jarring scroll when the active item is already visible in the outline panel. `block: 'start'` would be wrong here (CONTEXT.md D-05).

7. **Tailwind v4 is available** — `@import "tailwindcss"` is already in index.css. The project uses inline styles for component-specific values and Tailwind utility classes for layout (see `ReviewerV2Shell.tsx` `className="flex flex-col h-screen overflow-hidden"`). Either approach is acceptable for OutlinePane; inline styles are preferred for dynamic values (depth indent, active state).
