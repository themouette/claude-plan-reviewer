<objective>
Add the `Section` type to `types.ts` and extend `ContentPane.tsx` with an optional `onSectionsFound` prop. After `planHtml` loads and React commits the DOM, a `useEffect` walks `planRef.current.querySelectorAll('h1,h2,h3,h4,h5,h6')` to build a `Section[]` and calls `onSectionsFound`. This is the data-flow bridge that feeds the heading tree from ContentPane up to ReviewerV2Shell (Plan 03).

Purpose: Establishes the `Section[]` contract and the ContentPane → Shell data channel that OutlinePane depends on. Without this, Plan 03 has no heading data to render.
Output: `Section` type in `types.ts`; extended `ContentPane.tsx` with heading walk; updated `ContentPane.test.ts` with source-read assertions for the new wiring.
</objective>
