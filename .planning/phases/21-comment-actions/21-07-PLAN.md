<objective>
Fix two UAT failures from 21-HUMAN-UAT.md:

**Failure 4 — Gutter-icon action tray:** Gutter icon click selects text correctly but the SelectionToolbar does not appear. Root cause: `handleAdd` calls `document.createRange().selectNodeContents(paragraphElement)`, which sets boundary points to the ELEMENT node, not its text children. `getRangeOffsets` in `useTextSelection` only walks TEXT nodes (NodeFilter.SHOW_TEXT), so it never matches the element-level boundary containers and returns null, leaving `storedOffsets.current = null`. When ContentPane renders, `offsets = getOffsets()` is null → SelectionToolbar condition `selectedText && offsets && !formState` fails on `offsets` → toolbar never renders.

**Failure 5 — Auto-submit on conflict (D-03):** Opening a second annotation form does not auto-submit the first. Root cause: While a form is open, `PlanContent.handleMouseMove` short-circuits on `selectedText` so `hoveredParagraph` stops tracking, the GutterIcon unmounts, and there is no UI path to trigger `handleAction` again. The D-03 auto-submit code in `handleAction` is never reached.

Fix requires four coordinated changes:
1. `handleAdd` in ContentPane: use TreeWalker to build text-node-anchored range
2. `GutterIcon`: add `data-gutter-icon` attribute so AnnotationForm can exempt it from click-outside cancel
3. `PlanContent`: accept `formOpen` prop; unblock hover tracking and GutterIcon visibility when `formOpen` is true
4. `AnnotationForm`: skip cancel when mousedown target is inside `[data-gutter-icon]`
5. `handleAdd` in ContentPane: auto-submit existing form before selecting new paragraph

Output:
- `ContentPane.tsx`: fixed handleAdd + formOpen wiring + auto-submit in handleAdd
- `GutterIcon.tsx`: data-gutter-icon attribute
- `PlanContent.tsx`: formOpen prop + updated hover/gutter conditions
- `AnnotationForm.tsx`: skip-cancel guard for gutter icon
- Test files updated for each change
</objective>
