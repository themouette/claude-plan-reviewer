## Typography

Source: `index.css` `.plan-prose` rules + existing SelectionToolbar, CommentBubble, OutlinePane components. No new type scales introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.6 | Plan prose (inherited from `<body>`) |
| Label | 14px | 400 | 1.5 | Textarea placeholder + input text; "Post Comment" / "Dismiss" / "Save Changes" / "Discard Changes" button labels; predefined action menu items; edit mode textarea |
| Caption | 13px | 400 | 1.4 | Annotation type tag in expanded bubble (carried from Phase 20); icon button aria-label fallback |
| Heading | 14px | 600 | 1.4 | Bubble anchor-text preview header (carried from Phase 20); pill labels in SelectionToolbar |
| Badge | 11px | 600 | 1 | Section count badge number in OutlinePane |

Exactly 4 distinct sizes in this phase: 16px (inherited prose), 14px (label + pill), 13px (caption), 11px (badge).
Exactly 2 weights: 400 (regular) + 600 (semibold — pill labels, bubble anchor-text header, badge numbers).

Note: SelectionToolbar pills already use `font-size: 13px, font-weight: 600` — the annotation creation form's "Post Comment" button follows the same pill styling (13px/600 for consistency), while the inline "Dismiss" link uses 13px/400.

Note: 13px (Caption) and 14px (Label) are intentionally 1px apart. Caption is used for secondary metadata (type tags); Label is used for all interactive control text. Distinction is deliberate — they serve different hierarchy levels.

## Component Inventory

Components modified or introduced in Phase 21:

### Annotation Creation Form (new — inline with SelectionToolbar)

**Location:** `ContentPane.tsx` — rendered instead of `SelectionToolbar` when `formState !== null`

**Trigger:** Any pill click or quick-action menu item click causes `handleAction` to set `formState`. The SelectionToolbar disappears and the form appears at the same `position: fixed` coordinates (anchored to `lastRect.bottom + 6` of the selection).

**Focal point:** The `<textarea>` is the primary visual anchor. It receives `autoFocus` on mount, draws the eye as the largest interactive element, and has the widest contrast boundary (`var(--color-border)` outline against `var(--color-surface)` background). All other form elements (buttons, type label) are secondary.

**Visual spec:**
- Container: `position: fixed`, same `top`/`left` calculation as SelectionToolbar (reuse `lastRect`)
- `background: var(--color-surface)`, `border: 1px solid var(--color-border)`, `border-radius: 6px`, `padding: 8px`, `box-shadow: 0 2px 8px rgba(0,0,0,0.3)`, `z-index: 20`
- Width: 280px (matches `TOOLBAR_WIDTH` constant — reuse it)
- Textarea: `width: 100%`, `min-height: 64px`, `font-size: 14px`, `font-family: inherit`, `background: transparent`, `color: var(--color-text-primary)`, `border: 1px solid var(--color-border)`, `border-radius: 4px`, `padding: 8px`, `resize: vertical`
- Textarea `placeholder`: "Add a comment…" for Comment type; for Delete/Replace/predefined actions, textarea is pre-filled (placeholder not shown)
- Button row: `display: flex`, `justify-content: flex-end`, `gap: 8px`, `margin-top: 8px`
- "Post Comment" button: `font-size: 13px`, `font-weight: 600`, `height: 28px`, `padding: 0 12px`, `border-radius: 4px`, `background: var(--color-focus)`, `color: #fff` (or `var(--color-bg)`), `border: none`, `cursor: pointer`, `min-width: 44px`
- "Dismiss" link-button: `font-size: 13px`, `font-weight: 400`, `height: 28px`, `padding: 0 8px`, `border-radius: 4px`, `background: transparent`, `color: var(--color-text-secondary)`, `border: none`, `cursor: pointer`

**Focus behavior:** Textarea receives `autoFocus` on mount so the user can start typing immediately.

**Keyboard contract:**
- `Cmd+Enter` (macOS) / `Ctrl+Enter` (Windows/Linux): submits the form
- `Escape`: cancels (calls `setFormState(null)`, restores `selection-lock` Highlight clearing)
- Click outside the form container: cancels (same as Escape)

**Auto-submit on conflict (D-03):** If `formState !== null` when a new pill click fires, the current form's textarea value is submitted via `onAddAnnotation` before the new `formState` is set. No visual flash — the swap is instantaneous.

### CommentBubble — edit/delete controls (extended)

**File:** `ui/src/reviewer-v2/CommentBubble.tsx`

**New props:**
```ts
isEditing: boolean
onEdit: () => void
onRemove: () => void
```

**Edit + delete icons — shown when `isFocused === true`:**
- Rendered as a `<div style={{ display: 'flex', justifyContent: 'flex-end', gap: 4, marginTop: 8 }}>` at the bottom of the expanded bubble
- Pencil icon button: `aria-label="Edit comment"`, Unicode character `✎` (U+270E) or `✏` (U+270F), `font-size: 14px`, `color: var(--color-text-secondary)`, hover: `color: var(--color-text-primary)`, size 20×20px
- Delete icon button: `aria-label="Delete comment"`, Unicode character `×` (U+00D7), `font-size: 16px`, `color: var(--color-text-secondary)`, hover: `color: var(--color-accent-deny)`, size 20×20px
- Both buttons: `background: transparent`, `border: none`, `border-radius: 3px`, `cursor: pointer`, `padding: 0`, `display: flex`, `align-items: center`, `justify-content: center`
- Focus ring on `:focus-visible`: `outline: 2px solid var(--color-focus)`, `outline-offset: 2px`

**Edit mode visual spec (when `isEditing === true`):**
- The `<p>` comment body is replaced by a `<textarea>` pre-filled with `annotation.comment`
- Same textarea styling as the Annotation Creation Form (14px, 64px min-height, `var(--color-border)` border, `padding: 8px`)
- Focal point: The `<textarea>` is the primary visual anchor in edit mode — it receives `autoFocus` on mount and is the largest interactive element within the bubble
- Same button row below: "Save Changes" button (same style as "Post Comment": `var(--color-focus)` background) + "Discard Changes" link-button (same style as "Dismiss": `var(--color-text-secondary)` color, transparent background)
- `autoFocus` on textarea mount
- `Cmd+Enter` / `Ctrl+Enter`: saves (calls `onEdit` with new text → `editAnnotation(id, newText)` in Shell)
- `Escape` or click outside: cancels (reverts textarea, calls `onEdit` equivalent to cancel)
- Bubble remains `position: sticky` while editing (D-12) — CommentPane switches `position` from `absolute` to `sticky` when `editingId === annotation.id`

**Pinning while editing (D-12):**
- CommentPane applies `position: sticky; top: 16px` to the editing bubble's wrapper (instead of `position: absolute; top: {computedTop}px`)
- When editing ends, reverts to `position: absolute`

## Interaction Contract

### Annotation Creation Flow (COMMENT-04)

**Pill click → form open:**
1. User selects text or clicks gutter `+` icon → `SelectionToolbar` appears
2. User clicks "Comment", "Delete", "Replace", or a quick-action menu item
3. `handleAction(type, anchorText, prefillComment?)` is called in `ContentPane`
4. If `formState !== null` (pending form exists): auto-submit current form first, then proceed
5. `setFormState({ type, anchorText, anchorStart, anchorEnd, prefill: prefillComment ?? defaultPrefill(type) })` — `SelectionToolbar` unmounts, `AnnotationForm` mounts at same fixed coordinates
6. `selection-lock` CSS Highlight stays active (D-04) — `resetTextSelection()` is NOT called until form submit or cancel

**Pre-fill values by action type:**
| Action | `prefill` value |
|--------|----------------|
| Comment pill | `""` (empty) |
| Delete pill | `"Delete"` |
| Replace pill | `"Replace"` |
| Clarify this | `"clarify this"` |
| Needs test | `"needs test"` |
| Give me an example | `"give me an example"` |
| Out of scope | `"out of scope"` |
| Search the web | `"search internet"` |
| Search codebase | `"search codebase"` |

**Form submit:**
1. `onAddAnnotation({ id: crypto.randomUUID(), type, anchorText, comment: textareaValue, anchorStart, anchorEnd })` is called
2. `setFormState(null)` — form unmounts
3. `resetTextSelection()` — `selection-lock` Highlight cleared

**Form cancel (Escape / click outside / "Dismiss" button):**
1. `setFormState(null)` — form unmounts, no annotation created
2. `resetTextSelection()` — `selection-lock` Highlight cleared

### Gutter Icon → Full Paragraph Selection (D-06)

1. User hovers paragraph → `GutterIcon` appears
2. User clicks `+`
3. `handleAdd(paragraphElement)` is called in `ContentPane` (signature extended to accept the paragraph element)
4. `ContentPane` calls `window.getSelection()`, creates a `Range` spanning the paragraph's full text via `Range.setStart` / `Range.setEnd` on the paragraph's first/last text nodes
5. `window.getSelection()?.addRange(range)` — dispatches a `selectionchange` event
6. `useTextSelection` picks up the new selection → `selectedText` is set → `SelectionToolbar` appears, exposing all 3 pills and predefined actions
7. From this point the user follows the same pill-click flow as manual text selection

### Edit Flow (COMMENT-05, D-10)

1. User clicks a CommentBubble → bubble becomes focused (`focusedCommentId` set in Shell)
2. Pencil icon appears in expanded bubble
3. User clicks pencil → Shell sets `editingId = annotation.id`; `CommentPane` receives `editingId`
4. CommentBubble with matching `annotation.id` receives `isEditing: true` → displays textarea instead of `<p>`
5. User edits text; submits via "Save Changes" button or `Cmd+Enter`
6. `onEdit(newComment)` prop is called → Shell calls `editAnnotation(annotation.id, newComment)` from `useAnnotations`
7. Shell sets `editingId = null` → bubble returns to expanded display mode

### Delete Flow (COMMENT-05, D-09)

1. User focuses a bubble (clicks it)
2. × icon appears in expanded bubble
3. User clicks × → `onRemove()` prop is called → Shell calls `removeAnnotation(annotation.id)`
4. Annotation is removed immediately from store; bubble unmounts; CommentPane recomputes layout
5. No confirmation dialog

### Section Count Update (OUTLINE-04, D-13)

1. `ReviewerV2Shell` calls `useSectionAnnotationCounts(sections, annotations, planRef)` on every render where `sections` or `annotations` changes
2. Hook returns `Map<sectionId, number>` — passed as `annotationCounts` prop to `OutlinePane`
3. OutlinePane renders badges inline with each section title (count > 0 only)
4. Adding a comment in ContentPane triggers a re-render of Shell → `useSectionAnnotationCounts` recomputes → badge updates immediately

## Accessibility Contract

- Annotation creation form: `role="dialog"` not required (it is a popover, not a modal — content behind it remains interactive); `aria-label="Add annotation"` on the container `<div>`
- Textarea: `aria-label="Comment text"` (no visible `<label>` in compact form); `aria-required="false"` (empty submit permitted for non-empty pre-fill types)
- Submit button: `type="submit"` within a `<form onSubmit={...}>` or `type="button"` with explicit onClick — either is acceptable; Cmd+Enter must work regardless
- "Dismiss" button: `type="button"`, `aria-label="Dismiss annotation form"` if icon-only (text label "Dismiss" is present, so aria-label not strictly needed)
- Pencil icon button: `type="button"`, `aria-label="Edit comment"`, `aria-pressed` not required (it toggles mode, not a toggle state)
- Delete icon button: `type="button"`, `aria-label="Delete comment"`
- Section badge `<span>`: `aria-label="{count} comments"` so screen readers announce count in context of the outline item
- Focus ring: `outline: 2px solid var(--color-focus); outline-offset: 2px` on `:focus-visible` for ALL new interactive elements — same pattern as `OutlinePane .outline-button:focus-visible`
- Keyboard trap: The annotation creation form does NOT trap focus (it is not a modal); Tab navigates freely beyond the form
