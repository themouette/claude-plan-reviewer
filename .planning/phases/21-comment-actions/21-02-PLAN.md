## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user textarea -> onAddAnnotation -> Annotation.comment | Free-form user input becomes annotation state |
| browser selection state -> handleAdd Range API | Programmatic selection re-enters the same selectionchange flow as manual user selection |
| ContentPane formState transitions | Auto-submit (D-03) and form-cancel paths both manipulate the global `selection-lock` highlight |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-05 | Tampering | Auto-submit (D-03) creating stale annotation | mitigate | `latestFormValueRef.current` is updated on every textarea `onChange`; auto-submit reads the most recent value before opening the new form. Test asserts the ref read is present in `handleAction`. |
| T-21-06 | Repudiation | Form cancel silently discards user input | accept | Documented behavior per D-02 ("Cancelled forms do not save"). UX intent is explicit. |
| T-21-07 | Information Disclosure | Programmatic Range API exposes browser selection state | accept | Same surface as manual selection; no new escape path is introduced. The selectNodeContents call targets a paragraph element already in the user's view. |
| T-21-08 | Denial of Service | Repeated rapid pill clicks triggering auto-submit loop | accept | Each pill click is a synchronous React state transition. React 19 batching ensures auto-submit and new-form-open settle in one render. RESEARCH.md Pitfall 2 documents this is not a race. |
| T-21-09 | Tampering | `selection-lock` highlight not cleared on cancel | mitigate | `handleFormCancel` MUST call `resetTextSelection()`. Acceptance criterion asserts the call is present inside `handleFormCancel` body (and absent from `handleAction`). |
| T-21-SC | Tampering | npm/cargo installs | accept | No new package installs in this plan (verified by Phase 21 RESEARCH.md — Architectural Responsibility Map confirms zero new dependencies). |
