<objective>
Add a `review <file>` subcommand to the `plan-reviewer` CLI that reads any markdown file from disk, renders it in the browser review UI, and outputs neutral `{"behavior":"allow"|"deny"}` JSON to stdout. This enables scripts and agent workflows to invoke plan-reviewer without constructing hook-specific JSON.

Purpose: Make plan-reviewer usable as a standalone markdown review tool, not just a hook handler. Any agent or script can pipe a markdown file through `plan-reviewer review <file>` and parse the neutral JSON output.

Output: Modified `src/main.rs` with the new subcommand + new integration test file proving the end-to-end flow.
</objective>

```rust
use std::net::TcpStream;
use std::process::{Command, Stdio};
use std::time::{Duration, Instant};

fn find_free_port() -> u16 {
    let listener = std::net::TcpListener::bind("127.0.0.1:0").unwrap();
    listener.local_addr().unwrap().port()
}

fn wait_for_port(port: u16, timeout: Duration) -> bool {
    let deadline = Instant::now() + timeout;
    while Instant::now() < deadline {
        if TcpStream::connect(format!("127.0.0.1:{port}")).is_ok() {
            return true;
        }
        std::thread::sleep(Duration::from_millis(50));
    }
    false
}

fn binary_path() -> std::path::PathBuf {
    assert_cmd::cargo::cargo_bin("plan-reviewer")
}
```

```rust
fn spawn_review_flow(file_path: &str, port: u16) -> std::process::Child {
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg -> filesystem | User-supplied file path passed to `std::fs::read_to_string` |
| File content -> HTML render | Markdown content rendered by comrak, sanitized by ammonia |
| Browser -> local server | HTTP on 127.0.0.1 only; user posts approve/deny decision |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.1-01 | Tampering | `run_review_flow` file read | accept | `std::fs::read_to_string` is read-only; path comes from the user's own terminal invocation; no sandboxing needed for a local CLI tool. Symlinks are followed intentionally (standard Unix behavior). |
| T-07.1-02 | Information Disclosure | File content served via `/api/plan` | accept | Server binds to 127.0.0.1 only (verified in server.rs `TcpListener::bind`). Content is only accessible on localhost. Same trust boundary as all existing flows. |
| T-07.1-03 | Elevation of Privilege | Markdown with embedded script tags | mitigate | Comrak renders markdown server-side; ammonia sanitizes the HTML output (already in pipeline from Phase 1 ammonia addition). XSS vectors in markdown content are stripped before browser rendering. No new code path bypasses ammonia. |
| T-07.1-04 | Spoofing | Malicious process posting to /api/decide on localhost | accept | Same risk as all existing flows. Local-only tool; any process running as the same user can POST to the server. Acceptable for a single-user local tool. |
| T-07.1-05 | Denial of Service | Very large markdown file | accept | `read_to_string` loads entire file into memory. For a local CLI tool invoked by the user, this is acceptable — the user controls the input. No file size limit needed. |
</threat_model>
