# Phase 3 Plan 1: Cargo-dist Release Pipeline and Install Script Summary

**One-liner:** cargo-dist 0.31.0 configured for four cross-platform targets with Node.js pre-build injection, ad hoc macOS codesigning, and a custom `curl | sh` install.sh that calls `plan-reviewer install` after binary placement.

## What Was Built

### Task 1: Cargo.toml + release workflow

**Cargo.toml changes:**
- Added `[[bin]]` section with `name = "plan-reviewer"` and `path = "src/main.rs"` — renames the output binary from `claude-plan-reviewer` to `plan-reviewer` (D-03)
- Added `repository` field to `[package]` — required by `dist generate` to emit GitHub CI
- Added `[workspace.metadata.dist]` with all four targets, `install-path = "~/.local/bin"`, `github-build-setup = "../build-setup.yml"`, `install-updater = false` (D-01, D-02, D-04)
- Added `[profile.dist]` with `lto = "thin"` (generated by `dist init`)

**`.github/build-setup.yml`:**
Pre-build steps injected into every `build-local-artifacts` matrix job. Installs Node.js 20 and runs `cd ui && npm install && npm run build` before the Rust compile. Prevents Pitfall 1 (npm not found during cross-compilation).

**`.github/workflows/release.yml`:**
Generated by `cargo dist generate`. Contains: `plan` job, `build-local-artifacts` matrix job (runs on macOS and Ubuntu runners for all four targets), `build-global-artifacts` job, `host` job, `announce` job. Manually patched to add macOS ad hoc codesign step after `dist build` (D-10, DIST-04).

Commit: `6a12dc5`

### Task 2: install.sh

Custom `curl | sh` installer at repo root. Satisfies DIST-02 (no Rust/Node required), DIST-03 (PATH warning), D-04 (installs to `~/.local/bin`), D-09 (calls `plan-reviewer install`).

Platform detection covers all four release targets:
- Darwin arm64 → `aarch64-apple-darwin`
- Darwin x86_64 → `x86_64-apple-darwin`
- Linux x86_64 → `x86_64-unknown-linux-musl`
- Linux aarch64/arm64 → `aarch64-unknown-linux-musl`

`REPO="OWNER/claude-plan-reviewer"` uses a placeholder per T-03-07 (must be set before first release).

Commit: `26faffd`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] `github-build-setup` path resolution**
- **Found during:** Task 1, Step 4 (`dist generate`)
- **Issue:** Plan specified `github-build-setup = ".github/build-setup.yml"` but cargo-dist resolves this path relative to `.github/workflows/`, producing `.github/workflows/.github/build-setup.yml` (not found)
- **Fix:** Changed path to `"../build-setup.yml"` in Cargo.toml; file remains at `.github/build-setup.yml`
- **Files modified:** Cargo.toml
- **Commit:** `6a12dc5`

**2. [Rule 1 - Bug] `working-directory` dropped by github-build-setup injection**
- **Found during:** Task 1, Step 5 (inspecting generated release.yml)
- **Issue:** cargo-dist's `github-build-setup` injection drops the `working-directory:` field from YAML steps; generated release.yml had `npm install && npm run build` without `cd ui`
- **Fix:** Changed `build-setup.yml` to use `cd ui` inline in the `run:` block instead of `working-directory:` field
- **Files modified:** `.github/build-setup.yml`
- **Commit:** `6a12dc5`

**3. [Rule 3 - Blocking] `dist generate` requires `repository` in `[package]`**
- **Found during:** Task 1, Step 4 (`dist init` / `dist generate`)
- **Issue:** `dist generate` fails with "Github CI support requires you to specify the URL of your repository" when `repository` field is absent from `[package]`
- **Fix:** Added `repository = "https://github.com/themouette/claude-plan-reviewer"` to `[package]` in Cargo.toml
- **Files modified:** Cargo.toml
- **Commit:** `6a12dc5`

**4. [Rule 3 - Process] Codesign step re-applied after second `dist generate`**
- **Found during:** Task 1, Step 6 (after fixing build-setup.yml)
- **Issue:** `dist generate` was run a second time to pick up the `cd ui` fix, which overwrote the manually patched codesign step
- **Fix:** Re-applied the codesign step after the second `dist generate` run (as documented in Pitfall 3 of RESEARCH.md)
- **Files modified:** `.github/workflows/release.yml`
- **Commit:** `6a12dc5`
