Create `install.sh` at the repo root. This is the file users run with `curl -fsSL https://raw.githubusercontent.com/OWNER/REPO/main/install.sh | sh`.

**Requirements the script must satisfy:**
- DIST-02: Installs without Rust, Node.js, or any runtime (downloads pre-built binary directly)
- DIST-03: Detects if `~/.local/bin` is on PATH and warns clearly if not
- D-04: Default install location is `~/.local/bin`
- D-09: Calls `plan-reviewer install` after placing the binary

**Full script content:**

```sh
#!/bin/sh
# install.sh — plan-reviewer installer
# Installs the plan-reviewer binary to ~/.local/bin and wires the Claude Code hook.
# Usage: curl -fsSL https://raw.githubusercontent.com/OWNER/REPO/main/install.sh | sh
set -eu

REPO="OWNER/claude-plan-reviewer"
INSTALL_DIR="${HOME}/.local/bin"
BINARY="plan-reviewer"

# --- Platform detection ---
OS="$(uname -s)"
ARCH="$(uname -m)"

case "${OS}" in
  Darwin)
    case "${ARCH}" in
      arm64)  TARGET="aarch64-apple-darwin" ;;
      x86_64) TARGET="x86_64-apple-darwin" ;;
      *)      echo "Unsupported macOS architecture: ${ARCH}" >&2; exit 1 ;;
    esac
    ;;
  Linux)
    case "${ARCH}" in
      x86_64)          TARGET="x86_64-unknown-linux-musl" ;;
      aarch64|arm64)   TARGET="aarch64-unknown-linux-musl" ;;
      *)               echo "Unsupported Linux architecture: ${ARCH}" >&2; exit 1 ;;
    esac
    ;;
  *)
    echo "Unsupported operating system: ${OS}" >&2
    exit 1
    ;;
esac

# --- Resolve latest release tag ---
echo "Detecting latest release..."
LATEST_TAG="$(curl -fsSL "https://api.github.com/repos/${REPO}/releases/latest" \
  | grep '"tag_name"' \
  | sed 's/.*"tag_name": *"\([^"]*\)".*/\1/')"

if [ -z "${LATEST_TAG}" ]; then
  echo "Could not determine latest release tag from GitHub API." >&2
  exit 1
fi

echo "Installing ${BINARY} ${LATEST_TAG} for ${TARGET}..."

# --- Download and extract ---
ARCHIVE_NAME="${BINARY}-${LATEST_TAG}-${TARGET}.tar.xz"
DOWNLOAD_URL="https://github.com/${REPO}/releases/download/${LATEST_TAG}/${ARCHIVE_NAME}"

TMPDIR="$(mktemp -d)"
trap 'rm -rf "${TMPDIR}"' EXIT

echo "Downloading ${DOWNLOAD_URL}..."
curl -fsSL "${DOWNLOAD_URL}" -o "${TMPDIR}/${ARCHIVE_NAME}"

echo "Extracting..."
tar -xJf "${TMPDIR}/${ARCHIVE_NAME}" -C "${TMPDIR}"

# --- Install binary ---
mkdir -p "${INSTALL_DIR}"
cp "${TMPDIR}/${BINARY}-${LATEST_TAG}-${TARGET}/${BINARY}" "${INSTALL_DIR}/${BINARY}"
chmod +x "${INSTALL_DIR}/${BINARY}"
echo "Installed ${BINARY} to ${INSTALL_DIR}/${BINARY}"

# --- PATH check (DIST-03) ---
case ":${PATH}:" in
  *":${INSTALL_DIR}:"*)
    # Already on PATH — no action needed
    ;;
  *)
    echo ""
    echo "WARNING: ${INSTALL_DIR} is not on your PATH."
    echo "To add it, run one of the following depending on your shell:"
    echo ""
    echo "  bash/zsh:   echo 'export PATH=\"\${HOME}/.local/bin:\${PATH}\"' >> ~/.profile"
    echo "  fish:       fish_add_path ~/.local/bin"
    echo ""
    echo "Then restart your shell or run: export PATH=\"\${HOME}/.local/bin:\${PATH}\""
    echo ""
    ;;
esac

# --- Wire Claude Code hook (D-09) ---
echo "Wiring Claude Code ExitPlanMode hook..."
"${INSTALL_DIR}/${BINARY}" install

echo ""
echo "plan-reviewer ${LATEST_TAG} installed successfully."
echo "Run '${BINARY} --help' to get started."
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| install.sh → GitHub API | Script fetches release tag from api.github.com — response is trusted implicitly |
| install.sh → GitHub Releases | Script downloads binary archive from releases.github.com — no signature verification |
| plan-reviewer install → ~/.claude/settings.json | Binary writes to user's Claude Code config — must not corrupt existing config |
| release.yml → GitHub Actions runner | CI executes codesign on the built binary — runner environment is trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | install.sh download | accept | Binary is downloaded over HTTPS from github.com (TLS + cert-pinning by curl). No checksum verification in the script — acceptable for a developer tool targeting power users who can inspect the script. DIST-06 (full notarization) in v2 adds OS-level integrity check. |
| T-03-02 | Tampering | GitHub API tag resolution | accept | Tag resolved over HTTPS. An attacker controlling the DNS of api.github.com could serve a malicious tag; this threat is shared with all curl-install patterns and accepted as industry norm for developer tools. |
| T-03-03 | Elevation of Privilege | install.sh chmod +x | mitigate | Binary installed to `~/.local/bin` (user-writable, no sudo). Does not touch system directories. No elevation. |
| T-03-04 | Tampering | settings.json write in plan-reviewer install | mitigate | Read-modify-write pattern preserves all existing keys. Idempotency check prevents duplicate hook entries. `serde_json::from_str` falls back to empty object on parse failure (Pitfall 4 in RESEARCH.md accepted). Executor must implement: read → parse → merge → write_pretty. |
| T-03-05 | Information Disclosure | current_exe() in install subcommand | accept | `std::env::current_exe()` returns the absolute path of the binary. This path is written to `~/.claude/settings.json`, a user-owned local file. No information leaves the machine. |
| T-03-06 | Denial of Service | release.yml codesign step | mitigate | `codesign --force --sign -` (ad hoc) does not require network access or Apple certificates. Failure would only block the release — not a runtime DoS. The step includes error visibility via `echo "Signed: $f"` for each file. |
| T-03-07 | Spoofing | install.sh OWNER placeholder | mitigate | The REPO variable in install.sh must be set to the correct GitHub org/user before release. Executor must replace `OWNER` with the actual value. Document this in the script header comment. |
</threat_model>
