# Phase 27 — UI Design Contract

> Visual and interaction contract for Phase 27: Inline Comments.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — inline CSS with CSS custom property tokens |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between icon buttons in bubble footer; margin between type tag and controls |
| sm | 8px | Textarea internal padding; bubble card padding (8px 12px); button internal padding; gap between Submit/Cancel buttons |
| md | 16px | File header row padding (8px 16px); diff pane section padding; file list item padding |
| lg | 24px | Depth indent step in file list (`16 + depth * 16`) |
| xl | 32px | — |
| 2xl | 48px | Empty/error state top padding |
| 3xl | 64px | — |

Exceptions:
- Comment gutter `+` button: 20×20px minimum tap target (matches existing `bubble-icon-btn` 20×20px pattern from `CommentBubble.tsx`).
- File comment trigger button in file header: 24px height (matches existing prev/next navigation buttons in `DiffPane.tsx`).
- Textarea minimum height: 64px (established in `AnnotationForm.tsx` and `CommentBubble.tsx`).

Source: `ui/src/code-review/DiffPane.tsx` (file header `padding: 8px 16px`, button `height: 24`); `ui/src/reviewer-v2/CommentBubble.tsx` (bubble padding, icon button 20×20); `ui/src/reviewer-v2/AnnotationForm.tsx` (textarea 64px min-height).

## Typography

|------|------|--------|-------------|

## Color

| Role | Token | Dark mode value | Light mode value | Usage |
|------|-------|----------------|-----------------|-------|
| Dominant (60%) | `--color-bg` | `#0f1117` | `#f8fafc` | Page background, diff pane background |
| Secondary (30%) | `--color-surface` | `#1a1d27` | `#f1f5f9` | Comment card background, file header background, form background, stats strip |
| Border | `--color-border` | `#2d3148` | `#cbd5e1` | Card borders, textarea border, separator lines |
| Text primary | `--color-text-primary` | `#f1f5f9` | `#0f172a` | Comment text, file basename, form label |
| Text secondary | `--color-text-secondary` | `#94a3b8` | `#64748b` | Timestamp, file path, placeholder, icon button default color, gutter `+` button |
| Accent (10%) | `--color-focus` | `#3b82f6` | `#2563eb` | See reserved list below |
| Destructive | `--color-accent-deny` | `#ef4444` | `#dc2626` | Delete icon button hover state; delete action |

Accent (`--color-focus`) reserved for:
1. File list badge background (comment count per file, per D-10)
2. Gutter `+` button border/text on hover (line comment trigger affordance)
3. File comment trigger button background on hover
4. Comment form Submit button background (`var(--color-focus)`)
5. Focus ring on all interactive elements (`outline: 2px solid var(--color-focus)`)
6. Active file indicator in `FileListPane` (left border, already established)

Destructive (`--color-accent-deny`) reserved for: delete icon button hover color only. No confirmation dialog — delete is immediate (per D-05/CONTEXT.md).

Source: `ui/src/index.css` (full token definitions); `ui/src/code-review/DiffPane.tsx` (established surface/border usage); `ui/src/code-review/FileListPane.tsx` (focus border, active state); CONTEXT.md D-06, D-10.

## Component Inventory

New components introduced in Phase 27 (under `ui/src/code-review/`):

### `HunkCommentForm` (`HunkCommentForm.tsx`)

Inline textarea form rendered via `renderAnnotation` (line comments) or in the file section header area (file comments). GitHub-style — no popover, no overlay.

Structure:
```
<div role="group" aria-label="Add a comment">
  <textarea
    autoFocus
    placeholder="Add a comment…"
    aria-label="Comment text"
    minHeight=64px
    width=100%
    resize=vertical
    fontFamily=inherit
    fontSize=14px
    background=var(--color-surface)
    color=var(--color-text-primary)
    border=1px solid var(--color-border)
    borderRadius=4px
    padding=8px
  />
  <div justifyContent=flex-end gap=8px marginTop=8px>
    <button "Cancel"
      height=28px padding=0 8px fontSize=13px fontWeight=400
      background=transparent color=var(--color-text-secondary)
      border=none borderRadius=4px
      focus: outline=2px solid var(--color-focus) outlineOffset=2px
    />
    <button "Add Comment"
      height=28px padding=0 12px fontSize=13px fontWeight=600
      background=var(--color-focus) color=#fff
      border=none borderRadius=4px
      focus: outline=2px solid var(--color-focus) outlineOffset=2px
    />
  </div>
</div>
```

Keyboard contract:
- `Cmd+Enter` / `Ctrl+Enter`: submit
- `Escape`: cancel / close form

Source: `ui/src/reviewer-v2/AnnotationForm.tsx` (established pattern — adapted for inline diff context, not floating/fixed positioned).

### `CommentBubble` (`CommentBubble.tsx`)

Submitted comment card rendered inline below the annotated line (via `renderAnnotation`) or above the diff content (file-level comments).

Structure:
```
<article aria-label="Comment on line {N}">
  border=1px solid var(--color-border)
  borderLeft=3px solid var(--color-focus)   ← accent left border, always blue (single type)
  borderRadius=6px
  padding=8px 12px
  background=var(--color-surface)

  <p comment text fontSize=14px fontWeight=400 lineHeight=1.5 color=var(--color-text-primary) />
  <span timestamp fontSize=12px color=var(--color-text-secondary) marginTop=4px />

  <div justifyContent=flex-end gap=4px marginTop=8px>
    <button aria-label="Edit comment" ✎
      width=20px height=20px fontSize=14px
      border=none background=transparent borderRadius=3px
      color=var(--color-text-secondary)
      hover: color=var(--color-text-primary)
      focus: outline=2px solid var(--color-focus)
      class=bubble-icon-btn (transition: color 0.1s ease)
    />
    <button aria-label="Delete comment" ×
      width=20px height=20px fontSize=16px
      border=none background=transparent borderRadius=3px
      color=var(--color-text-secondary)
      hover: color=var(--color-accent-deny)
      focus: outline=2px solid var(--color-focus)
      class=bubble-icon-btn (transition: color 0.1s ease)
    />
  </div>

  [when editing — replaces text + controls]
  <textarea ... (same spec as HunkCommentForm textarea) />
  <div justifyContent=flex-end gap=4px marginTop=8px>
    <button "Cancel" height=28px ... />
    <button "Save Changes" height=28px background=var(--color-focus) ... />
  </div>
</article>
```

Difference from `reviewer-v2/CommentBubble.tsx`: no `position: absolute`, no `top` prop, no compact/expanded collapse states, no bidirectional hover cross-highlight. This is a static inline card — no floating layout needed (per RESEARCH.md "State of the Art").

Source: `ui/src/reviewer-v2/CommentBubble.tsx` (icon characters, button dimensions, color tokens, edit/delete pattern); CONTEXT.md D-05 ("reviewer-v2-style bubble").

### Gutter `+` Button (inline in `DiffPane.tsx` via `renderGutterUtility`)

```
<button type="button" aria-label="Add comment to this line"
  width=20px height=20px
  background=transparent border=none
  borderRadius=3px cursor=pointer
  color=var(--color-text-secondary)
  fontSize=14px fontWeight=600
  display=flex alignItems=center justifyContent=center
  hover: color=var(--color-focus) background=var(--color-annotation-highlight)
  focus: outline=2px solid var(--color-focus) outlineOffset=2px
>
  +
</button>
```

Visibility: rendered by `renderGutterUtility` — @pierre/diffs shows the gutter element on line hover internally. The button is always mounted but visually appears only when a line is hovered.

### File Comment Trigger Button (inline in `DiffPane.tsx` file header)

Placed at the trailing end of the file header row (after the additions/deletions stats), before the chevron group.

```
<button type="button" aria-label="Add file-level comment"
  height=24px padding=0 10px
  fontSize=12px fontWeight=400
  background=transparent
  border=1px solid var(--color-border)
  borderRadius=4px cursor=pointer
  color=var(--color-text-secondary)
  hover: color=var(--color-text-primary) borderColor=var(--color-focus)
  focus: outline=2px solid var(--color-focus) outlineOffset=2px
>
  + Comment
</button>
```

### File List Badge (inline in `FileListPane.tsx` `renderFileNode`)

Appears after the change counts span when `commentCounts[filename] > 0`. Zero-comment files show no badge.

```
<span
  background=var(--color-focus) color=#fff
  borderRadius=10px fontSize=11px fontWeight=600
  padding=1px 6px marginLeft=6px
  flexShrink=0
>
  {count}
</span>
```

Source: CONTEXT.md D-10 (badge spec literal), RESEARCH.md Code Example "Badge in FileListPane".

## Interaction States

### Gutter `+` Button States

| State | Visual |
|-------|--------|
| Default | Hidden (not visible — the gutter renders it but @pierre/diffs controls hover-show) |
| Line hovered | `color: var(--color-text-secondary)`, visible in gutter |
| Button hovered | `color: var(--color-focus)`, `background: var(--color-annotation-highlight)` |
| Focus-visible | `outline: 2px solid var(--color-focus)` |

### HunkCommentForm States

| State | Visual |
|-------|--------|
| Open | Rendered inline below the target line (via `renderAnnotation`) with `autoFocus` on textarea |
| Submitting (empty) | Submit button remains active — empty comments are allowed (match reviewer-v2 pattern) |
| Closed | Component unmounted; `pendingLineAnchor` state in `DiffPane` set to `null` |

### CommentBubble States

| State | Visual |
|-------|--------|
| Default | Card with left border accent, text, timestamp, edit/delete icon buttons visible |
| Edit mode | Textarea replaces text; "Save Changes" and "Cancel" buttons replace icon buttons |
| Deleting | Immediate removal — no confirmation dialog |

### File Comment Trigger Button States

| State | Visual |
|-------|--------|
| Default | Ghost button (`+ Comment`) in file header trailing area |
| Hovered | `color: var(--color-text-primary)`, `border-color: var(--color-focus)` |
| Form open | Button hidden / form rendered between file header and diff content |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — line comment form | **Add Comment** |
| Primary CTA — file comment form | **Add Comment** (same — context disambiguated by placement) |
| Cancel action (forms) | **Cancel** |
| Edit save action | **Save Changes** |
| Edit cancel action | **Cancel** |
| Gutter button label | `aria-label="Add comment to this line"` |
| File comment trigger label | `aria-label="Add file-level comment"` |
| Textarea placeholder | **Add a comment…** |
| Edit textarea placeholder | (no placeholder — pre-filled with existing text) |
| Edit button aria-label | **Edit comment** |
| Delete button aria-label | **Delete comment** |
| File list badge aria | No aria-label on badge span — parent button's title carries full filename context |
| Empty comment pane | No empty state UI needed — `+` gutter button is the discovery surface |
| Error state | Not applicable — comments are session state only, no fetch errors |
| Destructive action: delete | Immediate; no confirmation copy needed (per CONTEXT.md D-05: "clicking × removes the bubble immediately with no confirmation dialog") |

Keyboard shortcuts to surface in tooltips (or aria-describedby if practical):
- Form submit: `Cmd+Enter` / `Ctrl+Enter`
- Form cancel: `Escape`

Source: `ui/src/reviewer-v2/AnnotationForm.tsx` ("Add a comment…" placeholder, "Dismiss"/"Post Comment" — adapted for code review context); CONTEXT.md COMMENT-03 success criteria ("clicking × removes the bubble immediately with no confirmation dialog").

## Architecture Constraints (for executor reference)

These are not UI design decisions — they enforce correctness and prevent regressions:

1. **MUST NOT import from `reviewer-v2/`** — ESLint `no-restricted-imports` rule enforced. Copy patterns into `code-review/` directly.
2. **New components live under `ui/src/code-review/`** — not in `reviewer-v2/` or any shared directory.
3. **`pendingLineAnchor` state stays in `DiffPane`**, not lifted to `CodeReviewApp` — keeps `CodeReviewApp`'s `comments` array clean of transient state (per RESEARCH.md Pattern 4 recommendation).
4. **`commentCounts` derived from submitted comments only** — never from pending state (Pitfall 4 in RESEARCH.md).
5. **`renderAnnotation` must return `ReactNode`** — import `FileDiff` from `@pierre/diffs/react`, not bare `@pierre/diffs` (Pitfall 1 in RESEARCH.md).
6. **`getHoveredLine()` called inside click handler**, not at render time (Pitfall 3 in RESEARCH.md).
7. **Line annotations disabled for `PatchDiff` path** — binary files and files without `old_content`/`new_content` cannot use `lineAnnotations`. File-level comment button still appears in the header (Pitfall 2 in RESEARCH.md + open question 2 answer).
