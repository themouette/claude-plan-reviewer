<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| annotate_content string -> disk | install() writes the string to commands/annotate.md; no user input crosses this boundary |
| commands/annotate.md -> Claude Code | Claude Code reads the file and registers the slash command; content is trusted (written by install()) |
| Bash tool invocation -> plan-reviewer review | The prompt instructs Claude to run plan-reviewer review with a file path; path comes from conversation history or $ARGUMENTS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | annotate_content string in Rust binary | accept | String is a compile-time constant in the binary; only writable by running plan-reviewer install claude. No runtime injection path. |
| T-11-02 | Information Disclosure | Temp file written by Claude for fallback (SLSH-04) | accept | Temp file is written to /tmp/ by Claude at user request; content is the last Claude response, which the user has already seen. No new disclosure surface. |
| T-11-03 | Elevation of Privilege | Path argument to plan-reviewer review (Rule A) | accept | $ARGUMENTS comes from the user typing the command; they are passing their own file path to their own local binary. No privilege escalation vector. |
| T-11-04 | Spoofing | Malicious .md path injected via conversation history (Rule B) | accept | Claude selects the path from conversation history of the current session, which the user controls. An adversarial path could only come from the user's own earlier tool calls. Local-only tool; acceptable risk. |
</threat_model>
