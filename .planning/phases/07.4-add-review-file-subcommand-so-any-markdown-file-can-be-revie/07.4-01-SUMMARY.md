# Phase 07.4 Plan 01: Add review <file> Subcommand Summary

**`plan-reviewer review <file.md>` subcommand reads any markdown file, opens the browser review UI, and outputs neutral `{"behavior":"allow"|"deny"}` JSON — no hook JSON construction needed**

## Performance

- **Duration:** ~21 min
- **Started:** 2026-04-11T06:00:00Z
- **Completed:** 2026-04-11T06:21:00Z
- **Tasks:** 2
- **Files modified:** 3

## Accomplishments

- Added `Commands::Review { file: String }` variant to the CLI enum with doc comment
- Added `run_review_flow` function that reads a file from disk (never stdin), runs the browser review UI, and writes neutral `{"behavior": ...}` JSON to stdout via `build_opencode_output`
- Added debug-mode frontend asset guard to `run_review_flow` (closing a gap — `run_opencode_flow` was missing this guard)
- Added 4 integration tests covering approve, deny, missing-file error, and `/api/plan` content serving
- Added 2 unit tests for CLI parsing with and without file argument

## Task Commits

1. **Task 1: Commands::Review variant and run_review_flow** - `15ad9c8` (feat)
2. **Task 2: Integration tests for review subcommand** - `4789c70` (feat)

## Files Created/Modified

- `src/main.rs` — Added `Commands::Review`, `run_review_flow`, dispatch arm, 2 unit tests
- `tests/integration/review_subcommand.rs` — 4 integration tests (created)
- `tests/integration/main.rs` — Added `mod review_subcommand`

## Decisions Made

- Reused `build_opencode_output` for neutral JSON output — the format is identical (`{"behavior":"allow"|"deny","message":"..."}`) so no new function was needed
- Set `stdin(Stdio::null())` in the test helper to explicitly enforce and document the no-stdin contract
- Added the debug asset guard to `run_review_flow` as a deviation fix (Rule 2) since `run_opencode_flow` was missing it and RESEARCH.md Pitfall 4 had flagged it
