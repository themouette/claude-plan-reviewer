  <how-to-verify>
Launch the binary with a test plan (Task 1 starts it). Open the review URL printed to stderr.

**Test 1 -- Layout (D-01):**
- Confirm two-column layout: plan content on left, sidebar panel on right
- Confirm sidebar is approximately 320px wide with border-left separator
- Confirm header shows "Plan Review" on left and "Plan | Diff" tabs on right

**Test 2 -- Tab switching (D-05):**
- Click "Diff" tab -- plan content is replaced with diff view
- Click "Plan" tab -- plan content returns
- Confirm URL does not change when switching tabs

**Test 3 -- Diff view (D-06, D-07, DIFF-01/02/03):**
- Click Diff tab -- if the working tree has changes, a syntax-highlighted unified diff should appear
- If no changes: "No changes in working tree" empty state message
- Confirm diff has line numbers and color-coded additions/deletions

**Test 4 -- Global comment (D-04, ANN-04):**
- Confirm "Overall Comment" field is at top of sidebar, always visible
- Type something in the Overall Comment field
- Confirm it persists when switching tabs

**Test 5 -- Text selection and annotation creation (D-02, ANN-01/02/03):**
- Select text in the plan
- Confirm three pills appear in sidebar: "Comment", "Delete", "Replace"
- Click "Comment" -- a comment annotation card appears with the selected text quoted
- Select different text, click "Delete" -- a delete annotation card appears
- Select different text, click "Replace" -- a replace annotation card with replacement textarea

**Test 6 -- Annotation card hover highlight (D-03):**
- Hover over an annotation card -- the corresponding text in the plan should highlight (blue tint)
- Move mouse away -- highlight disappears

**Test 7 -- Deny with annotations (D-08, D-09, ANN-05):**
- With annotations present, click "Deny"
- Confirm the "Submit Denial" button is enabled even with an empty textarea
- Click "Submit Denial"
- Check stdout JSON: the `message` field should contain structured markdown with "## Annotations" section

**Test 8 -- Enter-to-approve suppression:**
- Focus a textarea in the sidebar, press Enter -- should NOT approve
- Focus outside sidebar (e.g., click plan area), press Enter -- should approve

Report any issues found.
